💡 配置说明

支持的 LLM 提供商（任何 OpenAI 兼容 API）：

  • OpenAI — https://api.openai.com/v1
  • CodeBuddy — 公司内部 API 地址
  • DeepSeek — https://api.deepseek.com/v1
  • Moonshot — https://api.moonshot.cn/v1
  • 其他 OpenAI 兼容服务...

降级模式：如果未配置 API Key 或 LLM 不可用，系统会自动使用模板引擎生成结果，功能不受影响。

安全说明：运行时配置仅在当前进程生效，重启后需要重新配置。建议在 backend/.env 文件中持久化配置。

使用指引

🚀 快速上手（3 分钟）

1
提交需求

点击侧边栏 「📝 提交需求」，在文本框中用自然语言描述你想要开发的系统。

示例输入：
"开发一个用户管理 API 系统，包含登录注册和数据库"
2
查看任务分解

系统自动将需求分解为多个开发任务，按 需求分析 → 架构设计 → 开发实现 → 测试验证 → 部署上线 五个阶段组织。

提交后会自动跳转到项目详情页，展示所有分解出的任务。

3
执行与跟踪

在项目详情页点击 「▶ 执行下一个任务」，系统会按依赖顺序逐个执行任务。每 8 秒自动刷新状态。

❓ 常见问题

Q: 需求描述越详细越好吗？
A: 是的。描述中包含的关键词越多（如 API、数据库、前端、登录等），系统能识别出的功能模块就越多，生成的任务也越完整。
Q: 技术栈偏好和项目名称是必填的吗？
A: 不是。这两个字段是可选的，不填也不影响任务分解。技术栈偏好会记录在项目信息中，方便后续开发参考。
Q: 任务执行失败怎么办？
A: 失败的任务会标红显示。当前版本的任务执行是模拟执行（标记状态变化），后续版本会接入真正的 AI Agent 实现代码生成。
Q: 数据保存在哪里？
A: 当前版本数据保存在后端内存中，重启服务会丢失。后续将支持数据库持久化。
Q: 怎么查看 API 详细文档？
A: 访问 /docs 查看自动生成的 Swagger UI 交互文档，可以直接在线调试接口。
