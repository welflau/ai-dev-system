# 工单统一时间轴 + 评论功能 实现方案

## 版本历史

| 版本 | 日期 | 内容 |
|---|---|---|
| v1.0 | 2026-06-16 | 初版：统一时间轴、人工评论保存、Agent 阶段总结、评论注入 prompt |
| v1.1 | 2026-06-16 | 新增：评论后 Agent 立刻回复，形成真正的对话（参考 Multicast 交互模式） |

---

## Context

当前工单抽屉中，产出物、操作日志、阶段总结、Agent 讨论分散在 4 个独立区块，用户需要上下翻动才能拼出工单完整经历。
本次改造将这四类信息合并为一条**统一时间轴**，并在其中嵌入人工评论入口：
- 用户可随时针对任意阶段发表指令，Agent 执行时优先读取人工评论，保证工单不偏离目标
- **v1.1 新增**：用户发评论后，当前负责的 Agent 立刻异步回复，形成真正的对话闭环

## 1. 数据库层

### 新增 `ticket_comments` 表（database.py）

在现有建表语句后追加：

```sql
CREATE TABLE IF NOT EXISTS ticket_comments (
    id          TEXT PRIMARY KEY,
    ticket_id   TEXT NOT NULL REFERENCES tickets(id),
    project_id  TEXT NOT NULL REFERENCES projects(id),
    author      TEXT NOT NULL,       -- "你" | "DevAgent" | "PlannerAgent"...
    author_type TEXT NOT NULL,       -- "human" | "agent" | "system"
    content     TEXT NOT NULL,
    phase       TEXT,                -- 可选：关联阶段标记（如 "dev_done"）
    created_at  TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_ticket_comments_ticket
    ON ticket_comments(ticket_id, created_at);
```

**不动 tickets 主表**，零迁移风险，向后完全兼容。

## 2. 后端 API 层（backend/api/tickets.py）

### 新增 3 个端点

#### 2a. 统一时间轴端点

```
GET /api/tickets/{ticket_id}/timeline
```

逻辑：
1. 查 `ticket_logs` WHERE ticket_id = ? ORDER BY created_at ASC（全量，不限条数）
2. 查 `ticket_comments` WHERE ticket_id = ? ORDER BY created_at ASC
3. Python 内存合并按 created_at 排序，每条加 `_type` 字段（`"log"` | `"comment"`）
4. 对 ticket_logs 中的 `detail` JSON 做解析，提取 `message`、`phase_summary`、`artifact_files` 等关键字段，方便前端直接使用
5. 返回统一列表

**信息分级**（在返回数据中加 `_tier` 字段）：
- `primary`：状态变更（from_status ≠ null）、阶段总结（action="phase_complete"）、人工评论、产出文件
- `secondary`：LLM 工具调用（action 以 "chat:" 开头）、详细编译日志

#### 2b. 新增评论

```
POST /api/tickets/{ticket_id}/comments
Body: { "content": "...", "phase": "dev_done"（可选）}
```

- 写入 ticket_comments 表，author="你"，author_type="human"
- 同时触发 SSE 事件 `comment_added`，推送给前端实时刷新

#### 2c. 删除评论

```
DELETE /api/tickets/{ticket_id}/comments/{comment_id}
```

仅允许删除 author_type="human" 的评论。

## 3. Agent 上下文注入（backend/orchestrator.py）

在 `process_ticket()` 方法构建 prompt 上下文后，注入近期人工评论：

```python
# 查询该工单所有人工评论（正序）
human_comments = await db.fetch_all(
    "SELECT author, content, created_at FROM ticket_comments "
    "WHERE ticket_id = ? AND author_type = 'human' ORDER BY created_at ASC",
    (ticket_id,)
)

if human_comments:
    lines = "\n".join(
        f"- [{c['created_at'][:16]}] {c['content']}"
        for c in human_comments
    )
    context["human_instructions"] = (
        "## ⚠️ 人工指令（最高优先级，必须严格遵循）\n"
        "以下是人工在工单中留下的评论和指令，执行时必须优先遵循，不得偏离：\n"
        + lines
    )
```

**关键**：人工评论作为 prompt 的第一个区块，优先级高于 Agent 自身判断。

### Agent 阶段完成后自动写总结评论

新增 `_write_phase_comment()` 辅助方法，在 PlannerAgent、ArchitectAgent、DevAgent、UXAgent、ArtAgent 各阶段完成时调用：

```python
async def _write_phase_comment(
    self, project_id, ticket_id, agent_type, summary, phase=None
):
    """Agent 阶段完成后写一条 agent 类型评论，供时间轴展示和下游 Agent 感知"""
    await db.insert("ticket_comments", {
        "id": "CMT-" + generate_id(),
        "ticket_id": ticket_id,
        "project_id": project_id,
        "author": agent_type,
        "author_type": "agent",
        "content": summary,
        "phase": phase,
        "created_at": now_iso(),
    })
    await event_manager.publish_to_project(project_id, "comment_added", {...})
```

### 4d. SSE 实时更新

```javascript
// comment_added 事件 → 刷新时间轴
eventSource.addEventListener('comment_added', (e) => {
    const data = JSON.parse(e.data);
    if (data.ticket_id) refreshTimeline(data.ticket_id);
});

// ticket_status_changed 事件也触发时间轴刷新
eventSource.addEventListener('ticket_status_changed', (e) => {
    const data = JSON.parse(e.data);
    if (data.ticket_id) refreshTimeline(data.ticket_id);
});
```

## 7. Agent 立刻回复评论（v1.1）

### 交互模式

```
用户发评论「这是完成了？」
        ↓ POST /comments 保存后立刻触发 BackgroundTask
Agent 读取：工单背景 + 完整评论历史 + 最新评论
        ↓ LLM 生成回复（2-4句，针对评论内容）
Agent 写入回复评论（author_type="agent"）
        ↓ SSE comment_added 推送
前端时间轴自动刷新，蓝色气泡出现
```

### 触发点（backend/api/tickets.py）

```python
@router.post("/api/tickets/{ticket_id}/comments")
async def add_ticket_comment(ticket_id, body, background_tasks: BackgroundTasks):
    # ... 保存评论 ...
    # 立刻异步触发 Agent 回复（不阻塞接口响应）
    background_tasks.add_task(
        orchestrator.reply_to_comment,
        ticket_id=ticket_id,
        project_id=ticket["project_id"],
        comment_content=content,
    )
    return {"id": comment_id, "created_at": created_at}
```

### 新方法 `reply_to_comment()`（backend/orchestrator.py）

```python
async def reply_to_comment(self, ticket_id, project_id, comment_content):
    """用户发评论后，由当前负责 Agent（或 ChatAssistant）立刻回复"""
    ticket = await db.fetch_one("SELECT * FROM tickets WHERE id=?", (ticket_id,))
    if not ticket:
        return

    # 读取完整评论历史作为对话上下文
    history = await db.fetch_all(
        "SELECT author, author_type, content, created_at "
        "FROM ticket_comments WHERE ticket_id=? ORDER BY created_at ASC",
        (ticket_id,)
    )

    # 选择回复的 Agent：优先用当前负责的 Agent，否则用 ChatAssistant
    agent_name = ticket.get("assigned_agent") or "ChatAssistant"
    agent = self.agents.get(agent_name) or self.agents.get("ChatAssistant")
    if not agent:
        return

    # 构建 prompt
    history_text = "\n".join(
        f"[{'用户' if c['author_type'] == 'human' else c['author']}] {c['content']}"
        for c in history[:-1]  # 不含最新那条（已在下方单独展示）
    )
    prompt = f"""你是 {agent_name}，正在处理工单「{ticket['title']}」。
当前状态：{ticket['status']}
工单描述：{(ticket.get('description') or '')[:300]}

== 历史对话 ==
{history_text or '（无）'}

== 用户最新评论 ==
{comment_content}

请针对用户评论简洁回复（2-4句）。
如涉及修改要求，说明你将如何处理；如是询问进展，直接说明当前状态和产出。"""

    try:
        # 调用 LLM（复用 llm_client，不走完整 Agent 流程）
        from llm_client import llm_client
        reply_text = await llm_client.simple_chat(prompt)
        if reply_text:
            await self._write_phase_comment(
                project_id, ticket_id, agent_name, reply_text.strip()
            )
    except Exception as e:
        logger.warning("reply_to_comment 失败: %s", e)
```
