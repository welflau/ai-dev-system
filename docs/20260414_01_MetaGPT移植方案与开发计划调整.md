## 一、移植目标

从 MetaGPT 移植 4 个核心模式到 AI Dev System，提升 Agent 自主性和输出质量：

| 优先级 | 模式 | MetaGPT 来源 | 我们的现状 | 移植价值 |
|--------|------|-------------|----------|---------|
| P0 | ActionNode 结构化输出 | `action_node.py` | 手动 json.loads + try/except | 减少 LLM 输出解析失败 |
| P1 | Role 内部状态机 | `role.py _think/_act` | Agent 无状态，外部驱动 | Agent 可自主决策下一步 |

## 二、MetaGPT 核心代码分析

### 2.1 ActionNode — 结构化输出（最值得移植）

MetaGPT 的 ActionNode 解决了一个核心问题：**LLM 输出不可靠，如何确保输出符合预期结构？**

```python
# MetaGPT: action_node.py 核心流程
class ActionNode:
    key: str                    # 节点名
    expected_type: Type         # 期望输出类型
    instruction: str            # LLM 提示词
    schema: str                 # "json" / "markdown" / "raw"
    instruct_content: BaseModel # 结构化输出结果

    async def fill(self, req, llm, schema="json"):
        """调用 LLM → 解析输出 → 验证 schema → 存入 instruct_content"""
        context = self.compile(req, schema)     # 编译 prompt（含格式约束）
        content = await llm.aask(context)        # 调 LLM
        parsed = parse_output(content, schema)   # 解析（json/markdown）
        output_class = create_model_class(...)   # 动态生成 Pydantic 模型
        self.instruct_content = output_class(**parsed)  # 验证并存储
        return self
```

**关键设计**：
- `compile()`: 自动在 prompt 末尾加格式约束（JSON schema / Markdown template）
- `create_model_class()`: 运行时动态生成 Pydantic 模型，类型安全
- 解析失败有 fallback（markdown → json → raw）
- 输出结果是类型化的 `BaseModel`，不是裸 dict

**我们的痛点**：
```python
# 我们现在的做法（fragile）
result = await llm_client.chat_json([...])  # 返回 dict 或 None
if result and isinstance(result, dict) and result.get("files"):  # 手动检查
    files = result["files"]  # 可能不是 dict
```

**移植后**：
```python
# 移植 ActionNode 后
node = ActionNode(
    key="develop",
    expected_type=DevResult,  # Pydantic 模型
    instruction="根据架构实现功能",
    schema="json",
)
result = await node.fill(req=context, llm=llm_client)
# result.instruct_content 是类型安全的 DevResult 对象
# result.instruct_content.files: Dict[str, str]  ← 保证存在且类型正确
```

### 2.2 Role 内部状态机（_think → _act 循环）

MetaGPT 的 Role 有三种执行模式：

```python
# MetaGPT: role.py
class RoleReactMode(Enum):
    REACT = "react"         # LLM 动态选择下一步 Action
    BY_ORDER = "by_order"   # 按顺序执行所有 Action
    PLAN_AND_ACT = "plan_and_act"  # 先规划再执行

class Role:
    async def _think(self) -> bool:
        """决定下一步做什么"""
        if self.rc.react_mode == BY_ORDER:
            self._set_state(self.rc.state + 1)    # 顺序推进
        elif self.rc.react_mode == REACT:
            next_state = await self.llm.aask(...)  # LLM 决定
        elif self.rc.react_mode == PLAN_AND_ACT:
            await self.planner.plan(...)           # 规划子任务

    async def _act(self) -> Message:
        """执行当前 Action"""
        response = await self.rc.todo.run(self.rc.history)
        msg = Message(content=response, cause_by=self.rc.todo)
        self.rc.memory.add(msg)
        return msg

    async def _react(self) -> Message:
        """Think-Act 循环"""
        for i in range(self.rc.max_react_loop):
            has_next = await self._think()
            if not has_next:
                break
            msg = await self._act()
        return msg
```

**我们的现状**：
```python
# 我们的 Agent 完全无状态
class DevAgent(BaseAgent):
    async def execute(self, task_name, context):
        if task_name == "develop":
            return await self.run_action("write_code", context)
        # 没有 think/act 循环，orchestrator 决定一切
```

**移植后的效果**：
- DevAgent 可以自主决定：先读代码 → 再设计 → 再写代码 → 自测
- 不需要 orchestrator 把每一步都编排好
- 复杂工单可以用 `PLAN_AND_ACT` 模式自动分解

### 2.3 Watch 订阅过滤

```python
# MetaGPT: role.py
class Role:
    def _watch(self, actions):
        """声明关心哪些 Action 的输出"""
        self.rc.watch = {str(t) for t in actions}

    @property
```

**我们的现状**：event_bus 广播所有 `ticket_ready` 事件，orchestrator 处理所有工单，没有过滤。

**移植后**：Agent 可以声明只关心特定类型的工单（如 DevAgent 只关心 `architecture_done` 状态的工单）。

### 2.4 Memory cause_by 索引

```python
# MetaGPT: memory.py
class Memory:
    storage: list[Message]
    index: DefaultDict[str, list[Message]]  # 按 cause_by 索引

    def add(self, message):
        self.storage.append(message)
        if message.cause_by:
            self.index[message.cause_by].append(message)

    def get_by_action(self, action) -> list[Message]:
        return self.index[str(action)]
```

**我们的现状**：`AgentMemory.get_recent()` 只按时间排序，无法精确查"DevAgent 上次开发产出了什么"。

**移植后**：

### 节点 v0.14.5: ActionNode 结构化输出

**目标**：LLM 输出自动解析为类型安全的 Pydantic 模型，减少解析失败。

**新增文件**：
```

backend/actions/schemas.py        # 各 Action 的输出 Schema 定义
```

**ActionNode 移植（精简版，适配我们的 LLM client）**：
```python
# action_node.py
from pydantic import BaseModel, create_model
from typing import Type, Dict, Any

class ActionNode:
    """结构化输出节点（移植自 MetaGPT ActionNode，适配本系统）"""

    def __init__(self, key: str, expected_type: Type[BaseModel],
                 instruction: str, schema: str = "json"):
        self.key = key
        self.expected_type = expected_type
        self.instruction = instruction
        self.schema = schema
        self.instruct_content = None  # 填充后的结构化结果

    async def fill(self, req: str, llm) -> "ActionNode":
        """调用 LLM → 解析 → 验证 → 存储"""
        # 1. 编译 prompt（加入 schema 约束）
        json_schema = self.expected_type.model_json_schema()
        prompt = self._compile(req, json_schema)

        # 2. 调用 LLM
        response = await llm.chat_json(
            [{"role": "user", "content": prompt}],
            max_tokens=16000,
        )

        # 3. 验证并存储
        if response and isinstance(response, dict):
            try:
                self.instruct_content = self.expected_type(**response)
                return self
            except Exception as e:
                logger.warning("Schema 验证失败: %s, 尝试宽松解析", e)

        # 4. 降级：尽量提取有效字段
        self.instruct_content = self._lenient_parse(response)
        return self

    def _compile(self, context: str, json_schema: dict) -> str:
        """编译 prompt = 上下文 + 指令 + 格式约束"""
        fields = []
        for name, prop in json_schema.get("properties", {}).items():
            type_str = prop.get("type", "string")
            fields.append(f'  "{name}": {type_str}')

        return f"""{self.instruction}

{context}

请返回以下 JSON 格式:
{{
{chr(10).join(fields)}
}}"""

    def _lenient_parse(self, data):
        """宽松解析：缺失字段用默认值"""
        if not data or not isinstance(data, dict):
            return self.expected_type.model_construct()
        try:
            return self.expected_type.model_construct(**data)
        except Exception:
            return self.expected_type.model_construct()
```

### 节点 v0.14.6: Role 内部状态机

**目标**：Agent 支持 think→act 循环，可自主决策，支持三种执行模式。

**改造 BaseAgent**：
```python
# agents/base.py（移植 MetaGPT Role 的状态机）
class ReactMode(str, Enum):
    SINGLE = "single"         # 单步执行（现有行为，orchestrator 驱动）
    BY_ORDER = "by_order"     # 按顺序执行所有 Action
    REACT = "react"           # LLM 动态选择下一步

class AgentContext:
    """Agent 运行时上下文（移植自 MetaGPT RoleContext）"""
    state: int = -1           # 当前 Action 索引
    todo: ActionBase = None   # 当前要执行的 Action
    memory: AgentMemory = None
    react_mode: ReactMode = ReactMode.SINGLE
    max_react_loop: int = 1

class BaseAgent(ABC):
    action_classes: List[Type] = []
    react_mode: ReactMode = ReactMode.SINGLE

    async def execute(self, task_name, context):
        """根据 react_mode 分发执行"""
        if self.react_mode == ReactMode.SINGLE:
            return await self.run_action(task_name, context)
        elif self.react_mode == ReactMode.BY_ORDER:
            return await self._react_by_order(context)
        elif self.react_mode == ReactMode.REACT:
            return await self._react_with_llm(context)

    async def _react_by_order(self, context):
        """按顺序执行所有 Action"""
        result = {}
        for action_name, action in self._actions.items():
            r = await action.run(context)
            result.update(r.to_dict())
            context.update(r.data)  # 前一步输出作为后一步输入
        return result

    async def _react_with_llm(self, context):
        """LLM 动态选择下一步 Action（类似 MetaGPT REACT 模式）"""
        for i in range(self.rc.max_react_loop):
            action_name = await self._think(context)
            if not action_name:
                break
            result = await self.run_action(action_name, context)
            context.update(result)
        return result
```

**DevAgent 改造示例**：
```python
class DevAgent(BaseAgent):
    action_classes = [WriteCodeAction, SelfTestAction]
    react_mode = ReactMode.BY_ORDER  # 先写代码，再自测

    # execute() 自动按顺序执行 WriteCode → SelfTest
    # 不需要手动编排
```

**预期效果**：
- ✅ DevAgent 可以自动执行"写代码 → 自测"序列
- ✅ 复杂 Agent 可以用 REACT 模式让 LLM 决定下一步
- ✅ 简单 Agent 保持 SINGLE 模式，向后兼容
- ✅ orchestrator 只需触发 Agent，不需要关心内部步骤

**验证标准**：
1. DevAgent 设为 BY_ORDER → 自动执行 WriteCode + SelfTest
2. 设为 REACT → LLM 决定是否需要额外步骤
3. 设为 SINGLE → 行为与现在完全一致（兼容）

### 节点 v0.14.7: Watch 订阅 + cause_by 索引

**目标**：Agent 声明关心的事件类型，Memory 按 action 索引。

**改造 Memory**：
```python
# memory.py（移植 MetaGPT Memory.index 模式）
class AgentMemory:
    def __init__(self, project_id):
        self.project_id = project_id
        self._index: Dict[str, List[MemoryItem]] = defaultdict(list)

    async def add(self, item: MemoryItem, cause_by: str = None):
        """添加记忆并索引"""
        if cause_by:
            self._index[cause_by].append(item)

    def get_by_action(self, action: str) -> List[MemoryItem]:
        """按 Action 类型检索"""
        return self._index.get(action, [])

    def get_by_actions(self, actions: Set[str]) -> List[MemoryItem]:
        """按多个 Action 类型检索"""
        result = []
        for a in actions:
            result.extend(self._index.get(a, []))
        return sorted(result, key=lambda x: x.created_at, reverse=True)
```

**改造 BaseAgent 添加 Watch**：
```python
class BaseAgent(ABC):
    watch_actions: Set[str] = set()  # 关心的 Action 类型

    def _watch(self, actions: List[str]):
        self.watch_actions = set(actions)

    @property
    def important_memory(self):
        """只获取自己关心的记忆"""
        if not self._memory:
            return []
        return self._memory.get_by_actions(self.watch_actions)
```

## 五、移植后架构对比

### 改造前
```
Orchestrator (全能中心)
  ↓ 轮询/事件
  ↓ 查 transition_rules
  ↓ 调 agent.execute(task_name)
Agent (无状态函数)
  ↓ 手动调 LLM
  ↓ 手动解析 JSON
  ↓ 返回 dict
```

### 改造后（融合 MetaGPT 模式）
```
Orchestrator (编排层)
  ↓ 事件驱动
  ↓ 查 SOP 规则
  ↓ 调 agent.execute()
Agent (Role, 有状态机)
  ↓ _think() 决定下一步
  ↓ _act() 执行 Action
  ↓ 循环（BY_ORDER / REACT）
Action (能力单元)
  ↓ ActionNode.fill() 调 LLM
  ↓ Schema 验证
  ↓ 返回 ActionResult (类型安全)
Memory (带索引)
  ↓ cause_by 索引
  ↓ watch 过滤
  ↓ DB + Git 持久化
```

**关键改变**：
- Agent 从"无状态函数"升级为"有状态角色"
- LLM 输出从"裸 dict"升级为"类型安全的 Pydantic 模型"
- Memory 从"按时间查"升级为"按 action 精确索引"
- 保持我们的优势：SOP 配置、事件驱动、持久化、可追溯

## 六、风险和兼容性

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| ActionNode 改造影响已有 Action | 所有 Action 需适配 | 保留 `run()` 原始接口，ActionNode 可选 |
| react_mode 改变 Agent 行为 | orchestrator 逻辑需调整 | 默认 SINGLE 模式完全兼容 |
| watch 过滤可能遗漏事件 | 工单卡住 | 保留轮询兜底（30s 扫描） |
| Pydantic 模型增加复杂度 | 学习成本 | Schema 定义集中在一个文件 |

**兼容性承诺**：所有改造默认使用现有行为，新模式需要显式开启。
