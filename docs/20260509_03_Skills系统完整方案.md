## 一、目标架构

### 两层配置模型

```
┌─────────────────────────────────────────────┐
│          Layer 1：全局默认配置               │
│  ⚙️ 系统设置 → Skills 全局管理              │
│  决定哪些 Skill 对所有项目默认开启/关闭      │
└─────────────────────────────────────────────┘
                     ↓ 继承 + 覆盖
┌─────────────────────────────────────────────┐
│          Layer 2：项目级覆盖                 │
│  项目 Settings → Skills                      │
│  可覆盖全局默认，并添加项目专属自定义 Skill  │
└─────────────────────────────────────────────┘
```

**规则**：
- 全局开启 + 项目未覆盖 → 该项目开启
- 全局关闭 + 项目覆盖开启 → 该项目开启
- 全局开启 + 项目覆盖关闭 → 该项目关闭
- 项目自定义 Skill 只在本项目可见

### 三类 Skill 对比

| 类型 | 存放位置 | 注册方式 | 适用范围 |
|------|---------|---------|---------|
| **内置 Skill** | `skills/packs/` | `skills.json` 手动注册 | 全局，随系统发布 |
| **Marketplace Skill** | `skills/marketplace/` | 自动扫描发现，无需注册 | 全局，复制即可用 |
| **项目自定义 Skill** | `data/project_skills/{id}/` | 前端上传，DB 注册 | 仅本项目可见 |

## 三、Marketplace 目录：复制即可用

### 3.1 工作原理

`skills.json` 加一条 `scan_dir` 条目，指向 `marketplace/` 目录：

```json
"marketplace-skills": {
    "type": "scan_dir",
    "scan_dir": "marketplace",
    "inject_to": ["ChatAssistant"],
    "enabled": true
}
```

`SkillLoader` 扫描时：
1. 发现所有 `marketplace/*/SKILL.md`
2. 读取 frontmatter（`name` + `description`）
3. 生成索引表注入 system prompt
4. 同时在内存建立 `skill_id → 文件路径` 映射，供 `load_skill` 按需加载

**操作流程**：
```bash
# 想用联网搜索？直接复制
cp -r "C:\Users\admin\.codebuddy\skills-marketplace\skills\multi-search-engine" \
      "backend/skills/marketplace/"

# 重启服务，ChatAssistant 的索引里自动出现该 Skill
# 用户问「搜索今天的游戏排行榜」→ AI 自动 load_skill → fetch_url 执行搜索
```

### 3.2 Marketplace Skill 的全局 Enable 控制

Marketplace Skill 默认全部开启（出现在索引表里）。若需关闭某个，在全局设置里 Disable 即可（写入 `global_skill_settings` 表）。

## 四、全局 Skill 配置（新增）

### 4.1 数据库：global_skill_settings 表

```sql
CREATE TABLE IF NOT EXISTS global_skill_settings (
    skill_id    TEXT PRIMARY KEY,
    enabled     INTEGER DEFAULT 1,   -- 1=开启, 0=关闭
    updated_at  TEXT NOT NULL
);
```

优先级：`global_skill_settings` > `skills.json` 中的 `enabled` 字段（无 DB 记录时读 skills.json）。

### 4.2 新增 API

```
GET    /api/skills                            — 全局 Skill 列表（含 DB 覆盖后的状态）
POST   /api/skills/{skill_id}/enable          — 全局开启（写 global_skill_settings）
POST   /api/skills/{skill_id}/disable         — 全局关闭
DELETE /api/skills/{skill_id}/override        — 删除 DB 覆盖，恢复 skills.json 默认值
```

### 4.3 全局配置 UI（⚙️ 系统设置）

在系统设置 Modal 的现有 Tab 旁边加 **🎓 Skills** Tab：

```
⚙️ 系统设置
```

## 六、_get_available_skill_ids 逻辑更新

现有逻辑 → 新三层逻辑：

```python
async def _get_available_skill_ids(project_id=None) -> list:
    from skills import skill_loader

    # Step 1: skills.json 基础列表
    base = {sid: cfg.get("enabled", True)
            for sid, cfg in skill_loader.skills.items()}

    # Step 2: 应用全局 DB 覆盖（global_skill_settings）
    global_rows = await db.fetch_all("SELECT skill_id, enabled FROM global_skill_settings")
    for r in global_rows:
        base[r["skill_id"]] = bool(r["enabled"])

    # Step 3: 应用项目级覆盖（project_skills）
    if project_id:
        proj_rows = await db.fetch_all(
            "SELECT skill_id, enabled FROM project_skills WHERE project_id=?",
            (project_id,)
        )
        for r in proj_rows:
            base[r["skill_id"]] = bool(r["enabled"])

    return [sid for sid, enabled in base.items() if enabled]
```

## 八、使用场景示例

### 场景一：添加联网搜索
```
1. 复制 multi-search-engine/ 到 backend/skills/marketplace/
2. 重启服务
3. 所有项目的 ChatAssistant 索引里自动出现「Multi Search Engine」
4. 如需对某项目关闭：项目 Settings → Skills → 关闭
```

### 场景二：UE 项目专属 Skill
```
1. 系统设置 → Skills：unreal-cpp-dev 全局关闭（非 UE 项目用不到）
2. UE 项目 Settings → Skills：覆盖开启 unreal-cpp-dev
3. 其余项目不受影响
```

### 场景三：项目专属业务规范
```
1. 项目 Settings → Skills → 上传 game-design-spec.md
2. 只在本项目可见
3. AI 助手被问到设计规范时自动 load_skill("custom.game-design-spec")
```
