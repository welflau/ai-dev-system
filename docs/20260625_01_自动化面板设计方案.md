# 自动化面板设计方案

## 背景

在**项目内侧栏**新增「自动化」入口，支持配置定时触发的 AI 任务（直接调 LLM，不产生工单/需求）。
参考截图：CodeBuddy 自动化面板 + 添加弹窗。

**已确认决策：**
- 初版只做**项目内**自动化（全局自动化后续迭代）
- 触发时直接调 LLM，**不走 ChatAssistant 工单流程**，结果存 automation_runs.result_msg
- 弹窗**不含 Skills 选择**
- 任务执行完成通过 **SSE 推送**，自动化面板实时刷新状态

---

## 一、数据模型

### 新增表：`automation_tasks`

```sql
CREATE TABLE automation_tasks (
    id              TEXT PRIMARY KEY,
    project_id      TEXT NOT NULL,          -- 归属项目（初版必填，全局版本后续迭代）
    name            TEXT NOT NULL,          -- 任务名称，如"每日 AI 新闻推送"
    description     TEXT,                   -- 任务描述（模板卡片副标题）
    prompt          TEXT NOT NULL,          -- 直接发给 LLM 的提示词（不走工单流程）
    model           TEXT DEFAULT 'auto',    -- 模型：auto / claude-sonnet-4-6 / claude-opus-4-8

    -- 调度配置
    schedule_type   TEXT NOT NULL,          -- daily | interval | once
    cron_expr       TEXT,                   -- cron 表达式（由前端构造，once 时为空）
    schedule_label  TEXT,                   -- 人类可读，如 "每天 09:00 周一至周五"
    valid_from      TEXT,                   -- 生效开始日期（ISO 8601，可空）
    valid_until     TEXT,                   -- 生效结束日期（ISO 8601，可空）

    -- 状态
    enabled         INTEGER DEFAULT 1,      -- 1=启用 0=禁用
    next_run_at     TEXT,                   -- 下次执行时间（ISO 8601）
    last_run_at     TEXT,                   -- 上次执行时间
    last_run_status TEXT,                   -- success | failed | running

    created_at      TEXT NOT NULL,
    updated_at      TEXT NOT NULL
);
```

### 新增表：`automation_runs`

```sql
CREATE TABLE automation_runs (
    id           TEXT PRIMARY KEY,
    task_id      TEXT NOT NULL REFERENCES automation_tasks(id) ON DELETE CASCADE,
    status       TEXT NOT NULL,    -- running | success | failed
    triggered_by TEXT DEFAULT 'auto',  -- auto | manual
    started_at   TEXT NOT NULL,
    finished_at  TEXT,
    duration_ms  INTEGER,
    result_msg   TEXT,             -- 助手回复摘要（前200字）
    error_msg    TEXT,
    created_at   TEXT NOT NULL
);
```

## 二、后端

### 2.1 调度器 `backend/automation_scheduler.py`

不引入 APScheduler（避免与 asyncio 混用），用与现有 `poll_loop` 一致的模式：

```
async def automation_loop():
    while True:
        await _tick()
        await asyncio.sleep(60)   # 每分钟检查一次

async def _tick():
    now = datetime.utcnow()
    tasks = 查询 automation_tasks WHERE enabled=1 AND next_run_at <= now
    for task in tasks:
        if 在有效日期范围内:
            asyncio.create_task(_run_task(task))
        计算下次 next_run_at（用 croniter）并写回 DB

async def _run_task(task):
    写 automation_runs 记录（status=running）
    推 SSE: automation_task_started
    直接调 llm_client 发送 task.prompt（不走 ChatAssistant，不创建工单）
    等待 LLM 响应，截取前 500 字存 result_msg
    更新 automation_runs（status=success/failed, result_msg, duration_ms）
    更新 automation_tasks.last_run_at / last_run_status
    推 SSE: automation_task_finished（携带 task_id, status, result_msg 摘要）
```

**SSE 事件（推送到项目频道）：**
- `automation_task_started` — `{task_id, task_name, run_id}`
- `automation_task_finished` — `{task_id, task_name, run_id, status, result_msg, duration_ms}`

**依赖：**
- `croniter` — 解析 cron 表达式、计算 next_run_at
- 已有的 `llm_client` / `db` / `event_manager` 模块，无需新增依赖

### 2.2 API `backend/api/automation.py`

```
GET    /api/automation/tasks              # 列表（支持 ?project_id= 过滤）
POST   /api/automation/tasks              # 创建任务
GET    /api/automation/tasks/{id}         # 详情
PUT    /api/automation/tasks/{id}         # 更新
DELETE /api/automation/tasks/{id}         # 删除
POST   /api/automation/tasks/{id}/enable  # 启用
POST   /api/automation/tasks/{id}/disable # 禁用
POST   /api/automation/tasks/{id}/run-now # 立即执行
GET    /api/automation/tasks/{id}/runs    # 执行历史（最近 20 条）
GET    /api/automation/templates          # 获取模板列表
```

### 3.3 添加弹窗

字段（严格对应截图）：

| 字段 | 控件 | 说明 |
|------|------|------|
| 名称 | text input | 必填 |
| 工作空间 | 项目选择器（+号展开下拉） | 对应 project_id，必填（初版只做项目内） |
| 提示词 | textarea | 必填，底部有 `Auto ∨` 模型选择 |
| 执行频率 | Tab 切换：每天 / 按间隔 / 单次 | |
| — 每天 | 时间选择 + 星期多选（周一～周日） | → cron: `分 时 * * 星期` |
| — 按间隔 | 数字 + 单位（分钟/小时/天） | → cron: `*/N * * * *` 等 |
| — 单次 | 日期时间选择器 | → schedule_type=once |
| 生效日期区间（可选） | 日期范围选择器 | valid_from / valid_until |
| 操作 | 取消 / 添加 | |

```
onAutomationTaskStarted(data)      - 更新对应任务卡片为"运行中"状态
onAutomationTaskFinished(data)     - 刷新任务卡片状态 + 在记录列表头部插入新行
```

## 四、执行频率 → cron_expr 转换规则

| 用户选择 | cron_expr 示例 | schedule_label |
|---------|---------------|----------------|
| 每天 09:00 周一至周五 | `0 9 * * 1-5` | 每天 09:00 周一至周五 |
| 每天 09:00 每天 | `0 9 * * *` | 每天 09:00 |
| 每 6 小时 | `0 */6 * * *` | 每 6 小时 |
| 每 30 分钟 | `*/30 * * * *` | 每 30 分钟 |
| 每周五 18:00 | `0 18 * * 5` | 每周五 18:00 |
| 单次：2026-06-26 09:00 | schedule_type=once，不用 cron | 2026-06-26 09:00 执行一次 |
