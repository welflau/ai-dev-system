# 思考过程分组展示方案（J-3 升级）

> 日期：2026-05-17  
> 背景：当前 ADS 将推理链和工具调用分为两个独立面板，看不出哪段推理对应哪次工具调用。  
> 参考：国内某 AI 助手的"每轮推理+工具绑定"展示方式  
> 目标：每轮 LLM 调用作为一个"步骤组"，推理文字 + 工具调用绑定显示

---

## 一、目标效果

```
[第1轮]  💭 推理: "需要搜索游戏排行榜信息..."   [展开/折叠]
          └ 🌐 联网搜索  ✓ 8条结果  "2026年热门游戏"  (1240ms)

[第2轮]  💭 推理: "第一轮没有直接结果，换关键词继续搜..."  [展开/折叠]
          └ 🌐 联网搜索  ✓ 5条结果  "Steam排行榜"  (980ms)
          └ 🌐 联网搜索  ✓ 3条结果  "热门手游排行"  (820ms)

[第3轮]  💭 推理: "已收集足够信息，整理回复..."
          （无工具调用，直接输出文字）
```

每轮有标题（第N轮 + 推理摘要），可折叠，工具调用嵌套在本轮内。

---

## 二、现状分析

### 当前事件流

```
Round 1:
  thinking_delta × N  → 推理文字（蓝色面板，全合并）
  thinking_done
  tool_use_block       → 工具调用决策
  tool_start           → 工具开始（紫色面板，平铺）
  tool_done            → 工具结束

Round 2:
  thinking_delta × N  → 第2轮推理（无法与第1轮区分）
  thinking_done
  tool_use_block × 2
  tool_start × 2
  tool_done × 2
```

**问题**：前端无法判断哪些 thinking 属于第几轮，所有 thinking 合并进同一个面板，所有工具调用平铺在另一个面板。

## 三、技术方案

### 3.1 后端：新增 RoundStartEvent

**`backend/query_engine/events.py`**

```python
@dataclass
class RoundStartEvent:
    """每轮 LLM 调用开始（J-3 升级：用于前端分组显示）"""
    round: int   # 从 1 开始
```

**`backend/query_engine/engine.py`** — 在每轮循环开始时 emit：

```python
for round_no in range(self.max_rounds):
    yield RoundStartEvent(round=round_no + 1)  # ← 新增，第一行
    
    if reason := self.budget.check():
        ...
```

### 3.2 后端透传

**`backend/agents/chat_assistant.py`** — chat_stream / chat_global_stream：

```python
elif isinstance(event, RoundStartEvent):
    yield {"type": "round_start", "round": event.round}
```

**`backend/api/chat.py`** — 两条路径都加：

```python
elif etype == "round_start":
    yield _sse("round_start", {"round": ev["round"]})
```

### 3.3 前端：步骤组渲染

**核心思路**：用 `round_start` 事件作为分组边界，每组内包含推理文字 + 该轮工具调用。

#### 数据结构变化

```javascript
// 原来（平铺）
thinkingSteps = [
    { tool: "web_search", summary: "...", duration_ms: 1240 },
    { tool: "web_search", summary: "...", duration_ms: 980 },
]

// 新（分组）
thinkingRounds = [
    {
        round: 1,
        reasoning: "需要搜索游戏排行榜信息...",  // thinking_done 的 text
        steps: [
            { tool: "web_search", summary: "8条结果", duration_ms: 1240 }
        ]
    },
    {
        round: 2,
        reasoning: "第一轮没有直接结果，换关键词...",
        steps: [
            { tool: "web_search", summary: "5条结果", duration_ms: 980 },
            { tool: "web_search", summary: "3条结果", duration_ms: 820 }
        ]
    }
]
```

#### _sendChatStreaming 修改

```javascript
// 新增状态变量
let _rounds = [];         // thinkingRounds
let _curRound = null;     // 当前轮次对象
let _roundsPanel = null;  // 整体面板容器

// round_start 事件
} else if (eventName === 'round_start') {
    _curRound = { round: data.round, reasoning: '', steps: [] };
    _rounds.push(_curRound);
    _renderRoundsPanel();  // 新建或更新面板

// thinking_delta / thinking_done
} else if (eventName === 'thinking_delta') {
    _reasoningBuf += data.delta || '';
    _updateCurrentRoundReasoning(_reasoningBuf);  // 实时更新当前轮推理文字

} else if (eventName === 'thinking_done') {
    if (_curRound) _curRound.reasoning = data.text || _reasoningBuf;
    _reasoningBuf = '';

// tool_done
} else if (eventName === 'tool_done') {
    if (_curRound) {
        _curRound.steps.push({
            tool: data.tool, summary: data.summary,
            duration_ms: data.duration_ms, args_hint: data.args_hint
        });
    }
    _updateCurrentRoundSteps();
```

## 五、历史消息渲染更新

`thinking_json` 字段需要存储分组结构（目前存平铺数组）。

**DB 兼容方案**：`thinking_json` 新支持两种格式：
- 旧格式（平铺）：`[{tool, summary, duration_ms}, ...]`
- 新格式（分组）：`[{round, reasoning, steps:[...]}, ...]`

通过 `item.round !== undefined` 区分，加载历史时按对应格式渲染，向下兼容旧数据。
