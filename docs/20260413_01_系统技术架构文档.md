## 一、系统概述

AI 自动开发系统是一个**工单驱动的 AI 自动开发管理平台**，通过多 Agent 协作，实现从需求分析、架构设计、代码开发、测试验收到部署上线的全自动化软件开发流程。

**核心能力**：
- 需求自动拆解为可执行工单
- 6 个 AI Agent 协作完成软件开发全流程
- Git 仓库集成（分支管理、代码提交、合并）
- CI/CD 自动构建部署（dev/test/prod 三环境隔离）
- AI 聊天面板（自然语言操控项目）
- 实时 SSE 事件推送 + 看板可视化

## 五、Agent 系统

### 6 个 AI Agent

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| **ProductAgent** | 需求分析、拆单、验收 | 需求描述 | PRD + 工单列表 + 依赖关系 |
| **ArchitectAgent** | 增量架构设计 | 工单 + 已有代码 | 架构方案 + architecture.md |
| **DevAgent** | 编写代码 + 自测 | 架构 + 已有代码 | 代码文件 + dev-notes.md + 自测结果 |
| **TestAgent** | 5 层测试 + 报告 | 代码产物 | test-report.md + 测试用例 + 截图 |
| **ReviewAgent** | 代码审查 | 代码文件 | 评分 + 问题 + 建议 |
| **DeployAgent** | 环境部署 | 项目 + 环境类型 | 预览 URL + Dockerfile |

### 状态转换规则

```
工单状态                    → 下一步 Agent              → 新状态
────────────────────────────────────────────────────────────────
pending                    → ArchitectAgent.设计架构     → architecture_in_progress
architecture_done          → DevAgent.开发              → development_in_progress
development_done           → ProductAgent.验收          → (passed/rejected)
acceptance_passed          → TestAgent.测试             → testing_in_progress
acceptance_rejected        → DevAgent.返工              → development_in_progress
testing_failed             → DevAgent.修复              → development_in_progress
testing_done               → (终态，CI/CD 接管)
```

### 增量开发机制

1. **_build_context()** 在 Agent 执行前读取仓库已有文件
2. **ArchitectAgent** 看到已有代码后设计增量架构（不推翻重来）
3. **DevAgent** 在现有 index.html 基础上添加功能（不从零生成）
4. **自测**：DevAgent 完成后自动运行 5 项检查（文件/入口/语法/规范/非空）

### TestAgent 5 层测试

| 层级 | 内容 | 前端 | 后端 |
|------|------|------|------|
| 1. 静态分析 | 文件完整性 + 语法 + 入口 | JS node --check | Python compile |
| 2. 代码审查 | LLM 评分 1-10 | ✓ | ✓ |
| 3. 功能测试 | 按类型分发 | HTTP server + 页面内容检查 | Python import 检查 |
| 4. 测试用例生成 | LLM 生成 pytest | ✓ | ✓ |
| 5. pytest 执行 | 运行生成的测试 | ✓ | ✓ |

通过率 < 60% → **testing_failed** → DevAgent 自动返工

## 六、核心流程

### 需求到部署全流程

```
┌──────────────────────┐
│  用户提交需求          │  前端表单 / AI 聊天对话
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  ProductAgent 拆单    │  分析需求 → 生成 PRD → 拆分为工单
│  创建 feat 分支       │  feat/YYYYMMDD-req-XXXXXX
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  编排器 10s 轮询      │  扫描 pending 工单 → 检查依赖
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  ArchitectAgent      │  读取已有代码 → 增量架构设计
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  DevAgent 开发+自测   │  在已有代码基础上修改 → 5 项自测
│  代码提交到 feat 分支  │  commit: "[DevAgent] 开发: 排行榜功能 (index.html)"
└──────────┬───────────┘
           ↓ 自动部署 dev 环境
┌──────────────────────┐
│  ProductAgent 验收    │  审查代码是否符合需求
│  不通过 → 打回返工     │
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  TestAgent 测试       │  5 层测试 → 生成报告 + 截图
│  不通过 → 打回修复     │  通过率 < 60% 自动打回
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  需求完成             │  所有工单 testing_done
│  feat → develop 合并  │  自动合并 + 部署 test 环境
└──────────┬───────────┘
           ↓
┌──────────────────────┐
│  CI/CD Pipeline      │  develop 构建 → merge main → master 构建
│  自动部署 prod 环境    │  端口隔离：dev+0, test+100, prod+200
└──────────────────────┘
```

### Git 分支策略

```
main (生产)  ←──── develop (测试) ←──── feat/YYYYMMDD-req-XXXXXX (开发)
     ↑                  ↑                          ↑
  prod 部署          test 部署              dev 部署 + Agent 工作
```

## 九、实时通信

**SSE (Server-Sent Events) 事件类型**

| 事件 | 触发时机 | 数据 |
|------|----------|------|
| ticket_status_changed | 工单状态变更 | ticket_id, from, to, agent |
| requirement_decomposed | 需求拆单完成 | requirement_id, ticket_count |
| requirement_completed | 需求全部完成 | requirement_id |
| agent_working | Agent 开始工作 | agent, action, ticket_id |
| log_added | 新日志条目 | id, agent_type, action, detail |
| ci_build_started | CI 构建开始 | build_id, build_type |
| ci_build_completed | CI 构建完成 | build_id, status |
| ci_build_failed | CI 构建失败 | build_id, error_message |
