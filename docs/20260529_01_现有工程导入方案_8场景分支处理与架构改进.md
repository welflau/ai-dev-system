## 3. 现有架构缺陷分析

### 3.1 GitManager类职责过重（893行代码）
**问题**:
- 路径管理 + Git操作 + 导入逻辑混在一起
- 违反单一职责原则
- 重启后路径映射丢失风险

### 3.2 静默错误处理模式
```python
try:
    # 关键操作
except Exception as e:
    logger.warning("操作失败（忽略）: %s", e)  # 静默继续执行
```

### 3.3 状态管理复杂
- 34种工单状态转换逻辑过度复杂
- 并发控制限制：`_MAX_CONCURRENT_PER_PROJECT=1`
- 内存状态与数据库状态不一致风险

#### 4.2.2 重试机制
```python
async def import_with_retry(import_func, max_retries=3):
    for attempt in range(max_retries):
        try:
            return await import_func()
        except (NetworkError, TimeoutError) as e:
            if attempt == max_retries - 1:
                raise
            await asyncio.sleep(2 ** attempt)  # 指数退避
```
