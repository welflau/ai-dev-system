---

## 一、目标范围

建立一套**全局共享、Git 版本管理、可本地检索**的美术资产库，支撑 UX Agent 和美术 Agent 的资产生产需求，实现：

- 图标、插画、图片、游戏资源按需检索，优先本地复用
- 本地库未命中时自动触发网络搜索并入库
- AI 生图结果自动入库
- 多项目/多实例通过 Git 同步共享同一套资产
- 任意 Agent 可以请求资产，不感知来源细节

---

## 二、技术选型确认

| 组件 | 选型 | 理由 |
|---|---|---|
| 图标来源 | `@iconify/json`（npm，70MB）| 200,000+ 图标，本地全量，零 API 调用 |
| 高质量图标备选 | `tabler/tabler-icons`（clone）| 6,092 图标，MIT，SVG 文件直接用 |
| 插画来源 | unDraw CDN + Storyset API | 无完整 GitHub 包，按需动态入库 |
| 图片来源 | Pexels API（免费密钥）| 无限额，商用免费，无需署名 |
| 游戏资源 | Kenney 预置包（zip 手动下载）| CC0 授权，5万+资源，一次下载长期使用 |
| 3D 材质 | Poly Haven REST API（无密钥）| 3D 游戏项目按需下载 |
| AI 生图 | 图片处理 Agent（见独立设计文档）| 异步非阻塞，结果自动入库 |
| 资产存储 | 独立 Git 仓库 `art-assets-global` | 版本管理 + 多实例共享 |
| 检索索引 | SQLite `art_assets` 表 | 快速检索，可从 manifest.json 重建 |

#### 1.2 数据库表

```sql
CREATE TABLE IF NOT EXISTS art_assets (
    id            TEXT PRIMARY KEY,
    name          TEXT NOT NULL,
    description   TEXT,
    tags          TEXT NOT NULL,        -- JSON 数组
    type          TEXT NOT NULL,        -- icon/illustration/photo/sprite/tileset
    style         TEXT,                 -- flat/outline/filled/realistic/cartoon/pixel
    format        TEXT NOT NULL,        -- svg/png/jpg/webp
    width         INTEGER,
    height        INTEGER,
    file_path     TEXT NOT NULL,        -- 相对于资产仓库根目录
    source        TEXT,                 -- iconify/pexels/undraw/kenney/ai_generated/manual
    source_ref    TEXT,                 -- 来源库 ID，如 "lucide:home"
    source_url    TEXT,
    project_scope TEXT DEFAULT 'global', -- global 或 project_id
    used_count    INTEGER DEFAULT 0,
    last_used_at  TEXT,
    added_at      TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_art_assets_type  ON art_assets(type);
CREATE INDEX IF NOT EXISTS idx_art_assets_tags  ON art_assets(tags);
CREATE INDEX IF NOT EXISTS idx_art_assets_style ON art_assets(style);
```

#### 2.4 统一搜索路由逻辑

```python
class ArtAssetRouter:
    async def resolve(self, request: AssetRequest) -> AssetResult:
        # 1. 查本地库
        if hit := await self.search_local(request): return hit
        # 2. 路由到对应外部 API
        if request.type == "icon":
            return await self.search_iconify_online(request)    # 在线补充
        elif request.type == "photo":
            return await self.search_pexels(request)
        elif request.type == "illustration":
            return await self.fetch_undraw(request)
        elif request.type in ["sprite","tileset","game_ui"]:
            return await self.search_kenney_local(request)      # 本地 Kenney 包
        # 3. AI 生图（异步）
        return await self.request_image_agent(request)
```

### Phase 4：AI 生图结果自动入库（1 天）

**目标**：图片处理 Agent 生成的图片完成后自动写入资产库，下次相似请求直接复用。

#### 4.1 生图完成回调入库

```python
# 图片处理 Agent 生成完成后
async def on_image_generated(result: ImageGenResult):
    asset_id = await art_asset_manager.add_asset(
        name=result.prompt[:50],
        description=result.prompt,
        tags=extract_tags_from_prompt(result.prompt),  # LLM 抽取关键词
        type=result.type,            # illustration/photo/icon
        style=result.style,
        format="png",
        width=result.width,
        height=result.height,
        file_path=result.saved_path,
        source="ai_generated",
        source_ref=f"prompt:{result.prompt[:100]}",
    )
    # git commit + DB 写入
    return asset_id
```

#### 4.2 相似度匹配（避免重复生图）

在发送生图请求前，先用 prompt 关键词查资产库：
```python
# 如果库中有语义相近的已生成图片，直接复用
existing = await search_library(
    tags=extract_tags_from_prompt(prompt),
    type=request_type,
    source="ai_generated",
    min_similarity=0.7   # 标签重合度阈值
)
if existing: return existing  # 不重新生图
```

#### Phase 4 验收标准
- [ ] 生图完成后资产库中可检索到该图片
- [ ] 相似 prompt 第二次请求直接命中本地，不再调用生图 API

## 四、API 接口总表

```
# 资产检索
GET  /api/art-assets
     ?q=<关键词>
     &type=icon|illustration|photo|sprite|tileset
     &style=flat|outline|filled|realistic|cartoon|pixel
     &format=svg|png|jpg
     &source=iconify|pexels|kenney|ai_generated
     &tags=<逗号分隔>
     &limit=20&offset=0
     → [{ id, name, type, style, file_url, used_count, ... }]

# 获取文件
GET  /api/art-assets/{id}/file
     → 二进制文件流

# 手动上传
POST /api/art-assets/upload
     Body: multipart/form-data { file, name, tags, type, style, description }

# 触发网络搜索入库
POST /api/art-assets/search-web
     Body: { query, type, style, limit=5 }
     → [{ id, name, preview_url, source }]  # 搜到但尚未入库的候选
POST /api/art-assets/import-from-search
     Body: { ids: [...] }                    # 用户确认后入库

# 同步远端资产库
POST /api/art-assets/sync
     → { pulled: N, new_assets: N }

# 统计信息
GET  /api/art-assets/stats
     → { total, by_type, by_source, top_used, recent_added }
```

## 六、与其他模块的依赖关系

```
art_assets_repo（Git 仓库）
    ↑ 写入
ArtAssetManager（backend/art_asset_manager.py）
    ↑ 调用                        ↑ 写入
ArtAssetRouter                图片处理 Agent
（搜索路由决策）               （AI 生图后自动入库）
    ↑ 调用
美术 Agent / UX Agent / 任意 Agent
```

**依赖 Phase 关系**：
- Phase 1 是所有后续 Phase 的前置（必须先有基础设施）
- Phase 2、3 可以并行（网络搜索和 Kenney 预置互不依赖）
- Phase 4 依赖图片处理 Agent 完成（可并行开发，集成时需 Agent 完成）
- Phase 5 可以最后做，不影响 Agent 使用资产库
