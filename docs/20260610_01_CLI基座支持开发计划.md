# CLI 基座支持开发计划

> 日期：2026-06-10  
> 背景：当前系统 LLM 基座仅支持 API 方式（Anthropic / OpenAI 兼容），  
> 本计划新增 CLI 方式接入，支持 `claude`、`codebuddy` 等本地 CLI 工具，  
> 满足内网隔离、免 API Key、快速调试等场景需求。

---

## 一、目标

1. **新增 `cli` 接入模式**，与现有 `anthropic` / `openai` 模式并列，通过 `LLM_API_FORMAT=cli` 切换
2. **支持多种 CLI 工具**，通过 `LLM_CLI_TYPE` 适配不同工具的调用参数差异
3. **前端零感知**——上层 Agent / Chat 逻辑无需改动
4. **UI 可配置**——在现有 LLM 配置弹窗中扩展 CLI 相关字段，动态保存到 `.env`

---

## 二、改动范围总览

```
backend/config.py          新增 4 个 CLI 配置项
backend/llm_client.py      新增 _call_cli() + _CLI_ADAPTERS 适配表

frontend/app.js            showLLMConfigModal / saveLLMConfig 扩展

```

### Phase 2：LLM 客户端层（0.5 天）

#### 2-1. `llm_client.py` — CLI 适配表

在文件顶部新增适配注册表，描述各 CLI 工具的参数构造方式：

```python
# 各 CLI 工具的参数适配
_CLI_ADAPTERS = {
    "claude": {
        # claude --print --model <model> -p "<prompt>"
        "build_cmd": lambda cli, model, prompt: [
            cli, "--print", "--model", model, "-p", prompt
        ],
        "stdin": False,
        "parse": "raw_stdout",   # 直接读 stdout
    },
    "codebuddy": {
        # codebuddy ask --model <model> "<prompt>"
        "build_cmd": lambda cli, model, prompt: [
            cli, "ask", "--model", model, prompt
        ],
        "stdin": False,
        "parse": "raw_stdout",
    },
    "custom": {
        # 自定义：prompt 通过 stdin 传入，cmd 直接执行
        "build_cmd": lambda cli, model, prompt: [cli],
        "stdin": True,
        "parse": "raw_stdout",
    },
}
```

#### 2-2. `LLMClient` 新增属性与方法

```python
class LLMClient:
    def __init__(self):
        # 现有字段...
        self.cli_type    = settings.LLM_CLI_TYPE
        self.cli_cmd     = settings.LLM_CLI_CMD
        self.cli_model   = settings.LLM_CLI_MODEL or settings.LLM_MODEL
        self.cli_timeout = settings.LLM_CLI_TIMEOUT

    async def _call_cli(self, messages, temperature, max_tokens) -> tuple:
        """通过本地 CLI 子进程调用 LLM"""
        import asyncio
        adapter = _CLI_ADAPTERS.get(self.cli_type, _CLI_ADAPTERS["custom"])
        prompt  = self._messages_to_prompt(messages)
        cmd     = adapter["build_cmd"](self.cli_cmd, self.cli_model, prompt)

        stdin_data = prompt.encode() if adapter["stdin"] else None
        stdin_pipe = asyncio.subprocess.PIPE if adapter["stdin"] else None

        proc = await asyncio.create_subprocess_exec(
            *cmd,
            stdin=stdin_pipe,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )
        try:
            stdout, stderr = await asyncio.wait_for(
                proc.communicate(input=stdin_data),
                timeout=self.cli_timeout,
            )
        except asyncio.TimeoutError:
            proc.kill()
            return "[LLM_UNAVAILABLE] CLI 调用超时", None

        if proc.returncode != 0:
            err = stderr.decode(errors="replace")[:200]
            logger.error("CLI 调用失败 (rc=%d): %s", proc.returncode, err)
            return self._fallback_response(messages), None

        return stdout.decode(errors="replace").strip(), {
            "input_tokens": None,   # CLI 无精确统计
            "output_tokens": None,
        }

    def _messages_to_prompt(self, messages) -> str:
        """将 messages 列表拼成纯文本 prompt（CLI 模式使用）"""
        parts = []
        for m in messages:
            role    = m.get("role", "")
            content = m.get("content", "")
            if isinstance(content, list):
                # vision blocks 等，提取文本部分
                content = " ".join(
                    b.get("text", "") for b in content if isinstance(b, dict)
                )
            if role == "system":
                parts.append(f"[System]\n{content}")
            elif role == "user":
                parts.append(f"[User]\n{content}")
            elif role == "assistant":
                parts.append(f"[Assistant]\n{content}")
        return "\n\n".join(parts)
```

#### 4-2. `app.js` — 逻辑扩展

```javascript
// 接入方式切换时动态显示/隐藏字段
function onLLMFormatChange(format) {
    document.getElementById('llmApiFields').style.display =
        format === 'cli' ? 'none' : 'block';
    document.getElementById('llmCliFields').style.display =
        format === 'cli' ? 'block' : 'none';
}

// showLLMConfigModal 回填
async function showLLMConfigModal() {
    const data = await api('/llm/status');
    const fmt = data.api_format || 'anthropic';
    document.getElementById('llmApiFormat').value = fmt;
    onLLMFormatChange(fmt);  // 触发显隐
    // CLI 字段回填
    document.getElementById('llmCliType').value    = data.cli_type || 'claude';
    document.getElementById('llmCliCmd').value     = data.cli_cmd  || 'claude';
    document.getElementById('llmCliTimeout').value = data.cli_timeout || 120;
    // ...原有 API 字段回填保持不变...
}

// saveLLMConfig 扩展
async function saveLLMConfig() {
    const format = document.getElementById('llmApiFormat').value;
    const payload = { api_format: format, /* ...原有字段... */ };

    if (format === 'cli') {
        payload.cli_type    = document.getElementById('llmCliType').value;
        payload.cli_cmd     = document.getElementById('llmCliCmd').value.trim();
        payload.cli_timeout = parseInt(document.getElementById('llmCliTimeout').value) || 120;
    }
    await api('/llm/config', { method: 'POST', body: payload });
    // ...toast & close 保持不变...
}
```

## 四、各 CLI 工具适配说明

| 工具 | `cli_type` | 调用命令示例 | 流式支持 | Tool Use |
|------|-----------|-------------|----------|---------|
| Claude Code | `claude` | `claude --print --model claude-sonnet-4-5 -p "..."` | 可扩展（`--output-format stream-json`）| 降级文本协议 |
| CodeBuddy | `codebuddy` | `codebuddy ask --model ... "..."` | 待确认 | 降级文本协议 |
| 自定义 | `custom` | `<LLM_CLI_CMD>`，prompt 经 stdin 传入 | 不支持 | 降级文本协议 |

> **流式扩展（后续迭代）**：Claude CLI 支持 `--output-format stream-json`，可逐行 yield `text_delta` 事件，与现有 SSE 推送体系对接，作为 Phase 2 再做。

## 五、不影响的功能（无需改动）

- 所有 Agent（DevAgent / TestAgent / PlannerAgent 等）— 调用链路不变
- SSE 事件推送、会话记录、Token 统计日志（CLI 模式 token 字段记 `null`）
- LightAI 图片生成配置（独立配置块，互不影响）
- `LLM_API_FORMAT=anthropic/openai` 原有路径完全不受影响

## 六、验收标准

| 验收项 | 说明 |
|--------|------|
| 环境变量切换 | `.env` 设置 `LLM_API_FORMAT=cli` 重启后生效 |
| 前端配置弹窗 | 切换"CLI 模式"后 API 字段隐藏，CLI 字段显示，保存后持久化 |
| 连通性测试 | 点击"测试 LLM"按钮，CLI 不存在时报错，存在时显示版本信息 |
| 正常对话 | Chat 面板发送消息，CLI 工具响应并显示到前端 |
| Tool Use 降级 | Agent 执行工单时，CLI 模式不崩溃，降级为文本协议运行 |
| 两种模式切换 | API → CLI → API 反复切换，`.env` 两套配置互不覆盖 |

## 七、风险与注意事项

1. **Windows 路径**：`asyncio.create_subprocess_exec` 在 Windows 需要工具在 PATH 中，或配置完整路径（含 `.exe` 扩展名）
2. **并发限制**：CLI 每次 fork 进程开销较大，高并发场景建议在 `_call_cli` 内加 `asyncio.Semaphore(4)` 限制
3. **Token 统计**：CLI 模式 `input_tokens` / `output_tokens` 均为 `None`，前端 "今日费用" 模块需容忍 `null` 值（目前已用 `?` 兜底，无需额外改动）
4. **CodeBuddy 参数**：CodeBuddy CLI 的非交互参数需实际验证后更新 `_CLI_ADAPTERS`，当前为推测值
