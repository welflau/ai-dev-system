---

## 1. 背景：现状 vs 竞品对比

### 1.1 当前系统思考过程 UI

当前实现（v0.20）：

```
用户发消息
  ↓ 三点动画（chatTyping）
  ↓ SSE 推送 tool_start → _chatThinkingAppend（创建折叠面板）
  ↓ SSE 推送 tool_done → _chatThinkingAppend（更新步骤）
  ↓ message_done → _chatThinkingFinish（折叠为「✦ 思考了 N 步」）
  ↓ 流式文字出现
```

**问题**：
- 没有区分「在调工具」和「在生成答案」两个阶段，用户不知道 AI 在做什么
- 工具调用展示粒度低：只显示「📚 加载 Skill」「🔍 搜索知识库」，缺少参数和结果细节
- 折叠后「思考了 N 步」过于简化，看不出做了什么
- 无执行统计（工具次数、耗时）

## 3. 改善方案

### 3.1 两阶段状态文字（高优先级，简单）

**当前**：只有三点动画（chatTyping）

**改为**：

| 阶段 | 显示文字 | 时机 |
|---|---|---|
| 初始等待 | `正在思考...` | 发消息后立即 |
| 工具调用中 | `正在调用工具...` | 第一个 tool_start 事件 |
| 生成回答 | `正在生成答案...` | 最后一个 tool_done 之后，text_delta 开始前 |

实现：在 `sendChatMessage()` 里把 chatTyping 的文字动态替换，而不是始终显示三个点。

---

### 3.2 思考面板升级：工具调用日志卡（中优先级）

**当前展示**：
```
✦ 思考了 3 步 ∨
● 📜 查提交历史  ✓ 最近 20 条提交...
● 📜 查提交历史  ✓ 最近 20 条提交...
```

**改为 CodeBuddy 风格**：
```
┌─────────────────────────────────────────────┐
│ 🔧 工具调用过程             [34 步 · 5.2s] ∧ │
├─────────────────────────────────────────────┤
│ 📚 加载 Skill                                │
│   unreal-cpp-dev · UE C++ 开发规范           │
│                                              │
│ 🔍 搜索知识库                                │
│   query: "角色移动实现"                      │
│   ✓ 找到 3 条相关文档                        │
│                                              │
│ 📜 查提交历史                                │
│   branch: main                               │
│   ✓ 最近 20 条: de0b18f [UX...]             │
└─────────────────────────────────────────────┘
```

**变化点**：
- 每个工具两行：**工具名** + 参数 + 结果摘要
- 右上角显示「步数 · 耗时」
- 工具名左侧图标，结果前加 ✓

---

### 3.3 执行统计（低优先级）

思考面板折叠后显示：

```
✦ 思考了 5 步 · 3.2s  ∨
```

展开后底部加：

```
共调用 5 个工具 · 耗时 3.2s
```

---

### 3.4 `args_hint` 改为双行展示（中优先级）

**当前**：工具名 + args_hint 挤在一行
```
● 📜 查提交历史 (branch: main)    ✓ 最近 20 条...
```

**改为**：
```
📜 查提交历史
  branch: main → ✓ 最近 20 条提交: de0b18f...
```

## 4. Phase 拆分

| Phase | 内容 | 难度 | 预计 |
|---|---|---|---|
| 1 | 两阶段状态文字（思考中 / 生成中） | 低 | 0.5h |
| 2 | 工具调用双行展示（名称 + 参数 + 结果） | 中 | 2h |
| 3 | 思考面板底部执行统计（步数 + 耗时） | 低 | 1h |
| 4 | 整体样式向 CodeBuddy 风格靠拢（卡片感） | 中 | 2h |

---

## 5. 与当前架构的兼容性

- 后端已有 `tool_start` / `tool_done` 事件，携带 `tool`、`label`、`summary`
- 只需在 `_chatThinkingAppend` 的渲染逻辑中调整 HTML 结构
- `args_hint` 字段后端已传，只是前端没有用于双行展示
- 耗时统计：记录 `_chatThinkingBegin()` 时间戳，`_chatThinkingFinish()` 时计算差值
