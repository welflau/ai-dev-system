# Hook 可观测性方案

> 日期：2026-05-14  
> 状态：方案设计  
> 背景：Harness Phase 1A 已实现 HookRegistry + 3 个内置 Hook，但运行时无 UI 可查看"哪些 Hook 在跑、跑了多少次、有没有失败"。

---

## 一、现状与问题

### 1.1 现有数据

| 数据 | 存储位置 | 可访问性 |
|------|---------|---------|
| 工具调用明细 | `tool_audit_log` 表 | 有 API，日志面板可见 |
| 工具报错记录 | `ticket_logs`（level=error）| 有 API，日志面板可见 |
| Hook 注册列表 | `hook_registry._hooks`（内存）| **无 API，UI 看不到** |
| Shell 限流计数 | `_shell_call_counts`（内存字典）| **无 API，UI 看不到** |
| Hook 执行失败 | `server.log`（warning 行）| 只能看 log 文件 |

### 1.2 痛点

- 不知道当前注册了几个 Hook、是什么
- 不知道 Shell 限流计数器到了多少（快触发了还是还早？）
- Hook 执行失败只在 server.log 里，难以发现
- 无法查看"某工具被调了多少次"的统计

---

## 二、目标

1. **运行时状态**：能看到已注册的 Hook 列表 + 每个 Hook 的调用次数、失败次数
2. **限流状态**：能看到各 Ticket 的 ShellAction 调用计数
3. **统计图表**：能看到过去 N 分钟内各工具调用频次（from `tool_audit_log`）
4. **失败追踪**：Hook 自身失败时也记录，不只写 log

## 三、数据层设计

### 3.1 HookRegistry 内部增加调用统计

```python
# hooks/registry.py 改动
class HookRegistry:
    def __init__(self):
        self._hooks: List[Callable] = []
        self._stats: Dict[str, dict] = {}   # fn.__name__ → {calls, errors, last_at}

    async def emit(self, ctx, blocking=False):
        for fn in self._hooks:
            name = fn.__name__
            stat = self._stats.setdefault(name, {"calls": 0, "errors": 0, "last_at": None})
            try:
                await fn(ctx) if iscoroutinefunction(fn) else fn(ctx)
                stat["calls"] += 1
                stat["last_at"] = now_iso()
            except Exception as exc:
                stat["errors"] += 1
                if blocking and ...: raise
                else: logger.warning(...)
```

### 3.2 新增 `hook_audit_log` 表（可选，持久化 Hook 失败）

```sql
CREATE TABLE IF NOT EXISTS hook_audit_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    hook_name   TEXT NOT NULL,
    event       TEXT NOT NULL,        -- PreToolUse / PostToolUse / ToolError
    tool_name   TEXT,
    project_id  TEXT,
    success     INTEGER NOT NULL,     -- 1=成功, 0=失败
    error_msg   TEXT,
    duration_ms REAL,
    created_at  TEXT NOT NULL
);
```

> **轻量替代方案**：只在内存里统计（不加新表），重启清零即可，无需持久化。

## 七、与现有系统的关系

```
HookRegistry（内存）
  ├─ _stats 字典（本方案新增）←→ GET /api/hooks/status
  └─ emit() 调用链          ←→ tool_audit_log 表 ←→ GET /api/hooks/stats

不新增 DB 表（hook_audit_log 作为可选扩展），
重启后统计清零（运行时观测，不需要持久化）。
```
