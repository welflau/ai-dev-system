> 创建时间：2026-04-30  
> 关联文档：20260430_KM对比分析与借鉴方案.md

## 一、背景与定位

随着 UX Agent 和美术 Agent 的引入，系统需要一套统一的视觉资源管理机制。资产库是整个美术体系的基础设施，解决三个核心问题：

1. **复用问题**：同一图标/插画被多个需求、多个项目反复生成，浪费 API 调用和时间
2. **一致性问题**：没有统一资产来源，DevAgent 随意引用不同风格的图标，界面混乱
3. **离线可用问题**：依赖外网实时搜索不稳定，需要本地持久化

## 二、整体架构

### 资产库与系统的关系

```
全局 Git 仓库（art-assets-global）
    ├── 所有资源文件（SVG/PNG/JPG）
    └── manifest.json（资产元数据索引）
         ↕ 启动同步 / 新增时同步
art_assets DB 表（快速检索索引）
         ↓
Art Agent / 图片处理 Agent / 任意 Agent 查询
```

- **Git 仓库**：持久化存储 + 版本历史，文件真正存在这里，支持多实例共享
- **manifest.json**：人可读的资产清单，是 DB 的来源，可 diff 可审查
- **DB 表**：快速检索索引，可从 manifest.json 随时重建，不是权威来源

### 资产生产完整优先级

```
资产请求（Art Agent / 图片处理 Agent / 任意 Agent）
    ↓
① 查本地资产库 DB（tags + type + style 匹配）
   命中 → 直接返回文件路径，used_count + 1
    ↓ 未命中
② 网络搜索（按资产类型路由到对应素材站 API）
   找到 → 下载 → 入库（git commit + DB 写入）→ 返回
    ↓ 未找到 / 素材站 API 未配置
③ AI 生图（图片处理 Agent 异步）
   生成完成 → 入库（git commit + DB 写入）→ 替换占位符
    ↓ AI 生图 API 未配置 / 超时
④ CSS/SVG 代码生成（纯逻辑资源，不进入资产库）
⑤ 占位图兜底（纯色块 + 尺寸 + 用途文字，不入库）
```

## 四、数据库表结构

```sql
CREATE TABLE art_assets (
    id           TEXT PRIMARY KEY,          -- AST-xxxxxxxx
    name         TEXT NOT NULL,             -- 资源名称（英文，用于文件名）
    description  TEXT,                      -- 中文描述（用于搜索）
    tags         TEXT NOT NULL,             -- JSON 数组，如 ["icon","navigation","home"]
    type         TEXT NOT NULL,             -- icon / illustration / photo / sprite / tileset / design_token
    style        TEXT,                      -- flat / outline / filled / realistic / cartoon / pixel
    format       TEXT NOT NULL,             -- svg / png / jpg / webp
    width        INTEGER,
    height       INTEGER,
    file_path    TEXT NOT NULL,             -- 相对于资产仓库根目录的路径
    source       TEXT,                      -- iconify / unsplash / pexels / undraw / kenney / ai_generated / manual
    source_ref   TEXT,                      -- 来源库中的 ID（如 "lucide:home"）
    source_url   TEXT,                      -- 原始 URL
    project_scope TEXT DEFAULT 'global',    -- global（全局共享）或 project_id（项目私有）
    used_count   INTEGER DEFAULT 0,         -- 被引用次数（越高搜索结果越靠前）
    last_used_at TEXT,
    added_at     TEXT NOT NULL
);

-- 搜索索引
CREATE INDEX idx_art_assets_tags ON art_assets(tags);
CREATE INDEX idx_art_assets_type ON art_assets(type);
CREATE INDEX idx_art_assets_style ON art_assets(style);
CREATE INDEX idx_art_assets_source ON art_assets(source);
```

## 六、资产路由决策逻辑（Art Agent 执行）

```python
def route_asset(asset: AssetRequest) -> ProductionPath:

    # 1. 先查本地库
    if hit := search_library(tags=asset.tags, type=asset.type, style=asset.style):
        return USE_LOCAL(hit)

    # 2. 网络搜索（按类型路由）
    if asset.type == "icon":
        result = iconify_search(asset.description, style=asset.style)
        if result:
            return DOWNLOAD_AND_STORE(result, source="iconify")

    elif asset.type == "illustration":
        result = undraw_search(asset.description)
        if result:
            return DOWNLOAD_AND_STORE(result, source="undraw")

    elif asset.type == "photo":
        result = pexels_search(asset.description)  # 需配置 PEXELS_API_KEY
        if result:
            return DOWNLOAD_AND_STORE(result, source="pexels")

    elif asset.type in ["sprite", "tileset", "game_ui"]:
        result = kenney_library_search(asset.tags)  # 本地预置的 Kenney 包
        if result:
            return USE_LOCAL(result)

    # 3. AI 生图（异步，返回占位符）
    if IMAGE_API_URL configured:
        return AI_GENERATE_ASYNC(asset)

    # 4. CSS/SVG 代码生成
    if asset.type in ["background", "gradient", "simple_shape"]:
        return GENERATE_CSS(asset)

    # 5. 占位兜底
    return PLACEHOLDER(asset.size, asset.description)
```

## 六·五、GitHub 现有资产库盘点

### 图标类：可以直接 clone，一次性预置

| GitHub 仓库 | 图标数 | 授权 | 推荐度 |
|---|---|---|---|
| `iconify/icon-sets` | **200,000+**（npm `@iconify/json`，70MB）| 各图标集原授权 | ⭐⭐⭐⭐⭐ 首选，包含所有主流图标集 |
| `tabler/tabler-icons` | 6,092（outline+filled）| MIT | ⭐⭐⭐⭐⭐ 质量极高，SVG 直取 |
| `lucide-icons/lucide` | 1,600+ | ISC | ⭐⭐⭐⭐ SVG 在 /icons/ 直接用 |
| `phosphor-icons/core` | 1,200+ × 6 粗细变体 | MIT | ⭐⭐⭐⭐ 变体丰富 |
| `simple-icons/simple-icons` | 3,000+（品牌 Logo）| CC0 | ⭐⭐⭐ 品牌图标专用 |

**图标类策略：一次 clone 终生使用，不需要运行时网络搜索。**

```bash
# 最优方案：Iconify 一个包覆盖所有主流图标集
npm install @iconify/json   # 70MB，200,000+ 图标，本地全量检索

# 或直接 clone 高质量专项库
git clone https://github.com/tabler/tabler-icons
git clone https://github.com/lucide-icons/lucide
```

### 插画/图片类：无完整 GitHub 库，需 API 动态入库

| 来源 | GitHub 情况 | 结论 |
|---|---|---|
| unDraw | 有 GitHub 但无完整 SVG 包 | 只能 CDN 按需引用或手动下载 |
| Unsplash / Pexels | 无 GitHub（版权限制不可批量）| API 搜索后动态入库 |
| Open Peeps / Humaaans | GitHub 以 Figma 格式为主 | SVG 需额外处理 |

**插画/图片策略：不预置，通过 API 搜索后动态写入资产库。**

### 游戏资源类：Kenney 无官方 GitHub，需手动下载

| 来源 | GitHub 情况 | 结论 |
|---|---|---|
| Kenney.nl | 官方只提供 zip 下载，无 GitHub | 手动下载核心包入库（见下方预置方案）|
| Poly Haven | 有 GitHub，REST API 无需密钥 | 3D 材质/HDRI，可动态下载 |

**游戏资产策略：核心 Kenney 包手动下载预置；3D 材质走 Poly Haven API 动态入库。**

## 七、Kenney 预置方案

Kenney.nl 提供的 5 万+ 资源是游戏项目最重要的开箱即用资产来源，建议系统初始化时预置。

**推荐预置的 Kenney 包**：

| 包名 | 内容 | 大小 | 用途 |
|---|---|---|---|
| UI Pack | 按钮/面板/图标/HUD 组件 | ~50MB | 游戏 UI |
| Pixel Platformer | 像素风角色/地图块/道具 | ~10MB | 2D 横版游戏 |
| Top-Down | 俯视角角色/场景/道具 | ~15MB | 俯视角游戏 |
| Fantasy | 奇幻风角色/怪物/道具 | ~20MB | RPG 游戏 |
| Roguelike | 地牢/怪物/技能图标 | ~8MB | Roguelike 游戏 |

**初始化脚本**（系统启动时检查并下载）：
```bash
# 如果 Kenney 包未下载，自动从官网获取
if not exists art-assets-global/game/kenney:
    wget https://kenney.nl/assets/download/...
    unzip → art-assets-global/game/kenney/
    vibecli assets import kenney  # 扫描并写入 manifest.json + DB
```

## 八、API 端点设计

```
# 搜索资产库
GET  /api/art-assets
     ?q=home+icon          # 关键词
     &type=icon            # 类型过滤
     &style=outline        # 风格过滤
     &format=svg           # 格式过滤
     &source=iconify       # 来源过滤
     &limit=20             # 数量

# 获取单个资产文件
GET  /api/art-assets/{id}/file

# 手动上传入库
POST /api/art-assets
     Body: { file, name, tags, type, style, description }

# 触发网络搜索并入库
POST /api/art-assets/search-web
     Body: { query, type, style, limit }

# 从远端 pull 最新资产库（多实例同步）
POST /api/art-assets/sync

# 资产库统计
GET  /api/art-assets/stats
     → { total, by_type, by_source, top_used }
```
