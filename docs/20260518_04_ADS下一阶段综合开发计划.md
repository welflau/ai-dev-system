## 一、核心架构问题（优先解决）

### 现状：能力被困在叶子节点

当前 ADS 的 Thinking、Compaction、Memory 注入等能力全部写死在 `ChatAssistantAgent`，
Orchestrator 的 DevAgent / TestAgent / PlannerAgent 完全无法受益：

```
当前（错误）:                    目标（正确）:
ChatAssistantAgent               BaseAgent / QueryEngine
  └─ Thinking ✅                   └─ Thinking ✅（所有 Agent 自动继承）
  └─ Compaction ✅                 └─ Compaction ✅
  └─ Memory 注入 ✅                └─ Memory 注入 ✅
  └─ Rules 注入 ✅                 └─ Rules 注入 ✅（已有）

DevAgent ❌                      DevAgent ✅（继承）
TestAgent ❌                     TestAgent ✅（继承）
UEEditorAgent（新）❌            UEEditorAgent（新）✅（继承）
```

**改造原则**：新功能一律在 `BaseAgent` 或 `QueryEngine` 实现，`ChatAssistantAgent` 只做 Chat 专属逻辑。

## 二、方向 A：基础设施层重构（对标 Claude Code）

> 参考：`docs/20260518_03_ADS对标ClaudeCode架构差距分析报告.md`

### A-1 Commands / 斜杠指令系统（P1，3天）

**问题**：ADS 只有 `/test` 一个调试命令，无法直接触发 Agent、Skill 或管理操作。

**设计**：
```
用户输入 /command args
  ↓ handleChatKeydown（前端拦截）
  ↓ POST /api/commands/{command}?args=...
  ↓ 读取 skills/commands/*.md 定义
  ↓ 路由到对应 Action / Agent
```

**内置优先实现的命令**：
| 命令 | 功能 |
|---|---|
| `/compact` | 手动触发历史压缩 |
| `/memory` | 查看/管理 Agent 记忆 |
| `/ue-bp-gen <desc>` | 直接触发 Blueprint 生成 |
| `/ue-run <code>` | 在 UE Editor 执行 Python |
| `/skills` | 查看当前项目已加载 Skills |
| `/think on/off` | 切换 thinking 模式 |

**文件**：
- `frontend/app.js`：扩展斜杠拦截
- `backend/api/commands.py`：新建命令路由
- `backend/skills/commands/*.md`：命令定义文件

```python
# backend/agents/base.py
class BaseAgent:
```

### B-1 Blueprint 生成 Action（P1，3-5天）

```python
# backend/actions/ue_blueprint_gen.py
class BlueprintGenAction(ActionBase):
    async def run(self, context):
        desc = context.get("description")
        # LLM 生成 UE Python 代码（使用 ue-blueprint-patterns skill）
        py_code = await self._generate_bp_code(desc)
        # 发送到 UE Editor
        result = await run_python(py_code, context["project_id"])
        return ActionResult(...)
```

**SOP 集成**：新增 `sop/fragments/ue_content_gen.yaml`，在 `architecture_done` 后插入。

**与模板卡片的关系**（不冲突）：

```
    ↓（有了基础）

```
