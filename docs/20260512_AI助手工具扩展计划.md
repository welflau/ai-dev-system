## 二、优先级规划

### P0 — 强烈推荐（高价值、低风险）

#### 1. ShellTool — 执行 Shell 命令

**价值**：AI 可以运行脚本、执行测试、查看输出、诊断环境问题

**设计**：
```python
class ShellAction(ActionBase):
    # 执行 shell 命令，返回 stdout/stderr/exit_code
    # 安全限制：
    #   - 白名单目录（project git_repo_path）
    #   - 黑名单命令（rm -rf / format / drop table 等）
    #   - 超时 30s
    #   - 执行前展示命令供用户确认（Tier 1 action）
```

**实现要点**：
- 复用 `backend/utils.py` 里现有的 subprocess 封装
- 白名单目录：只允许在当前项目目录执行
- 危险命令黑名单检测
- 输出截断（max 5000 字符）
- 工时估算：**1 天**

---

#### 2. GlobTool + GrepTool — 文件搜索

**价值**：AI 可以在项目里找文件、搜索代码内容，无需一一列举路径

**GlobTool 设计**：
```python
class GlobAction(ActionBase):
    # 参数: pattern（glob 通配符），base_dir（相对项目根）
    # 返回: 匹配的文件路径列表，按修改时间排序
    # 示例: pattern="**/*.py", base_dir="backend/agents"
```

**GrepTool 设计**：
```python
class GrepAction(ActionBase):
    # 参数: pattern（正则），path（目录/文件），include（glob 过滤）
    # 返回: 匹配行（文件名 + 行号 + 内容），最多 100 条
    # 内部用 Python re 或 ripgrep（若可用）
```

**实现要点**：
- 基础路径限制在 project `git_repo_path`（安全）
- GlobTool 返回相对路径
- GrepTool 结果截断，避免上下文爆炸
- 工时估算：**0.5 天**

---

#### 3. WebSearchAction — 原生联网搜索

**价值**：替代不稳定的 multi-search-engine Skill 方案，提供稳定的搜索能力

**设计**：
```python
class WebSearchAction(ActionBase):
    # 方案 A：腾讯元宝搜索 API（内部资源，稳定）
    #   需要 TENCENTCLOUD_WSA_APIKEY 环境变量
    # 方案 B：基于 fetch_url 的多引擎封装（无需 API key）
    #   内置 Bing/Google/Baidu URL 构造逻辑

    # 参数: query（搜索词），num_results（默认5）
    # 返回: [{title, url, snippet}, ...]
```

**实现要点**：
- 优先接腾讯元宝搜索 API（质量最好）
- 降级到 fetch_url 多引擎方案
- 对全局聊天和项目聊天都暴露（_GLOBAL_CHAT_TOOLS）
- 工时估算：**1 天**

### P2 — 谨慎评估（高风险，需确认机制）

#### 7. WriteFileAction / EditFileAction

- 自动 git stash 兜底
- 限制只能操作 project 目录

**建议时机**：待 P0/P1 完成后评估，参考 Gemini CLI `edit.ts` 的 diff 确认机制

## 四、安全约束（所有新工具共用）

1. **路径限制**：文件操作只允许在 `project.git_repo_path` 目录内
2. **输出截断**：所有工具返回内容不超过 8000 字符
3. **危险操作确认**：Shell 执行、文件写入需 Tier 1 确认（用户点击才执行）
4. **黑名单检测**：Shell 命令过滤 `rm -rf`、`format`、`DROP TABLE` 等危险命令
5. **只对项目内聊天暴露**：文件系统工具不在全局聊天中暴露（无项目上下文）
