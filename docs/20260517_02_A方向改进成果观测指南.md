import asyncio, sys

async def smoke_test():
    results = []

    # 1. Rules 注入
    from agents.chat_assistant import ChatAssistantAgent

## 六、A 方向改进的覆盖范围：ChatAssistantAgent vs Orchestrator Agents

> 问题：A 方向的 8 项改进，Orchestrator 的 DevAgent / TestAgent 等也用到了吗？

### 6.1 两条 AI 调用路径

系统里存在两条完全不同的 LLM 调用链：

```
聊天面板
  └─ ChatAssistantAgent.chat_stream()
       └─ QueryEngine  ← A 方向改进的主要受益者

工单自动化
  └─ Orchestrator.process_ticket()
       ├─ agent.run_action("write_code", ...)     ← ActionNode.fill()，独立 LLM 调用
       └─ agent._react_with_think(...)             ← QueryEngine（REACT 模式）
```

DevAgent、TestAgent 等大多数任务走 `run_action` → `ActionNode`，每个 Action 是**一次独立 LLM 调用**，没有"对话历史"概念。只有配置了 `REACT` 模式的 Agent 才走 QueryEngine。

### 6.2 各项改进的覆盖对照

| 能力 | ChatAssistantAgent | Orchestrator Agents | 说明 |
|---|:---:|:---:|---|
| **QueryEngine 调用** | ✅ | ⚠️ | Agent REACT 模式有；`run_action`（ActionNode）路径无 |
| **Budget 三维约束** | ✅ | ⚠️ | REACT 模式有；ActionNode 无 Budget |
| **Hooks（Pre/Post）** | ✅ | ✅ | `run_action_with_hooks` 覆盖 PRE/POST；但无 UserPromptSubmit / AssistantStop |
| **L1 Diminishing Returns** | ✅ | ❌ | Budget 对象相同，但 Agent REACT 循环未调用 `is_diminishing()` |
| **L3 Prompt Cache 分区** | ✅ | ❌ | ActionNode system prompt 是纯字符串，无 CACHE_BOUNDARY |
| **L4 Compaction** | ✅ | ❌ | Agent 每次独立调用，无多轮对话历史，无需压缩 |
| **L5 Rules 注入** | ✅ | ⚠️ | Agent 通过 ContextVar `_current_skills` 注入，路径不同但功能相近 |
| **L5 Memory 主动注入** | ✅ | ❌ | Agent system prompt 不含记忆，无自动查询 |
| **L6 nudge_hook** | ✅ | ❌ | Orchestrator 无 AssistantStop 事件 |
| **L7 cost_usd** | ✅ | ✅ | 共用 `llm_client._save_conversation`，**两者均已覆盖** |
| **L8 并行子任务** | ✅ | ✅ | Orchestrator 调度层改进，**两者均受益** |
| **L9 Feature Flags** | ✅ | ❌ | Agent Budget 从 `settings` 读固定值，无运行时覆盖 |

图例：✅ 已覆盖　⚠️ 部分覆盖　❌ 未覆盖

### 6.3 根因分析

**为什么大多数改进只在 ChatAssistant？**

ChatAssistant 是**持续对话**场景：用户与 AI 来回多轮，历史不断积累，因此需要 Compaction、Cache 分区、Memory 注入、Feature Flags。

Orchestrator Agent 是**单次任务执行**场景：`write_code`、`run_tests` 都是独立的一次 LLM 调用，输入输出明确，完成后无需保留上下文。Compaction / Cache 对它意义不大。

**什么对 Agent 也有价值但尚未做？**

| 项目 | 价值 | 改动量 |
|---|---|---|
| REACT 模式接入 Diminishing Returns | Agent 卡循环时提前止损（现在会跑满 `AGENT_MAX_TURNS`） | 小：`base.py:_react_with_think_inner` 加一行 `check()` |
| REACT 模式 Prompt Cache | Agent REACT system prompt 稳定部分（角色描述/工具列表）可缓存 | 中：需改 `_react_with_think_inner` system prompt 构建 |
| ActionNode Rules 注入对齐 | 确保 ChatAssistant 的 `get_rules_for_context()` 与 Agent 的 ContextVar 注入结果一致 | 小：审查两条路径的实际注入内容 |
