# ConfigPack 库设计方案

**日期**：2026-06-17  
**状态**：待评审

---

## 1. 背景与问题

当前系统在新建项目时，会根据 `traits` 自动安装后端 Skills（Python 代码），但对于开发者日常使用的 CLI 工具（Claude Code、Codebuddy）没有配套的配置安装机制。

开发者在项目初始化后，还需要手动：
- 在 `.claude/` 下写 `CLAUDE.md`、slash commands
- 在 `.codebuddy/` 下配 rules、agents、mcps
- 为不同项目重复同样的配置工作

**目标**：建一个 ConfigPack 库，新建项目时可挑选 Pack，系统自动将对应的 CLI 配置文件 copy 到项目目录，支持模板变量替换。

---

## 2. 核心概念

### ConfigPack

一个 Pack 是针对某类项目或工作场景的 CLI 配置集合，包含以下任意组合：

| 文件类型 | Claude Code 路径 | Codebuddy 路径 |
|---|---|---|
| rules | `.claude/CLAUDE.md`（追加） | `.codebuddy/rules/*.md` |
| agents | —（Claude 无此概念） | `.codebuddy/agents/*.md` |
| commands | `.claude/commands/*.md` | `.codebuddy/commands/*.md` |
| skills | `.claude/skills/*.md` | `.codebuddy/skills/*.md` |
| mcps | `.claude/settings.json`（merge） | `.codebuddy/mcps/*.json`（merge） |

### 与现有 Skills 库的区别

| | 现有 Skills | ConfigPack |
|---|---|---|
| 目标 | 后端 orchestrator 执行能力 | CLI 工具配置文件 |
| 载体 | Python 代码 | Markdown / JSON 文件 |
| 使用者 | 系统自动执行 | 开发者日常编码 |
| 安装位置 | `.claude/worktrees/.Agent/skills/` | `<project>/.claude/` 或 `<project>/.codebuddy/` |

### 3.2 pack.json 结构

```json
{
  "name": "ue5-dev",
  "display_name": "UE5 开发套件",
  "description": "UE5 项目全套 CLI 配置：构建命令、蓝图 Agent、编译 Skill",
  "version": "1.0.0",
  "tags": ["ue5", "gamedev", "cpp"],
  "auto_traits": ["engine:ue5"],
  "contains": ["rules", "agents", "commands", "skills", "mcps"],
  "targets": ["claude", "codebuddy"]
}
```

- `auto_traits`：匹配这些 traits 时自动勾选推荐（不强制安装）
- `targets`：声明支持哪些 CLI，安装时只处理存在的子目录

## 4. 模板变量替换

Pack 文件中可使用以下占位符，安装时替换为项目真实值：

| 占位符 | 来源 |
|---|---|
| `{{project_name}}` | 项目名称 |
| `{{repo_path}}` | 本地仓库路径 |
| `{{tech_stack}}` | 技术栈字符串 |
| `{{git_remote}}` | Git 远端地址 |

**替换规则**：
- 只替换白名单内的变量，其余原样保留
- Codebuddy rules 文件中的 `{{.CurrentDate}}` 等系统变量不替换
- 实现：简单 `str.replace()`，无模板引擎依赖

## 5. 安装逻辑

### 5.1 各文件类型处理方式

| 操作 | 适用类型 | 说明 |
|---|---|---|
| **copy** | rules、agents、commands、skills | 直接复制，同名文件覆盖 |
| **追加** | `CLAUDE.md` | 在末尾追加 `## Pack: <name>` section |
| **merge** | `settings.json`、`mcp.json` | 只往里加新的 key，不覆盖已有配置 |

### 5.2 CLI 检测逻辑

安装时自动检测项目支持哪些 CLI：

```python
def detect_cli_targets(project_path: str) -> list[str]:
    targets = []
    if os.path.exists(f"{project_path}/.claude"):
        targets.append("claude")
    if os.path.exists(f"{project_path}/.codebuddy"):
        targets.append("codebuddy")
    # 两个都不存在时，按 pack.json targets 字段创建目录
    return targets
```

### 5.3 安装流程（伪代码）

```python
def install_pack(pack_name, project_path, project_ctx):
    pack = load_pack(pack_name)
    targets = detect_cli_targets(project_path) or pack.targets

    for target in targets:                        # "claude" / "codebuddy"
        src_dir = pack.path / target
        if not src_dir.exists():
            continue

        for file in walk(src_dir):
            content = render_template(file.read(), project_ctx)
            dst = project_path / f".{target}" / file.relative_to(src_dir)

            if file.name == "CLAUDE.md":
                append_section(dst, pack.name, content)
            elif file.name in ("settings.json", "mcp.json"):
                merge_json(dst, json.loads(content))
            else:
                dst.parent.mkdir(parents=True, exist_ok=True)
                dst.write_text(content)
```

## 6. 与新建项目流程的集成

### 6.1 触发时机

在 `CreateProjectAction` 现有流程之后，新增 `_auto_install_packs()` 异步任务，与 `_auto_install_skills()` 并行执行。

```python
# create_project.py — CreateProjectAction.run() 末尾
asyncio.create_task(self._auto_install_skills(project_id, traits))
asyncio.create_task(self._auto_install_packs(project_id, traits, local_repo_path))
```

### 6.2 自动推荐逻辑

Traits → Pack 映射表（与现有 `_TRAIT_SKILL_MAP` 同构）：

```python
_TRAIT_PACK_MAP = {
    "engine:ue5": ["ue5-dev"],
    "engine:ue4": ["ue5-dev"],
    "platform:web": ["web-dev"],
    "framework:react": ["web-dev"],
    "vcs:git": ["git-workflow"],
}
```

### 6.3 UI 交互（ConfirmProject 卡片扩展）

在现有确认卡片下方增加 **Pack 选择区**：

```
┌─────────────────────────────────────┐
│ 项目名称: 我的射击游戏               │
│ 技术栈: UE5 / C++                   │
│ Traits: engine:ue5, category:game   │
├─────────────────────────────────────┤
│ 推荐 ConfigPack（可取消勾选）        │
│ ☑ ue5-dev     UE5 开发套件          │
│ ☑ git-workflow Git 工作流命令        │
│ ☐ web-dev     Web 前端配置           │
│ [+ 手动添加]                         │
└─────────────────────────────────────┘
```

- 推荐 Pack 默认勾选，用户可反选
- 支持手动选择未推荐的 Pack
- 选择在确认时传入 `selected_packs: list[str]`

## 7. 数据库变更

新增 `project_packs` 表，记录每个项目安装了哪些 Pack：

```sql
CREATE TABLE project_packs (
    id          TEXT PRIMARY KEY,
    project_id  TEXT NOT NULL,
    pack_name   TEXT NOT NULL,
    installed_at TEXT NOT NULL,
    targets     TEXT  -- JSON: ["claude", "codebuddy"]
);
```

用途：项目详情页展示已安装 Pack，未来支持"重新安装"操作。

## 9. 实施阶段

### Phase 1：基础设施
- `pack_installer.py`：模板替换 + 三种安装操作
- 创建 2-3 个内置 Pack（`ue5-dev`、`git-workflow`）
- 数据库 `project_packs` 表

### Phase 2：流程集成
- `create_project.py` 接入自动安装
- `confirm_project.py` 返回推荐 Pack 列表
- 前端确认卡片 Pack 选择 UI

### Phase 3：管理功能
- 项目详情页展示已安装 Pack
- 手动"重新安装"某个 Pack
- 支持用户自定义 Pack（`~/.ads/config_packs/`）

---

## 10. 不做的事（范围控制）

- 不做 Pack 版本追踪和自动更新（安装后归用户所有）
- 不做跨 Pack 依赖解析（Pack 作者自己打包依赖文件）
- 不做远端 Pack 市场（第三阶段以后再考虑）
- 不做软连接（Windows 兼容性问题，仅调试模式可选）
