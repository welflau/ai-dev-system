## 2. 架构设计

### 2.1 整体调用链

```
DevAgent / TestAgent
        │
        │ context["ucp_available"] == True
        ▼
UEEditorControlAction
        │ TCP JSON
        ▼
UCP Plugin (127.0.0.1:9876)
        │
        ▼
UnrealEditor.exe（持久化或按需启动）
```

### 2.2 UCP 插件安装方式

#### 安装步骤的可自动化程度

| 子步骤 | 可自动化 | 耗时 |
|---|---|---|
| 复制插件文件到 `Plugins/` | ✅ 完全自动（文件 copy） | ~1s |
| Editor 编译插件 | ⚠️ 需用户手动重启 Editor | 1-3 min |
| Edit→Plugins 勾选启用 | ⚠️ 需用户手动操作 | ~10s |

**结论**：只有第一步可以自动化，后两步必须用户配合。

---

#### 设计决策：嵌入 `propose_ue_framework` 确认卡

**切入点选择理由**：

- `confirm_project` 阶段只建 git 仓库，项目类型未确定，太早
- `propose_ue_framework` 是 UE 项目的「配置收口」，`InstantiateUETemplateAction` 在此执行，顺带 copy 零代价
- UCP 是 UE 专属能力，非必选，做成可选复选框避免强制

**完整流程**：

```
用户："做个 UE5 FPS"
    ↓
AI 弹「框架方案卡」（propose_ue_framework）
  + 新增复选框：☑ 启用编辑态 AI 控制（UCP 插件）  ← 新增
    ↓
用户确认 → InstantiateUETemplateAction 执行
  → 若勾选 UCP：从 backend/ue_plugins/ 复制插件到 {project}/Plugins/UnrealClientProtocol/
  → .uproject 自动包含插件引用（UE 标准行为）
  → git commit（含 Plugins/ 目录）
    ↓
AI 助手自动弹「操作指引卡」（一次性）：
  "UCP 文件已就位，还需手动完成两步：
   1. 打开 Editor（首次启动会自动编译，约 2 分钟）
   2. Edit → Plugins → 搜索 UnrealClientProtocol → 勾选 Enabled → 重启 Editor
   完成后编辑态 AI 控制即刻生效 ✅"
    ↓
「交付 & 环境」页 Editor 进程卡片实时显示连接状态：
  🟢 已连接（9876 可达）→ AI 可操纵 Editor
  ⚫ 未连接（Editor 未开或插件未启用）
```

---

#### 插件源码存放位置

UCP 源码内置到 AI Dev System 仓库：

```
backend/ue_plugins/
└── UnrealClientProtocol/       ← UCP 插件快照（锁定版本）
    ├── UnrealClientProtocol.uplugin
    ├── Source/
    └── Skills/
```

**为什么内置而不是 clone GitHub**：
- 版本锁定，避免上游 API break 影响已跑通的项目
- 安装时无网络依赖（内网/离线环境同样可用）
- 可在本地打 patch 修复 UCP 边缘 bug 而不影响上游

### 2.3 Editor 守护策略

常驻 Editor 进程是核心运维问题，分三种模式：

| 模式 | 触发 | 适合场景 |
|---|---|---|
| **手动模式（默认）** | 用户自己开着 Editor | 日常开发 |
| **按需模式** | UEEditorControlAction 发现连不上 9876 → 自动 launch Editor → 等待 ready | CI/自动化 |
| **持久化模式** | 项目设置开关 | 深度 AI 协作 |

**当前 Phase 1-5 只实现手动模式**——Editor 必须手动开着，连不上则 Action 返回 BLOCKED 并提示用户。

## 3. Phase 拆分（~9.5 天）

### Phase 1：底层 TCP 客户端 + 3 个原子操作（1 天）

**文件**：`backend/actions/ue_editor_control.py`（新建）

```python
class UEEditorControlAction(ActionBase):
    available_for_traits = {"any_of": ["engine:ue5", "engine:ue4"]}

    async def run(self, context):
        op   = context["op"]        # get_actors / get_property / set_property / call_function / spawn_actor
        args = context.get("args", {})
        host = context.get("ucp_host", "127.0.0.1")
        port = context.get("ucp_port", 9876)
        result = await self._ucp_call(host, port, op, args)
        return ActionResult(success=result.get("success", False), data=result)

    async def _ucp_call(self, host, port, op, args, timeout=10):
        """JSON-over-TCP，单次请求-响应"""
        ...
```

**实现的 3 个原子**：
- `get_actors_in_level`：取当前关卡所有 Actor（名称 + class + transform）
- `get_property`：读任意 UObject 的属性值
- `set_property`：写任意 UObject 的属性值（带 git stash 兜底）

**连接失败处理**：
- `ConnectionRefusedError` → `ActionResult(success=False, data={"status": "editor_not_running"})`
- Orchestrator 检测到 `status=editor_not_running` → `BLOCKED` + 推送提示消息

## 5. 依赖与风险

### 5.1 UCP 插件安装

**手动步骤**（开发 Phase 1 前必须完成）：
```
1. git clone https://github.com/Italink/UnrealClientProtocol
2. 复制 Plugins/UCP 到 F:\Projects\TestTPS\Plugins\UCP
3. UE 生成 .sln → 重新编译项目
4. 编辑器 Edit → Plugins → 启用 UnrealClientProtocol → 重启
5. 验证：Editor 启动后 9876 端口有监听
```

### 5.2 协议格式验证

UCP 的 JSON 协议需要先实验性验证（Phase 1 前做一次 nc/telnet 测试）。

### 5.3 UE 进程内存占用

常驻 Editor 约占 4-8GB RAM + 显存。当前开发机内存 151MB（系统余量），需确认机器有足够内存。

### 5.4 Latent / Delegate 不支持

UCP 已知限制：延迟执行函数（Latent）和 Delegate 类型参数无法通过 reflection 调用。
对策：这类操作仍走 C++ + UBT 路径；UCP 只处理同步操作。

## 6. 启动顺序建议

1. **确认 UCP 安装可行**：手动安装 UCP 到 TestTPS，验证 9876 端口连通（1-2小时）
2. **Phase 1** → **Phase 3** → **Phase 5**（核心价值：Reflexion 增强）
3. **Phase 2**（Editor 管理 UI，用户体验提升）
4. **Phase 4**（模板集成，影响范围大，后做）
5. **Phase 7**（兜底机制，与写操作同步做）
6. **Phase 6 / Phase 8**（BP 编辑 + 验收，最后做）
