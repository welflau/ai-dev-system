## 1. 背景与问题

### 1.1 当前 Skill 架构（被动注入）

**工作方式**：
```
启动时 / Agent 初始化时：
  skill_loader.build_prompt_for_agent("DevAgent")
      → 扫描 skills.json 中 inject_to=[DevAgent] 的条目
      → 按 traits_match 过滤
      → 把所有匹配的 prompt.md 全文拼接
      → 存入 self._skills_prompt

每次 LLM 调用时：
  system_prompt = 基础指令 + self._skills_prompt（全量注入） + 项目上下文
```

**关键代码（现状）**：

`backend/agents/base.py` — Agent 初始化时静态加载所有 Skills：
```python
# BaseAgent.__init__()
try:
    from skills import skill_loader
    # 所有 inject_to=[this_agent] 的 Skills 全量拼接成字符串
    self._skills_prompt = skill_loader.build_prompt_for_agent(self.agent_type)
except Exception as e:
    self._skills_prompt = ""
```

`backend/skills/loader.py` — `build_prompt_for_agent` 把 Skills 全文拼起来：
```python
def build_prompt_for_agent(self, agent_type, traits=None, current_file=None):
    skill_ids = self.get_skills_for_agent(agent_type, traits, current_file)
    sections = []
    for sid in skill_ids:
        content = self.get_skill_prompt(sid)       # 读取 prompt.md 全文
        if content:
            name = self.skills[sid].get("name", sid)
            sections.append(f"<!-- Skill: {sid} ({name}) -->\n{content}")
            # ↑ 不管 LLM 是否需要，全部追加进去
    result = "\n\n---\n\n".join(sections)
    self._agent_prompt_cache[cache_key] = result   # 缓存，不再按对话动态计算
    return result
```

`backend/agents/chat_assistant.py` — ChatAssistant 直接把 Skills 静态拼入 system prompt：
```python
# _build_system_prompt()
skills_section = (
    f"\n## 专业技能 (Skills)\n{self._skills_prompt}\n"
    if getattr(self, "_skills_prompt", "") else ""
)
# → 每次对话 system prompt 都含全量 Skill 内容，无论问题是否需要
```

`backend/skills/skills.json` — Skills 注册配置（inject_to 指定注入哪些 Agent）：
```json
{
  "unreal-cpp-dev": {
    "name": "Unreal Engine C++ 开发",
    "prompt_file": "packs/unreal_cpp_dev/prompt.md",
    "inject_to": ["DevAgent"],           // ← 注入时机：Agent 初始化
    "traits_match": { "any_of": ["engine:ue5", "engine:ue4"] },
    "enabled": true
  },
  "unreal-editor-control": {
    "inject_to": ["DevAgent", "TestAgent", "ChatAssistant"],  // ChatAssistant 也注入
    "traits_match": { "any_of": ["engine:ue5", "engine:ue4"] }
  }
}
```

**实际 Token 消耗案例**（UE5 游戏项目，ChatAssistant 一次对话）：
```
system prompt 组成：
  基础指令 + UE 意图路由            ~1200 tokens
  unreal-editor-control Skills 全文   ~800 tokens  ← 每次强制注入
  ──────────────────────────────────────────────
  合计                              ~2000 tokens

用户问："帮我查一下这个需求的进度"
→ 实际用到 unreal-editor-control 的概率：0%
→ 800 tokens 完全浪费
```

**问题清单**：

| 问题 | 说明 |
|---|---|
| 不可见 | 哪个 Skill 生效无从知晓，全混在 system prompt 里 |
| 无法追踪 | AI 用了哪个 Skill 的知识，无任何记录 |
| 浪费 Token | 每次对话注入全部 Skill 内容，不管当前问题是否需要 |
| 无法按项目配置 | 靠 traits 自动匹配，项目无法手动选择启用/禁用某个 Skill |
| 无法自定义 | 用户无法为项目添加专属 Skill，需要改代码 |
| 全局污染 | 同一 Agent 在所有项目里用同一套 Skill，不能差异化 |

## 2. 改造目标（主动触发架构）

**核心思路**：把 Skill 从「静态注入的背景知识」变成「AI 可以主动调用的工具」。

```
启动时：
  只生成 Skill 索引（名称 + 描述 + 触发场景），不加载全文

每次 LLM 调用时：
  system_prompt = 基础指令 + Skill索引表（轻量） + 项目上下文

AI 推理过程中：
  判断当前问题需要哪个 Skill
  → 显式调用 load_skill("unreal-cpp-dev")
  → Skill 全文动态注入到当前对话
  → 思考日志显示「📚 加载 Skill: UE C++ 开发规范」
```

**改造后代码草图**：

`backend/agents/base.py` — 初始化只生成索引，不加载全文：
```python
# BaseAgent.__init__() — 改造后
from skills import skill_loader
# 只生成轻量索引表，不加载 prompt.md 全文
self._skills_index = skill_loader.build_index_for_agent(self.agent_type)
# _skills_index 示例（~200 tokens）：
# | unreal-cpp-dev | 写UE C++/修复编译/UPROPERTY宏 |
# | git-workflow   | 提交/分支/合并/PR流程          |
```

`backend/actions/chat/load_skill.py` — 新增工具，按需加载单个 Skill 全文：
```python
class LoadSkillAction(ActionBase):
    async def run(self, context):
        skill_id = context["skill_id"]
        project_id = context.get("project_id")

        # 1. 验证该 Skill 对此项目可用
        available = await _get_project_skills(project_id)
        if skill_id not in available:
            return ActionResult(success=False, error=f"Skill {skill_id} 不可用")

        # 2. 读取 prompt.md 全文（原来是在 build_prompt_for_agent 里批量做的）
        content = skill_loader.get_skill_prompt(skill_id)

        # 3. 返回内容，LLM 在当前对话上下文中立即可用
        return ActionResult(
            success=True,
            data={"type": "skill_loaded", "skill_id": skill_id,
                  "name": skill_loader.skills[skill_id]["name"],
                  "content": content}
        )
```

`backend/agents/chat_assistant.py` — system prompt 改为注入索引而非全文：
```python
# _build_system_prompt() — 改造后
skills_section = ""
if self._skills_index:
    skills_section = f"""
## 可用 Skills（按需加载）

{self._skills_index}

如当前问题需要以上领域的深度规范知识，请先调用 load_skill 加载对应文档，
再回答用户。不确定是否需要时，可直接回答，不强制加载。
"""
# 索引仅 ~200 tokens，相比原来 ~2000 tokens 节省 90%
```

## 3. 修改前后详细对比

### 3.1 Token 消耗

**改造前**：
```
每次对话 system prompt（DevAgent 示例）：
  基础指令：        ~800 tokens
  unreal-cpp-dev：  ~600 tokens  ← 不管问什么都注入
  unreal-project：  ~400 tokens  ← 不管问什么都注入
  git-workflow：    ~300 tokens  ← 不管问什么都注入
  unreal-editor：   ~500 tokens  ← 不管问什么都注入
  ─────────────────────────────
  合计：           ~2600 tokens / 次
```

**改造后**：
```
每次对话 system prompt：
  基础指令：        ~800 tokens
  Skill 索引表：    ~200 tokens（只有名称+描述）
  动态加载 Skill：  ~600 tokens（只加载用到的 1-2 个）
  ─────────────────────────────
  合计：           ~1600 tokens / 次（节省约 38%）
```

**改造前**：需要修改代码，在 `backend/skills/packs/` 新建目录 + `skills.json` 注册。

```

**改造前**：
```
AI 无需决策，所有 Skill 已经在 system prompt 里了

## 4. 实现方案

### 4.1 新增 `load_skill` Action

**文件**：`backend/actions/chat/load_skill.py`

```python
class LoadSkillAction(ActionBase):
    """按需加载 Skill 全文到当前对话上下文"""

    tool_schema = {
        "name": "load_skill",
        "description": "当你需要特定领域知识时，加载对应的 Skill 文档。只在真正需要时调用。",
        "input_schema": {
            "skill_id": {"type": "string", "description": "Skill ID，从索引表中选择"},
            "reason": {"type": "string", "description": "为什么需要这个 Skill"}
        }
    }

    async def run(self, context):
        skill_id = context["skill_id"]
        project_id = context.get("project_id")
        # 1. 从项目配置的可用 Skill 里查找
        # 2. 读取对应的 prompt.md 内容
        # 3. 返回内容（LLM 会把它注入到当前对话上下文）
        content = await _load_skill_content(skill_id, project_id)
        return ActionResult(
            success=True,
            data={"type": "skill_loaded", "skill_id": skill_id, "content": content}
        )
```

### 4.2 Skill 索引生成

System prompt 里只放索引表：

```
## 可用 Skills（按需加载）

| Skill ID | 适用场景 |
|---|---|
| unreal-cpp-dev | 写 UE C++ 代码、修复编译错误、UPROPERTY/UFUNCTION 宏 |
| git-workflow | 提交代码、创建分支、合并、PR 流程 |
| my-fps-patterns | 本项目专属的 FPS 设计规范（自定义）|

如果当前问题需要以上任何领域的深度知识，先调用 load_skill 加载对应文档。
```

### 4.3 项目级 Skill 配置表

**新增 DB 表**：
```sql
CREATE TABLE IF NOT EXISTS project_skills (
    id           TEXT PRIMARY KEY,
    project_id   TEXT NOT NULL REFERENCES projects(id),
    skill_id     TEXT NOT NULL,        -- "unreal-cpp-dev" 或自定义 ID
    source       TEXT DEFAULT 'global', -- 'global' | 'custom'
    enabled      INTEGER DEFAULT 1,
    custom_path  TEXT,                 -- 自定义 Skill 文件路径
    created_at   TEXT NOT NULL
);
```

### 4.4 自定义 Skill 上传

**API**：`POST /api/projects/{id}/skills/upload`

接受 `.md` 文件，存入 `backend/data/project_skills/{project_id}/` 目录，自动注册到 `project_skills` 表。
