# Skills 系统管理约定

> 日期：2026-05-11
> 状态：当前有效

---

## 一、目录结构

```
backend/skills/
├── packs/                    # 内置 Skill（跟随系统版本发布）
│   ├── react_dev/prompt.md
│   ├── fastapi_dev/prompt.md
│   ├── git_workflow/prompt.md
│   ├── unreal_cpp_dev/prompt.md
│   ├── unreal_editor_control/prompt.md
│   └── unreal_project/prompt.md
│
├── marketplace/              # 市场目录（浏览用，不直接使用）
│   └── {skill-name}/SKILL.md  ← 复制进来即出现在市场面板
│
├── use_skills/               # 已安装目录（统一管理，AI 实际加载的来源）
│   ├── unreal-actor-editing/SKILL.md    ← UCP Skills（从 ue_plugins 迁移）
│   ├── unreal-blueprint-editing/SKILL.md
│   ├── ... (共 24 个 UCP Skills)
│   └── {marketplace-installed}/SKILL.md ← 从市场安装后复制进来
│
├── rules/                    # 全局编码规范（注入所有 Agent）
│   └── global.md
│
├── skills.json               # Skill 注册表（大分类 + 注入配置）
└── loader.py                 # SkillLoader（扫描 + 过滤 + 索引）

backend/ue_plugins/UnrealClientProtocol/Skills/   ← 保留原路径（插件源码用）
    不再作为 Skill 扫描路径，不影响 AI 使用
```

---

## 二、三类 Skill 说明

| 类型 | 存放位置 | 注册方式 | 使用范围 |
|------|---------|---------|---------|
| **内置 Skill** | `packs/` | `skills.json` 手动注册（有 `prompt_file`） | 全局，随系统版本发布 |
| **市场 Skill** | `marketplace/` | 自动扫描发现，**不直接使用** | 仅供浏览和安装 |
| **已安装 Skill** | `use_skills/` | `marketplace-skills` scan_dir 自动扫描 | 全局，AI 助手按需加载 |
| **项目专属 Skill** | `{git_repo}/.Agent/skills/` | 自动扫描，skill_id 前缀 `agent.*` | 仅本项目可见 |

## 三、安装流程

### 系统级安装（全项目可用）
```
marketplace/ → 用户在「市场」面板点「添加到系统」
            → 后端复制到 use_skills/ + 热重载
            → ChatAssistant 索引里自动出现
```

### 项目级安装（仅当前项目）
```
marketplace/ → 用户在项目「Skills → 市场」点「添加到项目」
或 AI 对话：「帮我安装 xxx skill」→ InstallProjectSkillAction
            → 后端复制到 {git_repo}/.Agent/skills/
            → 下一条对话起生效（AI 重建索引）
```

### 新建项目自动安装
- 创建项目时，系统根据 `traits` 自动安装匹配的 marketplace Skill 到项目 `.Agent/skills/`
- 映射规则见 `actions/chat/create_project.py` 的 `_TRAIT_SKILL_MAP`

## 四、skills.json 配置说明

```json
{
  "skill-id": {
    "name": "显示名称",
    "description": "描述（用于索引表）",
    "prompt_file": "packs/xxx/prompt.md",  // 内置 Skill 用这个
    "type": "scan_dir",                    // 或者 scan_dir 类型
    "scan_dir": "use_skills",              // 扫描目录（相对 skills/）
    "inject_to": ["ChatAssistant"],        // 注入给哪些 Agent
    "priority": "high",                    // high / medium / low
    "enabled": true,
    "traits_match": {                      // 按项目类型过滤（可选）
      "any_of": ["engine:ue5"]
    }
  }
}
```

**当前注册的条目：**

| ID | 类型 | 说明 |
|----|------|------|
| `react-dev` | prompt_file | React/HTML 开发规范，注入 DevAgent |
| `fastapi-dev` | prompt_file | FastAPI 后端规范，注入 DevAgent |
| `playwright-e2e` | prompt_file | E2E 测试规范 |
| `git-workflow` | prompt_file | Git 工程化规范 |
| `unreal-cpp-dev` | prompt_file | UE C++ 规范（UE 项目） |
| `unreal-editor-control` | prompt_file | UE 编辑态控制说明（UE 项目） |
| `unreal-project` | prompt_file | UE 项目结构说明（UE 项目） |
| `marketplace-skills` | scan_dir | 扫描 `use_skills/`，管理所有已安装 Skill |

## 五、三层优先级配置

AI 助手确定哪些 Skill 可用，按以下优先级叠加：

```
Layer 1  skills.json 默认 enabled 值
    ↓ 覆盖
Layer 2  global_skill_settings 表（系统设置 → Skills 全局 Toggle）
    ↓ 覆盖
Layer 3  project_skills 表（项目 Settings → Skills 项目级 Toggle）
    ↓ 追加
Layer 4  {project}/.Agent/skills/ 目录（项目本地，skill_id = agent.*）
```

## 六、UCP Skills 说明

UE Editor 控制相关的 24 个 Skill（`unreal-actor-editing`、`unreal-blueprint-editing` 等）：

- **原路径**：`ue_plugins/UnrealClientProtocol/Skills/`（插件源码，保留）
- **当前路径**：`use_skills/`（已迁移，AI 从这里加载）
- **加载方式**：`load_skill("unreal-actor-editing")` → AI 读取全文，再用 `ue_call` 执行操作
- **不直接注入**：UCP Skills 只在索引表里出现，不全量注入 prompt，按需加载

---

## 七、marketplace/ 目录约定

- 支持**子文件夹分类**：`marketplace/category/skill-name/SKILL.md`
- 临时下载目录（`download_*`、`.` 开头）自动跳过
- 放入即可在「市场」面板出现，**不需要改任何配置文件**
- 当前内容：CodeBuddy 官方 Skill 包（金融、开发工具等）+ 本地 Skill

---

## 八、常见操作速查

| 操作 | 方法 |
|------|------|
| 添加新 Skill 到市场 | 复制目录到 `marketplace/`，重启服务 |
| 安装到系统（全项目可用） | 系统设置 → Skills → 🛒 市场 → 添加到系统 |
| 安装到项目 | 项目 Settings → Skills → 🛒 市场 → 添加到项目 |
| AI 对话安装 | 「帮我安装 xxx skill」 |
| 全局开关某 Skill | 系统设置 → Skills → Toggle |
| 项目级开关某 Skill | 项目 Settings → Skills → Toggle |
| 热重载 Skills | 安装/卸载时自动触发；或重启服务 |
