## ① 对话一键流 + action state 持久化（~1 天）

### ①a 对话一键流（~4h）

**场景**：用户在全局 AI 助手说"做个 UE5 FPS"。

- 当前：`confirm_project` 建完项目 → 列表新增一条 → 用户需手动切到项目内再触发 `propose_ue_framework`
- 目标：建完项目后 **后端同条消息带上"自动跳转 + 自动调 propose"信号** → 前端一键完成跳转 + 弹方案卡

**实现点**：

1. `backend/actions/chat/confirm_project.py`（或对应实现）: 成功分支若 traits 含 `engine:ue*`，在返回结构里加字段：
   ```python
   "auto_next": {"type": "propose_ue_framework", "project_id": pid}
   ```
2. `frontend/app.js` 全局聊天处理 tool_use 结果时：
   - 识别 `auto_next` → 切换到该项目详情 + 打开项目内 AI 助手
   - 项目内 AI 助手自动调 `POST /api/projects/{pid}/ue-framework/propose`
   - 方案卡渲染到**项目内聊天**，不是全局聊天（避免混淆上下文）
3. 降级路径：`auto_next` 不识别时走老流程（建完项目就停）。

### ①b action state 持久化（~4h）

**场景**：所有 confirm_* 卡片（建项目 / UE 框架方案 / 需求确认 / 通用 confirm）刷新后都应该知道"之前点过了"。

- 当前：`uefw_done:{pid}` 用 localStorage 扛，只管 UE 一种
- 目标：通用方案——每条 chat_message 带一个 `action_state` 字段（pending / executed / cancelled），前端按状态分流渲染（已执行 → 灰色 + 历史摘要，已取消 → 灰色 + 原因）

**实现点**：

1. `chat_messages` schema 加列：
   ```
   action_state  TEXT  DEFAULT NULL     -- pending / executed / cancelled
   action_result TEXT  DEFAULT NULL     -- JSON 摘要：存被点按钮 / 时间 / 关键产物
   ```
2. 前端渲染 `confirm_*` / `propose_*` 卡片时按 `action_state` 分流：
   - `null` / `pending` → 正常渲染，按钮可点
   - `executed` → 置灰 + "✅ 已执行于 xx"
   - `cancelled` → 置灰 + "✗ 已取消"
3. 点击按钮成功后前端 `PATCH /api/chat/messages/{mid}/action-state` 更新状态
4. 清理 v0.18 的 `uefw_done:*` localStorage 逻辑（迁移完成后删）

## ② v0.19 UE run_playtest（1-2 周）

### 目标

让 TestAgent 能在 UE 项目上触发 **Automation Framework** 跑冒烟测试，输出通过 / 失败 + 截图 + 日志摘要。

### 核心 Action：`run_ue_playtest`

**接入方式**：
```
UnrealEditor.exe <uproject> -ExecCmds="Automation RunTests <TestName>;Quit" \
                 -nullrhi -unattended -buildmachine -log <output>
```

**关键决策**：

| 问题 | 方案 |
|---|---|
| 跑什么测试 | 先支持 `Project.Functional Tests`（UE 自带的 Functional Tests 层），项目可在 Automation 菜单录制 |
| 有无 GPU | `-nullrhi`（无 GPU 纯 CPU 渲染），适合 CI |
| 超时 | 默认 5 分钟，可在 SOP 里配 |
| 结果解析 | 正则抓 `Automation Test Succeeded: XXX` / `Automation Test Failed: XXX (<cause>)` |
| 截图 | `-AutomationScreenshotsDir=<path>` 失败时帮助排查 |

### Phase 拆分

| Phase | 内容 | 估时 |
|---|---|---|
| ②A | `run_ue_playtest` Action + 日志解析 + 超时 kill + 截图抓取 | 2 天 |
| ②B | SOP 加 `run_playtest` stage（engine_compile_passed → run_playtest → code_review）| 0.5 天 |
| ②C | 前端「播放测试」Tab：显示 pass/fail + 截图 gallery + log 折叠 | 2 天 |
| ②D | Automation 测试**自动生成脚手架**：ProductAgent 拆单时若涉及交互功能，自动附一个 Functional Test C++ 类雏形 | 2 天 |
| ②E | reject_goto：play_test_failed → DevAgent.fix_issues（v0.18 通用派生已支持）| 0.5 天 |
| ②F | smoke test + 真 TestFPS 实战 | 2 天 |

**合计 ~9 天**（有 buffer，按 v0.18 的经验很可能 3-4 天搞定）

### 风险

- headless UE 启动慢（30s+）→ 第一次跑可能超 5 分钟（着色编译），考虑预热
- Automation Framework API 变动大（UE4 vs UE5）→ 先只支持 UE5
- 没有现成测试时需引导用户录制 → UX 考虑

## ③ 仓库视图 / diff / 分支树（~1 天）

### ③a 文件浏览器增强（~3h）

- 大文件（>500KB）截断预览 + "查看完整" 按钮
- 语法高亮对齐：按扩展名显式声明 `highlightjs` 语言（当前 `.uproject` / `.Build.cs` / `.Target.cs` 都不认）
- 二进制文件显示"二进制文件，无法预览"占位（当前报错）

### ③b Diff 视图（~3h）

- 左右分栏 side-by-side（当前 unified）
- 行级高亮：++add --del
- 大段未变化行折叠（>10 行连续未变化→ 折叠成 "... 20 lines unchanged"）

### ③c 分支树形显示（~2h）

- 当前 `GET /api/projects/{pid}/branches` 返回扁平数组
- 新增：解析 upstream / merge-base，派生父子关系
- 前端渲染 `feat/* → develop → main` 树形 + 每分支 ahead/behind + 最近 commit
- 用 `git for-each-ref --format='%(refname:short) %(upstream:short) %(committerdate:iso) %(subject)'`
