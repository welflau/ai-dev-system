## Phase 0：基础设施准备（3 天）

> 前置条件：所有后续 Phase 都依赖这里的基础工作

### P0-1. SOP YAML 扩展（1 天）

支持 Agent 并行执行和依赖声明：

```yaml
# 新增字段
stages:
  - id: art_production
    parallel_with: development    # 与 development 并行
  - id: testing
    depends_on: [asset_integration]  # 等待完成后才开始
    parallel_with: code_review
  - id: write_code
    mode: IMPLEMENT               # 行为模式标签（Phase 4用）
```

**改动**：`sop/loader.py` + `orchestrator.process_ticket()` 支持并行调度

---

### P0-2. Agent 产出物写文件到 Git（1 天）

现状：中间产出物（PRD/架构设计/UX设计）只存 DB JSON，`path: null`。

**改动**：`orchestrator._process_ticket_result()` 每个 Agent 完成时同步写文件：

```
ProductAgent  → docs/Reqs/{req_id}/{ticket_id}/PRD.md
ArchitectAgent → docs/Reqs/{req_id}/{ticket_id}/架构设计.md
UX Agent      → docs/Reqs/{req_id}/{ticket_id}/UX设计.md        （新增Agent用）
美术 Agent    → docs/Reqs/{req_id}/{ticket_id}/视觉规范.md       （新增Agent用）
               docs/Reqs/{req_id}/{ticket_id}/asset_manifest.yaml
```

同时写入时自动打 `agent_scope` 标签（PRD.md → planner，UX设计.md → ux 等）

---

### P0-3. 数据库变更（0.5 天）

```sql
-- knowledge_index 加 scope 字段（向后兼容）
ALTER TABLE knowledge_index ADD COLUMN agent_scope TEXT DEFAULT NULL;

-- 新增四张领域知识表 + FTS5 索引

CREATE TABLE ux_knowledge (...);          -- G_Knowledge/design/ux/

```

---

### P0-4. KnowledgeLoader 基础框架（0.5 天）

新增 `backend/knowledge_config.yaml` + `backend/knowledge_loader.py`：
- 每个 Agent 声明自己的知识来源（project / domain / spec / insight）

## Phase 1：新 Agent 体系（7.5 天）

> 依赖：Phase 0 全部完成

### P1-1. 策划 Agent 从 ProductAgent 独立（1.5 天）

**现状**：ProductAgent 合并了"需求分类 + 写 PRD + 拆单"，PRD 质量不可控。

**改动**：
- 新建 `backend/agents/planner.py`（PlannerAgent）
- ProductAgent 保留"需求入口/分类"，拆单逻辑移到 PlannerAgent
- PlannerAgent system prompt 强调：验收标准可量化，PRD 末尾输出"资产需求线索"章节
- SOP YAML 新增 `planning` 阶段（ProductAgent 分类后流转到 PlannerAgent）
- 配置 knowledge_config.yaml 中 PlannerAgent 的知识来源

**PRD 末尾固定新增章节**：
```markdown
## 资产需求线索
- 音频：xxx
- 动画：xxx
- 图片：xxx
```

---

### P1-2. UX Agent（2 天）

**新建** `backend/agents/ux.py`（UXAgent）

**职责**：
- 产出交互流程图（Mermaid）
- 产出组件线框说明（组件清单 + 布局层级）
- 定义所有交互状态（正常/hover/disabled/loading/empty/error）
- 定义响应式规则
- 产出 `asset_manifest.yaml` 初版（UI 层资产）

**SOP 位置**：PlannerAgent → **UXAgent** → 美术Agent  
**Traits 触发**：`platform:web/wechat/mobile/desktop` 或 `category:game`

---

### P1-3. 美术 Agent（2 天）

**新建** `backend/agents/art.py`（ArtAgent）

**职责**：
- 读取 PRD 资产线索 + UX 组件清单
- 产出 Design Token（JSON/CSS 变量）
- 产出组件视觉规范（Markdown）
- 完善 asset_manifest.yaml（补全所有类型资产）
- 对每个资产判断生产路径（资产库→网络搜索→AI生图→CSS生成→占位）

**SOP 位置**：UXAgent → **ArtAgent**（与 ArchitectAgent+DevAgent 并行）  
**资产识别责任链**：UX（UI层资产）+ PRD线索 → 美术Agent汇总 → asset_manifest.yaml

---

### P1-4. 图片处理 Agent（异步服务型）（1.5 天）

**新建** `backend/agents/image_processor.py` + `image_requests` 表

**设计要点**：
- 任意 Agent 发出生图请求 → 立即返回 `[IMG_PENDING:xxx]` 占位
- 后台队列异步处理 → 生成完成后替换文档占位符 + 推送 SSE 事件
- 生成结果自动入 G_ArtRes 资产库
- 未配置 `IMAGE_GEN_API_URL` 时降级为 LLM 生成 ASCII 图

---

### P1-5. 技美并行流程接入（0.5 天）

利用 P0-1 的 SOP 扩展，配置完整的技美并行流程：

```yaml
stages:
  - id: ux_design
    agent: UXAgent
    outputs: [asset_manifest.yaml]
  - id: art_production
    agent: ArtAgent
    parallel_with: development
  - id: development
    agent: [ArchitectAgent, DevAgent]
    parallel_with: art_production
  - id: asset_integration      # 轻量校验，无LLM调用
    depends_on: [art_production, development]
  - id: testing
    parallel_with: code_review
    depends_on: [asset_integration]
  - id: code_review
    parallel_with: testing
    depends_on: [asset_integration]
```

## Phase 4：流程优化（3.5 天）

### P4-1. 行为模式标签 [MODE:xxx]（1 天）

在 SOP stage 配置里加 `mode` 字段，Agent 根据 mode 切换 prompt 开头：

```yaml
stages:
  - id: write_code
    agent: DevAgent
    mode: IMPLEMENT    # 专注、严谨
  - id: self_review
    agent: DevAgent
    mode: REVIEW       # 对抗、怀疑、主动找问题
```

7 种模式：EXPLORE / DESIGN / IMPLEMENT / REVIEW / DEBUG / VERIFY / REFLECT

---

### P4-2. 对抗性代码审查强制（0.5 天）

ReviewAgent 必须输出 🔴严重/🟡警告/🟢建议 三级列表，严重问题必须修复，禁止"代码质量良好"类敷衍结论。

---

### P4-3. Smart Probe 需求清晰度（1 天）

策划 Agent 生成 PRD 前，5 维度评分（用户价值/范围边界/交互方式/数值参数/关联影响）：
- ≥20：直接开发
- 15-19：自动补假设，标注 `[假设]`
- <15：提 ≤3 个关键问题（附默认假设，用户可说"按默认来"跳过）

---

### P4-4. 需求规模自适应（0.5 天）

| 规模 | 条件 | 跳过 |
|---|---|---|
| XS | 1工单、<50行 | ArchitectAgent |
| S | 1-2工单 | ArchitectAgent 轻量过 |
| M | 3-5工单 | 标准流水线 |
| L/XL | >5工单 | 触发主从模式 |

---

## Phase 5：记忆与智能（4.5 天）

### P5-1. Memory 持久化系统（1.5 天）[原 TODO A]

新增 `agent_memory` 表（decision/handoff/project_status/active_worker），Orchestrator 在关键节点写入，ChatAssistant 新增 `GetMemoryAction` 查询，回答"当初为什么这样设计"。

---

### P5-2. Insight 结构化 + 置信度（1 天）

knowledge_index 加 `confidence`（high/medium/low）和 `used_count` 字段：
- 高置信度 → 自动应用（DevAgent 直接按经验做）
- 中置信度 → 提示应用（显示建议，用户确认）

---

### P5-3. 三级内省机制（1 天）

- **需求级**（每需求完成）：自动生成复盘报告（返工次数/门禁通过/耗时分析）写入 knowledge_index
- **周期级**（每 3 需求）：触发效率统计 + LLM 生成改进建议

---

### P5-4. 前端资产库管理界面（1 天）

资产库浏览/搜索/上传/预览页面，支持触发网络搜索入库。
