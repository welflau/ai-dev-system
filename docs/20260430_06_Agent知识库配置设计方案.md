## 一、现状与问题

### 当前实现

所有 Agent 共用同一套知识检索，无差异化配置：

```python
# orchestrator._build_context() —— 对所有 Agent 一视同仁
context["knowledge_docs"] = self._load_knowledge_docs(project_id)
# → 读取 backend/docs/ + projects/{id}/docs/ 下所有 .md 文件，拼接成字符串
# → 不区分是哪个 Agent 在用，不区分知识类型
```

### 问题

1. **知识噪音**：策划 Agent 看到代码审查标准，DevAgent 看到游戏设计模式，无关内容占用 context
2. **无领域知识**：策划/UX/美术 Agent 完全没有对应的领域知识输入
3. **静态加载**：不论需求内容，全量加载所有文档，token 浪费且可能截断重要内容
4. **无优先级**：所有知识文档等权，高质量经验和低质量笔记混在一起

## 二、目标设计

### 三层知识结构

```
Layer 1：项目知识（所有 Agent，项目私有）
    knowledge_index 表（现有）+ tickets_fts（现有）
    内容：项目 docs/ 文档、历史 PRD、开发笔记、已完成工单经验
    所有 Agent 都能访问，但按 agent_scope 过滤

Layer 2：领域知识（按 Agent 类型，全局共享）
    planning_knowledge 表（G_DesignKnowledge/planning/）
    ux_knowledge 表（G_DesignKnowledge/design/ux/）
    engineering_knowledge 表（G_DesignKnowledge/engineering/）
    design_knowledge 表（G_DesignKnowledge/design/art/）
    每个 Agent 只访问本职领域的知识

Layer 3：Agent 专属规范（最小、固定）
    硬编码在 Agent 的 system prompt 或固定文件路径
    如：策划 Agent 的 PRD 模板、ReviewAgent 的代码审查清单
    不需要动态检索，直接注入
```

### 每个 Agent 的知识来源配置

| Agent | Layer 1（项目知识）| Layer 2（领域知识）| Layer 3（专属规范）|
|---|---|---|---|
| 策划 Agent | ✅ scope=planner（历史 PRD）| `planning_knowledge`（按 traits 路由到子目录）| PRD模板、验收标准写法 |
| UX Agent | ✅ scope=ux（历史 UX 文档）| `ux_knowledge`（交互模式库）| 交互状态检查清单 |
| 美术 Agent | ✅ scope=art（历史视觉规范）| `design_knowledge`（视觉设计知识）| 资产规格模板 |
| ArchitectAgent | ✅ scope=arch（历史架构文档）| `engineering_knowledge`（架构模式）| 接口设计规范 |
| DevAgent | ✅ scope=dev（历史代码+Insight）| `engineering_knowledge`（代码最佳实践）| 框架使用规范 |
| TestAgent | ✅ scope=test（历史测试报告）| — | 测试策略模板 |
| ReviewAgent | ✅ scope=review（历史审查记录）| — | 代码审查清单 |
| DeployAgent | ✅ scope=deploy | — | 部署检查清单 |

## 三、数据库变更

### 3.1 knowledge_index 表加 agent_scope 字段

```sql
-- 向后兼容：现有数据 agent_scope 默认为 NULL（等同 ALL）
ALTER TABLE knowledge_index ADD COLUMN agent_scope TEXT DEFAULT NULL;
-- agent_scope 取值：NULL/ALL / planner / ux / art / dev / arch / test / review

-- 同步更新 FTS 触发器（如有）
```

**写入时规则**：
- 项目 `docs/` 下的文档：根据文件名/路径自动判断 scope
  - `docs/Reqs/{id}/PRD.md` → scope=planner
  - `docs/Reqs/{id}/UX设计.md` → scope=ux
  - `docs/Reqs/{id}/架构设计.md` → scope=arch
  - `docs/Reqs/{id}/视觉规范.md` → scope=art
  - `docs/Reqs/{id}/test-report.md` → scope=test
  - 其他/README/普通文档 → scope=NULL（所有 Agent 可见）

### 3.2 新增领域知识表（通用结构）

```sql
-- 策划领域知识（来自 G_DesignKnowledge/planning/）
CREATE TABLE IF NOT EXISTS planning_knowledge (
    id          TEXT PRIMARY KEY,       -- PKB-xxxxxxxx
    filename    TEXT NOT NULL,          -- 相对于 G_DesignKnowledge/ 的路径
    title       TEXT NOT NULL,
    category    TEXT,                   -- games / apps / universal
    subcategory TEXT,                   -- rpg / fps / ecommerce 等
    tags        TEXT,                   -- JSON 数组
    content     TEXT NOT NULL,
    summary     TEXT,                   -- 200字摘要（快速展示）
    updated_at  TEXT NOT NULL
);
CREATE VIRTUAL TABLE IF NOT EXISTS planning_knowledge_fts
USING fts5(title, content, content='planning_knowledge', content_rowid='rowid', tokenize='trigram');

-- UX 领域知识（来自 G_DesignKnowledge/design/ux/）
CREATE TABLE IF NOT EXISTS ux_knowledge (
    id TEXT PRIMARY KEY, filename TEXT NOT NULL, title TEXT NOT NULL,
    category TEXT, tags TEXT, content TEXT NOT NULL, summary TEXT, updated_at TEXT
);
CREATE VIRTUAL TABLE IF NOT EXISTS ux_knowledge_fts
USING fts5(title, content, content='ux_knowledge', content_rowid='rowid', tokenize='trigram');

-- 工程领域知识（来自 G_DesignKnowledge/engineering/）
CREATE TABLE IF NOT EXISTS engineering_knowledge (
    id TEXT PRIMARY KEY, filename TEXT NOT NULL, title TEXT NOT NULL,
    category TEXT, tags TEXT, content TEXT NOT NULL, summary TEXT, updated_at TEXT
);
CREATE VIRTUAL TABLE IF NOT EXISTS engineering_knowledge_fts
USING fts5(title, content, content='engineering_knowledge', content_rowid='rowid', tokenize='trigram');

-- 设计领域知识（来自 G_DesignKnowledge/design/art/）

    id TEXT PRIMARY KEY, filename TEXT NOT NULL, title TEXT NOT NULL,
```

## 五、实现方案

### 5.1 新增 KnowledgeLoader 类

```python
# backend/knowledge_loader.py

import yaml
import logging
from pathlib import Path
from typing import Dict, List, Optional
from database import db

logger = logging.getLogger("knowledge_loader")

# 加载配置
_CONFIG: Dict = {}
def _load_config():
    global _CONFIG
    cfg_path = Path(__file__).parent / "knowledge_config.yaml"
    if cfg_path.exists():
        _CONFIG = yaml.safe_load(cfg_path.read_text(encoding="utf-8")) or {}

_load_config()


class KnowledgeLoader:
    """按 Agent 类型和项目 traits 加载对应的知识库内容"""

    def __init__(self, agent_name: str, project_id: str, traits: List[str]):
        self.agent_name = agent_name
        self.project_id = project_id
        self.traits = set(traits)
        self.config = _CONFIG.get("agents", {}).get(agent_name, {})

    async def load(self) -> Dict[str, str]:
        """返回各类知识内容，key 为类型标签，value 为拼接文本"""
        result = {}
        for source in self.config.get("knowledge_sources", []):
            src_type = source.get("type")
            try:
                if src_type == "project":
                    content = await self._load_project(source)
                    if content:
                        result["project_knowledge"] = content
                elif src_type == "domain":
                    content = await self._load_domain(source)
                    if content:
                        result["domain_knowledge"] = content
                elif src_type == "spec":
                    content = self._load_spec(source)
                    if content:
                        result["agent_spec"] = content
                elif src_type == "insight":
                    pass   # Insight 由 _fetch_prior_insights 单独处理，保持原逻辑
            except Exception as e:
                logger.warning("KnowledgeLoader [%s] %s 失败: %s", self.agent_name, src_type, e)
        return result

    async def _load_project(self, source: Dict) -> str:
        """从 knowledge_index 加载项目知识，按 scope 过滤"""
        scope = source.get("scope")
        char_limit = source.get("char_limit", 3000)

        if scope:
            rows = await db.fetch_all(
                """SELECT ki.filename, substr(ki.content, 1, 1000) AS preview
                   FROM knowledge_index ki
                   WHERE (ki.project_id = ? OR ki.project_id IS NULL)
                     AND (ki.agent_scope IS NULL OR ki.agent_scope = 'ALL' OR ki.agent_scope = ?)
                   ORDER BY ki.updated_at DESC LIMIT 10""",
                (self.project_id, scope)
            )
        else:
            rows = await db.fetch_all(
                """SELECT ki.filename, substr(ki.content, 1, 1000) AS preview
                   FROM knowledge_index ki
                   WHERE (ki.project_id = ? OR ki.project_id IS NULL)
                   ORDER BY ki.updated_at DESC LIMIT 10""",
                (self.project_id,)
            )

        if not rows:
            return ""

        parts = []
        total = 0
        for r in rows:
            entry = f"### {r['filename']}\n{r['preview']}"
            if total + len(entry) > char_limit:
                break
            parts.append(entry)
            total += len(entry)

        return "\n\n".join(parts) if parts else ""

    async def _load_domain(self, source: Dict) -> str:
        """从领域知识表加载，支持 path_filter_by_traits 路由"""
        table = source.get("table")
        char_limit = source.get("char_limit", 2000)
        result_count = source.get("result_count", 3)

        # 确定查询路径（按 traits 路由）
        path_filter = self._resolve_path_filter(source)

        fts_table = f"{table}_fts"

        # 先尝试全文检索，降级到直接查询
        try:
            if path_filter:
                rows = await db.fetch_all(
                    f"""SELECT t.title, substr(t.content, 1, 800) AS preview
                        FROM {fts_table} ft
                        JOIN {table} t ON ft.rowid = t.rowid
                        WHERE t.filename LIKE ?
                        ORDER BY ft.rank LIMIT ?""",
                    (f"{path_filter}%", result_count)
                )
            else:
                rows = await db.fetch_all(
                    f"""SELECT title, substr(content, 1, 800) AS preview
                        FROM {table}
                        ORDER BY updated_at DESC LIMIT ?""",
                    (result_count,)
                )
        except Exception:
            return ""

        if not rows:
            return ""

        parts = [f"### {r['title']}\n{r['preview']}" for r in rows]
        text = "\n\n".join(parts)
        return text[:char_limit]

    def _resolve_path_filter(self, source: Dict) -> Optional[str]:
        """按项目 traits 确定领域知识的子目录过滤"""
        # 简单的 path_filter（不依赖 traits）
        if "path_filter" in source:
            return source["path_filter"]

        # 按 traits 路由
        path_map = source.get("path_filter_by_traits", {})
        if not path_map:
            return None

        for trait_combo, path in path_map.items():
            if trait_combo == "default":
                continue
            required = set(t.strip() for t in trait_combo.split(","))
            if required.issubset(self.traits):
                return path

        return path_map.get("default")

    def _load_spec(self, source: Dict) -> str:
        """加载 Agent 专属规范文件（来自 G_DesignKnowledge）"""
        from config import settings
        kb_path = Path(getattr(settings, "GLOBAL_KNOWLEDGE_LOCAL_PATH", ""))
        if not kb_path.exists():
            return ""

        parts = []
        for filename in source.get("files", []):
            fpath = kb_path / filename
            if fpath.exists():
                try:
                    content = fpath.read_text(encoding="utf-8", errors="replace")
                    parts.append(f"### {fpath.name}\n{content[:1500]}")
                except Exception:
                    pass

        return "\n\n".join(parts)
```

### 5.2 改造 _build_context()

```python
# orchestrator._build_context() 改造
async def _build_context(self, ticket: Dict) -> Dict:
    # ... 现有代码 ...

    # === 知识库上下文（按 Agent 类型差异化加载）===
    # 确定当前阶段的 Agent 类型
    project_rules = await self._get_rules_for_project(ticket["project_id"])
    current_rule = project_rules.get(ticket["status"], {})
    current_agent_name = current_rule.get("agent", "")

    traits = context.get("traits", [])

    try:
        from knowledge_loader import KnowledgeLoader
        loader = KnowledgeLoader(current_agent_name, project_id, traits)
        knowledge = await loader.load()

        # 分类注入，让 Agent 知道每段知识的来源和类型
        context["project_knowledge"] = knowledge.get("project_knowledge", "")
        context["domain_knowledge"]  = knowledge.get("domain_knowledge", "")
        context["agent_spec"]        = knowledge.get("agent_spec", "")

    except Exception as e:
        logger.warning("知识库加载失败（非致命）: %s", e)
        # 降级到现有逻辑
        context["knowledge_docs"] = self._load_knowledge_docs(project_id)

    return context
```

### 5.3 Agent System Prompt 中的知识注入格式

```python
# 以策划 Agent 为例
def _build_planner_prompt(self, context: dict) -> str:
    project_kb = context.get("project_knowledge", "")
    domain_kb  = context.get("domain_knowledge", "")
    agent_spec = context.get("agent_spec", "")

    knowledge_section = ""
    if project_kb:

...
```

### 5.4 knowledge_index 写入时自动打 scope 标签

在 `orchestrator._process_ticket_result()` 中，Agent 完成后写 artifacts 时同步写 knowledge_index，并根据文件路径自动打 scope：

```python
# 文件路径 → agent_scope 映射
_PATH_SCOPE_MAP = {
    "PRD.md":         "planner",
    "UX设计.md":      "ux",
    "视觉规范.md":    "art",
    "架构设计.md":    "arch",
    "test-report.md": "test",
    "code-review.md": "review",
    "deploy.md":      "deploy",
    # 其他文件默认 None（所有 Agent 可见）
}

def _infer_scope(filepath: str) -> Optional[str]:
    filename = Path(filepath).name
    return _PATH_SCOPE_MAP.get(filename)
```

## 六、运行时流程

```
Orchestrator 确定当前阶段的 Agent（如 PlannerAgent）
    ↓
读取 knowledge_config.yaml 中 PlannerAgent 的配置
    ↓
KnowledgeLoader.load() 并行查询三层知识
    ├── Layer 1: knowledge_index WHERE scope IN (NULL, 'planner')
    ├── Layer 2: planning_knowledge WHERE filename LIKE 'games/rpg/%'
    │           （根据 project traits 自动路由到 games/rpg/）
    └── Layer 3: 读取 G_DesignKnowledge/universal/PRD模板.md
    ↓
三段知识分别放入 context 不同字段
    ↓
PlannerAgent._build_prompt() 将三段知识注入 system prompt 的不同章节
    ↓
PlannerAgent 生成 PRD（有完整的项目历史 + 领域模式 + 写作规范支撑）
```

## 七、初始化脚本

系统启动时，自动从 G_DesignKnowledge 同步数据到各领域知识表：

```python
# backend/scripts/sync_knowledge_repos.py

async def sync_planning_knowledge():
    """从 G_DesignKnowledge/planning/ 同步到 planning_knowledge 表"""
    kb_path = Path(settings.GLOBAL_KNOWLEDGE_LOCAL_PATH) / "planning"
    if not kb_path.exists():
        logger.warning("G_DesignKnowledge/planning/ 不存在，跳过同步")
        return

    count = 0
    for md_file in kb_path.rglob("*.md"):
        rel_path = md_file.relative_to(kb_path.parent)  # planning/games/rpg/战斗系统.md
        content = md_file.read_text(encoding="utf-8")

        parts = rel_path.parts  # ('planning', 'games', 'rpg', '战斗系统.md')
        category    = parts[1] if len(parts) > 1 else ""
        subcategory = parts[2] if len(parts) > 2 else ""
        title = md_file.stem

        await db.execute("""
            INSERT OR REPLACE INTO planning_knowledge
            (id, filename, title, category, subcategory, tags, content, summary, updated_at)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
        """, (
            f"PKB-{hashlib.md5(str(rel_path).encode()).hexdigest()[:8]}",
            str(rel_path),
            title,
            category,
            subcategory,
            json.dumps([category, subcategory, title], ensure_ascii=False),
            content,
            content[:200],
            now_iso(),
        ))
        count += 1

    # 重建 FTS5 索引
    await db.execute("INSERT INTO planning_knowledge_fts(planning_knowledge_fts) VALUES('rebuild')")
    logger.info("planning_knowledge 同步完成: %d 篇", count)
```
