## 1. confirm_project 卡片流式回复后不显示（修复）

**提交**：`ff3c4f8`

**根本原因**：`_buildAnyActionCardHtml` 对 `confirm_project` 类型直接返回 `''`，注释称「已在特殊路径处理」，实际根本没有特殊路径。历史加载走 `appendChatBubble` 另一条路径，所以刷新后有卡片，实时没有。

**修复**：新增 `_renderConfirmProjectCard(action)` 函数，找到最后一个 assistant 气泡直接插入卡片并触发 setTimeout 副作用（Preview 预览、路径填充等）。

## 4. get_build_logs SQL 列名错误（修复）

**提交**：`205268f`

`ticket_logs` 表没有 `message` 列，只有 `detail`。`get_build_logs` Action 的 SQL 引用了 `tl.message`，导致 AI 助手查询编译日志时报错。

修复：`tl.message` → `tl.detail`，WHERE 过滤条件同步修正。

## 6. 回复截断提示（stop_reason=max_tokens）

**提交**：`de1bf5b`

AI 说「稍等，我去处理…」但回复就此结束，用户不知道原因。

**实现**：
- `MessageDoneEvent` 加 `stop_reason` 字段
- `engine.py` 从 LLM `stop` 事件取 `stop_reason` 并透传至前端
- `max_tokens` 时在思考面板下方插入橙色警告：

```
⚠️ 回复被截断（超出单次输出上限）
发送 继续 让 AI 接着执行未完成的操作
```
