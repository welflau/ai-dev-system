## P6-2：竞品反拆管线（commit `edec3c4`）

### 背景

KM 三篇中提到「用 AI 浏览器自动化 + 视频分析持续反拆竞品，产出结构化文档入知识库」。策划 Agent 生成 PRD 时，能自动检索到竞品设计参考。

### 实现

#### CompetitorAnalysisAction（`backend/actions/chat/competitor_analysis.py`）

**流程**：抓取 URL 页面文本 → LLM 结构化分析 → 保存报告 → 入库

**产出字段**（`CompetitorAnalysisOutput`）：

| 字段 | 内容 |
|---|---|
| `core_loop` | 用户每次打开做什么，核心循环（操作→反馈→奖励→再操作）|
| `system_design` | 系统设计亮点（背包/战斗/成长/社交等）|
| `ux_highlights` | UX/交互设计值得借鉴的地方 |
| `monetization` | 商业化策略（内购/广告/订阅/道具类型/定价）|
| `strengths` | 3-5 个可借鉴的优势 |
| `weaknesses` | 2-3 个明显不足或玩家痛点 |
| `key_insights` | 最重要的 3 条洞察 |

**自动存入知识库**：
1. 报告保存到 `G_DesignKnowledge/planning/competitors/{game_slug}.md`
2. 内容入库 `planning_knowledge` 表（`category=competitors`）
3. 下次策划 Agent 分析类似游戏时，自动命中竞品报告

#### ChatAssistant 触发

system prompt 加入触发条件：
```
"分析 XXX 游戏/产品""竞品调研""看看 XXX 是怎么设计的" → competitor_analysis
```

#### 独立 API 端点

```
POST /api/competitor-analysis
  Body: { url, game_name, focus }
  → 分析结果 + saved_path + 报告全文

GET /api/competitor-analysis/list
  → 已分析的竞品列表
```

## P6-3：CLI 工具链（commit `edec3c4`）

### 新增 `backend/aidcs.py`

参考 KM 三篇的 vibecli 理念——把常用操作变成可记录、可追溯的命令。

**支持命令**：

| 命令 | 说明 |
|---|---|
| `aidcs start` | 启动后台服务 |
| `aidcs status` | 查看系统指标（处理中工单/内存/DB响应）|
| `aidcs projects` | 列出所有项目 |
| `aidcs req <pid>` | 列出项目需求 |
| `aidcs req <pid> "标题"` | 提交新需求 |
| `aidcs pause <pid>` | 批量暂停项目所有进行中需求 |
| `aidcs search <q>` | 搜索策划知识库 |
| `aidcs assets <q>` | 搜索美术资产库（33,263条）|
| `aidcs competitor <url>` | 触发竞品反拆分析 |
| `aidcs logs <pid>` | 查看最近操作日志 |

**使用示例**：
```bash
python backend/aidcs.py status
python backend/aidcs.py projects
python backend/aidcs.py competitor https://store.steampowered.com/app/xxx
python backend/aidcs.py search "背包系统设计"
python backend/aidcs.py assets "home icon"
```

**技术说明**：
- 纯 Python 标准库实现，无需额外依赖
- 通过 HTTP 调用本地 API（默认 localhost:8000）
- 支持 `AIDCS_API` 环境变量覆盖 API 地址
- Windows GBK 编码兼容（强制 UTF-8 stdout）
