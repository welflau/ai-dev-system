**步骤：**

- `success = 1`，`duration_ms` 有正数值

**可视化效果：** 无前端 UI，通过 SQLite 查询验证（后续可做审计仪表板）

### T1-2：shell_rate_limit_hook 限流验证

**步骤（Python 脚本）：**
```python
# 在 backend/ 目录运行
import asyncio, sys
sys.path.insert(0, '.')

async def test():
    from hooks.registry import hook_registry
    from hooks.builtin import shell_rate_limit_hook, _shell_call_counts
    from hooks.types import HookEvent, ToolHookContext

    hook_registry.register(shell_rate_limit_hook)
    _shell_call_counts['test-tk-001'] = 51   # 模拟超限

    ctx = ToolHookContext(
        event=HookEvent.PRE_TOOL_USE,
        tool_name='ShellAction',
        input={},
        ticket_id='test-tk-001',
    )
    try:
        await hook_registry.emit(ctx, blocking=True)
        print('❌ 应该抛出限流异常')
    except RuntimeError as e:
        print(f'✅ 限流生效: {e}')

asyncio.run(test())
```

**期望输出：** `✅ 限流生效: Shell 调用次数超限（ticket=test-tk-001, count=52, limit=50）`

## Phase 1B — 预算/配额约束

### T1-4：CHAT_MAX_TURNS 轮次限制

**步骤：**
1. 临时在 `.env` 设置 `CHAT_MAX_TURNS=2`，重启服务
2. 在 AI 助手发送一条会触发 3 次以上工具调用的消息，如：
   > 「先搜索知识库关于登录的内容，再查一下当前 git 分支，再列一下项目根目录文件」

**可视化效果（完整流程）：**

  └─────────────────────────────────┘
  
  同时出现 Toast 提示：
  ⚠️ AI 请求高风险操作审批：删除文件（rm -rf）

  ┌──────────────────────────────────────────┐
  │  🔒 待审批操作                          ✕│
  ├──────────────────────────────────────────┤

## 常见问题排查

### 审批弹窗没出现
- 检查 `permission_gate.check()` 是否被调用（在 executor.py 和 engine.py 中各有一处）
- 检查 SSE 连接是否正常（F12 → Network → EventStream）
- 检查 `backend/server.log` 是否有 `高风险操作需要审批` 字样

### tool_audit_log 没有记录
- 检查服务启动日志是否有「内置 Hooks 已注册」
- 确认走的是 QueryEngine 路径（`chat_stream` 内部用 `engine.run()`）
- `tool_audit_log` 写入在 POST_TOOL_USE，只有工具成功执行才写

### 子任务完成后父任务没恢复
- Orchestrator 轮询间隔约 5s，等待后刷新
- 检查子任务 status 是否真的到了 `testing_done` 或 `deployed`
- 查 `ticket_logs` 看是否有 `subtasks_complete` 记录
