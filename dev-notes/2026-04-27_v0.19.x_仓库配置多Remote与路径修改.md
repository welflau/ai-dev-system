# 开发日志 — 2026-04-27（仓库配置：多 Remote 管理 + 本地路径修改）

## 背景

用户反馈两个需求：
1. 项目本地 `git_repo_path` 之前完全不可修改（readonly disabled），换机器或迁移路径后无法纠正
2. 系统只支持单个 `origin` Remote，无法配置多个 Remote，也无法指定 AI push 到哪个目标

本次会话在同一天完整交付这两个功能。

---

## 一、本地路径修改

### 需求
用户能修改项目绑定的本地仓库路径，该操作影响 AI 读写代码的目录，需谨慎处理。

### 实现

**后端**（`backend/api/projects.py`）：
- 新增专用端点 `PUT /{pid}/repo-path`，与通用 `PUT /{pid}` 分离，明确语义
- 校验路径存在（`os.path.isdir`）；不存在 → 400
- 检测是否 git 仓库（`.git` 目录）；存在但非 git → 200 + `warning` 字段
- 写 `projects.git_repo_path` + 记日志

**前端**（`frontend/index.html` + `app.js`）：
- 本地仓库路径 input 旁加「✏️ 修改」按钮（`btn-warning-outline` 样式）
- 点击展开编辑区：黄色警告横幅 + 新路径输入框 + 保存/取消
- 保存前弹 `confirm()` 二次确认
- 后端报错时内联显示错误（不用 toast，方便用户看到具体原因）

## 二、多 Remote 管理

### 需求
- 支持配置多个 Git Remote（origin / upstream / 任意自定义名）
- 一个 Remote 标记为「Push 默认」，AI 的所有 push 操作走这个 Remote
- 用户能看到当前 push 目标，可随时切换

### 数据层

`database.py` `_auto_migrate` 新增两列：
```sql
projects.git_remotes     TEXT DEFAULT '[]'      -- [{name, url}, ...]
projects.git_push_remote TEXT DEFAULT 'origin'
```

`main.py` 启动时：
- 恢复 `git_push_remote` → `git_manager.set_push_remote()`
- 旧项目迁移：`git_remotes == '[]'` 且 `git_remote_url` 有值 → 自动补 `[{origin, url}]`

### GitManager（`backend/git_manager.py`）

```python
self._push_remotes: Dict[str, str] = {}   # 新增，类比 _custom_paths

def set_push_remote(project_id, remote_name)

async def push(project_id, remote=None, ...)   # remote=None → 读 _push_remotes，fallback "origin"
async def list_remotes(project_id)             # git remote -v，去重
async def add_remote(project_id, name, url)
async def remove_remote(project_id, name)
```

**关键**：所有现有 9 处 `push(project_id)` 调用**无需修改**，自动走配置的 push remote。

### 后端 API（`backend/api/projects.py`）

4 个新端点：

| 端点 | 说明 |
|---|---|
| `GET  /{pid}/git/remotes` | 列出所有 remote + push 默认标记 |
| `POST /{pid}/git/remotes` | 添加 remote `{name, url}` |
| `DELETE /{pid}/git/remotes/{name}` | 删除（保护：不能删 push 默认；至少保留一个） |
| `PUT  /{pid}/git/push-remote` | 设置 push 默认 `{remote_name}` |

所有写操作响应均从 git 实际状态重新读取（`_remotes_response`），保证前端拿到的是 git 真实情况。

**自动恢复**：`_remotes_response` 发现 git 里没有 remote 但 DB 有记录时，自动 `git remote add` 恢复（解决重启后 git config 丢失问题）。

`update_project`（通用 PUT）修改 `git_remote_url` 时同步更新 `git_remotes` 里 origin 条目，兼容旧逻辑。

### 前端（`frontend/`）

仓库配置 Tab 重构：

```
Remote 管理                          📤 Push → origin
origin  https://github.com/a/b   [📤 Push 默认] [🔗] [✕]
upstream https://github.com/x/y   [设为默认]     [🔗] [✕]
─────────────────────────────────────────────────────
＋ 添加 Remote    ← 折叠按钮，点击展开输入表单
```

- `loadRemotes()` + `_renderRemotes()` + `addRemote()` + `deleteRemote()` + `setPushRemote()`
- `loadSettingsRepo()` 改为 `Promise.all` 并行加载三个区域（remotes / repoStatus / branchInfo），消除串行等待

## 三、导入项目支持多 Remote

### 问题
`detect-local` 端点只读 `git remote -v` 第一行，多 remote 仓库导入后只保留第一个。

### 修复

**`detect-local`**（`api/projects.py`）：
```python
# 旧：只取第一行
result["git_remote_url"] = parts[1]

# 新：读全部，去重
seen = {name: url for line in out.splitlines() ...}
result["git_remotes"] = [{"name": n, "url": u} for n, u in seen.items()]
result["git_remote_url"] = remotes[0]["url"]   # 兼容旧字段
```

**`ProjectCreate` model**（`models.py`）：
- 新增 `git_remotes: Optional[List[Dict[str, str]]]`
- `git_remote_url` 改为可选（兼容旧建项目流程）

**`create_project`**（`api/projects.py`）：
- 存 `git_remotes` + `git_push_remote` 到 DB
- 非 origin 的 remote 注册到 git（`git remote add`）
- 设置 `git_manager._push_remotes`

**前端 `importProject`**：
- 新增 `_detectedGitRemotes` 变量
- 检测结果有 >1 个 remote 时，detectInfo 区显示提示「检测到 N 个 Remote: origin · upstream（将全部导入）」
- 提交时带上 `body.git_remotes`
