# 开发日志 — 2026-05-02 仓库文件浏览器「文件不存在」修复

## 问题现象

用户在工单详情抽屉点击产出文件（如 `docs/Reqs/6f78fe/46dc37/dev-notes.md`），仓库文件浏览器提示「加载失败：文件不存在」，但该文件确实存在于对应分支的 git 提交中。

实时日志显示：
```
API 请求: GET /projects/{id}/git/file?path=docs%2FReqs%2F...%2Fdev-notes.md
API 失败: ... - 文件不存在
API 请求: POST /projects/{id}/git/switch-branch   ← 前端尝试切换分支
```

## 根本原因

`git_manager.get_file_content()` 直接读取文件系统工作目录（`repo_dir / file_path`），没有指定分支。

Orchestrator 在处理工单时会频繁切换 git 分支（开发分支 → main → feature），导致用户查看文件时，工作目录已经被切到了其他分支，目标文件在当前工作目录不存在，所以返回 404。

这是一个典型的**竞态条件**：Agent 处理和用户查看在同一个 git 工作目录操作，互相干扰。

## 修复方案（commit `2fa6500`）

### backend/git_manager.py

`get_file_content()` 新增 `branch` 参数，实现三层降级策略：

```python
async def get_file_content(self, project_id: str, file_path: str,
                           branch: str = None) -> Optional[str]:
    # 1. 有 branch 参数 → git show branch:path（最可靠，不依赖工作目录）
    if branch:
        rc, out, _ = await self._run_git(repo_dir, "show", f"{branch}:{file_path}")
        if rc == 0:
            return out
    else:
        # 2. 无 branch → git show HEAD:path（读已提交内容）
        rc, out, _ = await self._run_git(repo_dir, "show", f"HEAD:{file_path}")
        if rc == 0:
            return out

    # 3. 降级 → 读工作目录文件（兼容未提交文件）
    target = repo_dir / file_path
    if not target.exists() or not target.is_file():
        return None
    return target.read_text(encoding="utf-8")
```

### backend/api/projects.py

`GET /projects/{id}/git/file` 新增 `branch` 查询参数：

```python
@router.get("/{project_id}/git/file")
async def get_git_file(project_id: str, path: str, branch: str = None):
    content = await git_manager.get_file_content(project_id, path, branch=branch)
```

### frontend/app.js

**viewRepoFile()**：读文件时从 `repoBranchSelect` 下拉获取当前选中分支，传给 API：

```js
const repoBranch = document.getElementById('repoBranchSelect')?.value || '';
const branchParam = repoBranch ? `&branch=${encodeURIComponent(repoBranch)}` : '';
const data = await api(`/projects/.../git/file?path=...${branchParam}`);
```

**openArtifactFile()**：不再调用 `POST /git/switch-branch`（原来会切换工作目录分支，干扰 Orchestrator），改为带 branch 参数直接读：

```js
// before: switch-branch + viewRepoFile（有竞态）
// after:  viewRepoFileWithBranch(path, branch)（无竞态）
setTimeout(() => viewRepoFileWithBranch(filePath, branch, null), 400);
```

## 效果

- 用户在任意时刻点击产出文件，都能通过 `git show branch:path` 读到正确内容
- 不再需要切换 git 工作目录分支，彻底消除与 Orchestrator 的竞态
- `git show HEAD:path` 作为兜底，无需指定分支时也读已提交内容而非工作目录
