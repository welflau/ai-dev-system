## 一、Dock 按钮状态切换没视觉反馈

### 1.1 表现
用户反馈："那个 📌 按钮并没有根据状态变化"。

### 1.2 问题
工单详情抽屉顶栏有个 📌 dock 按钮，点击切换 "浮动 / Dock（并排）" 模式。切换时：
- 图标硬编码在 HTML 里固定 📌，从不变化
- CSS 只给 `.active` 改了 `color: var(--accent)`，没改背景/边框 → 字色变化在小图标上看不出来

更隐蔽的第二个 bug：`_applyDrawerDockPreference()`（打开抽屉时按 localStorage 应用偏好）**只改 class 不改 title** —— 所以初次打开抽屉时 title 永远是 HTML 里的默认那句，看不出当前是哪个模式。只有手点切换时 title 才会更新。

### 1.3 修复（commit `3002bc6`）
抽 `_syncDockBtnState(isDocked)` 统一管理：
```js
function _syncDockBtnState(isDocked) {
    const btn = document.getElementById('drawerDockBtn');
    btn.classList.toggle('active', isDocked);
    if (isDocked) {
        btn.innerHTML = '📍';  // Dock 态 = 实心定位
        btn.title = 'Dock 模式开启（与主内容并排）。点击切回浮动';
        document.body.classList.add('has-docked-drawer');
    } else {
        btn.innerHTML = '📌';  // 浮动态 = 图钉
        btn.title = '浮动模式（遮盖主内容）。点击切 Dock 模式';
        document.body.classList.remove('has-docked-drawer');
    }
}
```
`toggleDrawerDock()` 和 `_applyDrawerDockPreference()` 都调它。

CSS 加强 `.active` 视觉：
```css
#drawerDockBtn.active {
    color: var(--accent);
    background: var(--accent-bg, rgba(59, 130, 246, 0.15));
    box-shadow: inset 0 0 0 1px var(--accent);
    border-radius: 4px;
}
```

## 二、诊断面板配色撞暗色主题

### 2.1 表现
用户截图 + 反馈："这个诊断结果的显示配色也太不协调，文字大小，颜色也不协调"。

截图现场：浅黄背景块 + 默认浅色字 → 几乎看不清。

### 2.2 问题
我写 `renderDiagnosisPanel()` 时偷懒用内联样式 + Light 主题思路：
```js
style="background: var(--bg-warning, #fff7e6); ..."
```

但：
1. `--bg-warning` 这个 CSS 变量在项目里**不存在**，所以每次都 fallback 到 `#fff7e6` 浅黄
2. 系统是暗色主题（`--bg: #0f1117`），浅黄块 + 父级继承的浅色字 = 反差极低
3. `<code>` 块用了 `--bg-secondary` 也不存在，再次 fallback 到未定义
4. h4 里"严重度 / 置信度"跟标题 font-weight 一样，视觉层级分不清

### 2.3 修复（commit `3e2b2ea`）
把内联样式全部抽到 `styles.css`，用**现有主题变量**重写：

- 背景 `--bg-card`：跟其他 drawer-section 一致的深色块
- 左边框用严重度色强调（`sev-low/medium/high` modifier）：
  ```css
  .diagnosis-panel.sev-high { border-left-color: var(--error); }
  ```
- 严重度徽章独立 `.diagnosis-sev-label`（彩色底 + 对应色字）：
  ```css
  .sev-high .diagnosis-sev-label { background: rgba(239,68,68,0.15); color: var(--error); }
  ```
- 内容改 grid 布局（label 68px + 1fr），字号统一 13px，文字颜色明确 `--text / --text-muted / --text-secondary`
- 相关文件 `<code>` 用 `--bg-elevated` 做对比底

**教训**：写组件内联样式时，fallback 到 hex 颜色是暗色主题下的常见坑。应该**只用已定义的 CSS 变量**，不要乱 fallback。

## 三、工单列表/看板没标出 blocked

### 3.1 表现
用户截图：工单列表（左侧"列表"导航）显示 6 个 TestFPS 工单全是 "待启动"。但 DB 里它们都是 blocked。

用户："工单列表并没有把阻塞的工单标志出来"。

### 3.2 问题（3 处独立代码都漏了）
前端不同视图的状态下拉/分组**没共享**同一份状态列表：

| 视图 | 渲染函数 | 问题 |
|---|---|---|
| 看板卡（detail page 的 board） | `renderTicketCard` | 上一轮已改好（🚧 角标 + blocked 选项） |
| **工单列表表格** | `_renderTicketRow` | **独立维护 allStatuses，没 blocked** |
| **工单列表看板** | `renderTicketListBoard` | **statusGroups 分组没 blocked 列** |

`_renderTicketRow` 的 select 匹配不到 blocked → HTML 规范下默认选第一个 option → 全部回显 "待启动"。

`renderTicketListBoard` 的分组逻辑 else 兜底到 "待启动" 组。

全局 `STATUS_LABELS` / `getStatusColor` 也没 blocked。

### 3.3 修复（commit `e925717`）
一次全部补齐：

1. **`STATUS_LABELS`**：加 `blocked: '🚧 已卡住'`
2. **`getStatusColor`**：`status === 'blocked' → var(--error)`
3. **`_renderTicketRow`**：
   - allStatuses 加 blocked option
   - 标题列加 `blockedBadge`（🚧 徽章）
   - 行加 `.tl-row-blocked` class
4. **`renderTicketListBoard`**：statusGroups 新增 `blocked` 列，颜色 `var(--error)`
5. **`styles.css`**：加 `.tl-row-blocked` 样式（左红条 + 红色底 + hover 加深）

### 3.4 架构级反思
**问题根源是代码复制**：4 处视图独立维护了各自的 "状态下拉 / 分组逻辑" 代码。任何新增状态（这次的 blocked、将来可能的 `zombie` / `retry_exhausted`）都要**同时改 4 处**才不漏。

短期可接受（v0.17 加的 blocked 足够用），但中期应该：
- 把 `allStatuses` 抽成 `STATUS_LIST` 全局常量（一次定义，各视图引用）
- 分组函数 `classifyStatus(s) → 'pending' | 'running' | 'blocked' | 'done'` 抽出来
- 新状态只改常量 + 分类函数，UI 自动跟上

加入 TODO，不阻塞本轮。
