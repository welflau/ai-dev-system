## 背景

v1.0 已实现统一时间轴和人工评论保存，但评论发出后 Agent 不会立刻响应，
只有等工单下次流转时才会将评论注入 prompt。

本次（v1.1）参考截图中 Multicast 的交互模式，实现：
**用户发评论 → Agent 立刻异步回复 → SSE 推送蓝色气泡出现**，形成真正的对话闭环。

## 实现细节

### tickets.py — 触发点

`BackgroundTasks` 已在 tickets.py 中有先例（`process_ticket` 也是这样触发的），
接口立即返回，回复在后台执行，不影响前端发送体验。

### orchestrator.py — reply_to_comment()

核心流程：

```
1. 查 tickets 表，获取工单信息（title、status、description、result 摘要）
2. 查 ticket_comments，读取完整对话历史
3. 选择 Agent：ticket.assigned_agent（当前负责）→ 回退 ChatAssistant
4. 组装 prompt：工单背景 + 历史对话 + 最新评论
5. llm_client.generate()，max_tokens=400，temperature=0.5
6. _write_phase_comment() 写入回复 + SSE 推送
```

**Agent 选择策略**：
- 工单进行中 → 用 `assigned_agent`（DevAgent / PlannerAgent 等），语气和知识域匹配
- 工单终态或无负责人 → 回退 `ChatAssistant`，保证任何状态都能回复

**Prompt 设计要点**：
- 历史对话中跳过"最新这条"（刚写入，避免重复出现）
- result 字段提取摘要（notes / prd_content / architecture），让 Agent 知道自己做了什么
- 末尾强调"不要 markdown 标题，直接自然语言"，避免回复格式过重

## 错误处理

`reply_to_comment` 整体 try-except，失败只 warning 不抛出，
保证评论保存成功后接口不受回复失败影响。
