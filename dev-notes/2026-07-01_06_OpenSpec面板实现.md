# OpenSpec 面板实现

**日期**：2026-07-01
**状态**：已上线 ✅

## 背景

在 Agent/扩展页的最右侧 Tab（📐 OpenSpec）中，替换占位页为完整功能面板。

## 功能设计

### 已安装内容（`#openspecInstalledCard`）
- 项目初始化（存在 `openspec/` 目录）后显示
- 说明 OpenSpec 提供的 Skills / Commands 已在各面板中以「来源：OpenSpec」标注

### 指令按钮区（`#openspecActionsCard`）
- 状态栏：显示 npm 可用性 / 英文版安装 / 中文版安装 / 项目是否初始化
- **安装 OpenSpec** 按钮组：
  - 英文版 → `npm install -g @fission-ai/openspec@latest`
  - 中文版 → `npm install -g @studyzy/openspec-cn@latest`
  - 已安装也可重复运行（升级）
- **初始化项目** 按钮组：
  - 英文版 → `openspec init`（在项目根目录执行）
  - 中文版 → `openspec-cn init`
  - 提示是否已初始化（检测 `openspec/` 目录是否存在且非空）

### 控制台输出（`#openspecConsoleCard`）
- 点击任意命令按钮后显示
- 实时 SSE 流式输出每行 stdout/stderr
- 命令完成后显示退出码
- 「清空」按钮隐藏控制台

## 后端（`backend/api/openspec.py`）

### `GET /api/projects/{project_id}/openspec/status`
```json
{
  "npm_available": true,
  "installed_en": false,
  "installed_cn": false,
  "initialized": false,
  "repo_path": "I:\\...",
  "openspec_dir": "I:\\...\\openspec"
}
```

检测方式：
- `npm_available`：`shutil.which("npm")`
- `installed_en/cn`：`shutil.which("openspec")` / `shutil.which("openspec-cn")`
- `initialized`：`Path(repo_path) / "openspec"` 存在且非空

### `POST /api/projects/{project_id}/openspec/run`
```json
{ "command": "install_en" }
```
command 可选值：`install_en` / `install_cn` / `init_en` / `init_cn`

返回 `text/event-stream`，SSE 事件：
- `start`：`{"cmd": "npm install -g ..."}`
- `line`：`{"text": "..."}`
- `done`：`{"exit_code": 0, "success": true}`
- `error`：`{"message": "..."}`

Windows 使用 `asyncio.create_subprocess_shell` 保证 PATH 扩展可找到 npm/openspec。

## 前端（`frontend/app.js`）

- `loadOpenSpecStatus()` — 切换到 openspec tab 时自动调用
- `_renderOpenSpecStatus(data)` — 渲染状态栏 + 初始化提示
- `runOpenSpecCmd(command)` — 调用后端接口，ReadableStream 消费 SSE
- `_appendOpenSpecLine(text)` — 追加输出到 `<pre>` 并自动滚动到底部
- `clearOpenSpecConsole()` — 清空并隐藏控制台卡片
