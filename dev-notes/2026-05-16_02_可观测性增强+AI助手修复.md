## 一、AI 助手无法对话修复

### 问题
DB 高负载时，AI 助手发消息无响应、无日志。

### 根因
`_chat_stream_generator` 第一行：
```python
await _save_chat_message(project_id, "user", req.message, ...)
```
这是**阻塞写 DB**。Orchestrator 并发工单占用写锁时，
此操作失败 → stream 在调 LLM 之前就崩溃 → 用户看到空响应。

### 修复
**`api/chat.py`**：用户消息保存改为 `asyncio.create_task`（fire-and-forget），
LLM 调用立即开始，DB 写入并发进行，互不阻塞。

**`database.py`**：`execute()` / `insert()` 加重试（3次，间隔 0.3/0.6/0.9s），
遇到 `database is locked` 自动等待重试。

**`orchestrator.py`**：并发 2→1，减少 DB 写锁竞争。

## 四、全局聊天日志可见性

### 问题
AI 助手（全局聊天，无项目上下文）的工具调用日志完全不可见。

### 根因
`audit_log_hook` 只推送到项目 SSE 频道（`project_id` 有值时），
全局聊天 `project_id=None` → 跳过推送 → 日志面板看不到。

### 修复链

```
audit_log_hook（hooks/builtin.py）
  project_id 有值 → publish_to_project（原有）
  project_id 为空 → publish("global", "log_added")  ← 新增

api/tickets.py
  新增 GET /api/events/global SSE 端点，订阅 global 频道

frontend app.js
  connectGlobalSSE()：页面加载后连接 /api/events/global
  log_added → _appendGlobalLogEntry()（全局日志面板）
            → appendLogEntry()（项目日志面板，如果在项目内）

index.html
  projectListPage 底部加 globalLogPanel
  与项目日志面板同款：搜索/过滤/清空/折叠/拖拽调高
```

### 效果
在全局 AI 助手页面发消息后，底部日志面板实时显示：
- 每个工具调用（`web_search (425ms) → query | ✓ 结果...`）
- QueryEngine 完成统计（`rounds:4 tokens:12345(4.1%) elapsed:3.2s`）
- 关键错误（`⚠️ Git Push 失败...`）
