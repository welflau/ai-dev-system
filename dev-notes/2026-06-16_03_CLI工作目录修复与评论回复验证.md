## Bug 修复：CLI 模式工作目录错误

### 问题

CLI 子进程（claude / codebuddy 等）启动时继承了后端进程的工作目录
`F:\A_Works\ai-dev-system\backend`，导致：
- AI 助手执行 `pwd` 时返回 backend 目录而非项目目录
- CLI 工具无法感知项目文件结构，代码生成/修改找不到正确路径

### 根因

`llm_client.py` 的 `_call_cli()` 和 `_call_cli_stream()` 两处
`create_subprocess_exec` 均未传 `cwd` 参数。

### 修复（backend/llm_client.py）

在两处 `create_subprocess_exec` 调用前，通过 `_llm_ctx.project_id` 查出
项目的 `git_repo_path`，验证目录存在后作为 `cwd` 传入：

```python
cwd = None
if _llm_ctx.project_id:
    proj = await db.fetch_one(
        "SELECT git_repo_path FROM projects WHERE id = ?",
        (_llm_ctx.project_id,),
    )
    if proj and proj.get("git_repo_path"):
        if os.path.isdir(proj["git_repo_path"]):
            cwd = proj["git_repo_path"]
            logger.info("🖥️  CLI cwd → %s", cwd)

proc = await asyncio.create_subprocess_exec(
    *cmd, ..., cwd=cwd,
)
```

**兜底**：`cwd=None` 时行为与修复前一致，不破坏无项目上下文的场景（全局 AI 助手）。

### 影响范围

所有使用 CLI 格式（claude / codebuddy / gemini）的 Agent 调用，
修复后工作目录自动切换到对应项目的 Git 仓库根目录。
