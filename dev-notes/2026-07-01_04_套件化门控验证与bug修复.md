## 背景

套件化门控代码（Task #1-4）完成后，做端到端验证时发现两个独立 bug，本次一并修复。

## Bug 1：pack_installer 写文件权限异常 → HTTP 500

**现象**：安装 `game-dev` 套件到 OG 项目时，installer 试图写入
`I:\A_Works\OG2\UnrealEngine_Code_Inner\.claude\skills\code-review\SKILL.md`，
该目录对 Python 进程只读，抛出 `PermissionError`，一路冒泡成 HTTP 500 Internal Server Error。

**根因**：`pack_installer.py` 的 `_install_files` 里，读文件有 try/catch（追加 `errors`），
但 `dst_root.mkdir` 和 `dst.write_text` 没有任何保护。

**修复**（`backend/pack_installer.py`）：
- `dst_root.mkdir` 加 try/catch → 失败则追加 errors 并 return，跳过该 target
- `dst.write_text` 加 try/catch → 失败追加 errors，继续处理其余文件

修复后行为：写入失败 → partial warning（errors 非空），不崩溃成 500。

## Bug 2：get_enabled_packs 模块名错误

**现象**：`get_enabled_packs(project_id)` 捕获到异常静默返回 `set()`，
导致所有项目都只注入 `global.md`，套件门控形同虚设。

**根因**：`backend/skills/loader.py` 里写的是 `from db import db`，
正确模块名是 `from database import db`（项目一贯用法）。

**修复**：单行 import 修正。

## 完整机制

```
用户在 Packs 面板点"安装"
  → POST /projects/{id}/packs/{pack}/install
  → pack_installer 复制文件到 .claude/.codebuddy
  → INSERT INTO project_packs (project_id, pack_name)
  → chat_assistant 每轮调 get_enabled_packs(project_id)
  → 查 project_packs 表，得到已安装套件集合
  → get_rules_for_context / get_skills_for_agent 按 enabled_packs 过滤
  → 只注入已安装套件的 rules/skills
```
