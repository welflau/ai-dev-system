## Phase A：仓库同步 + 路径配置

### 数据层
- `database.py`：`_auto_migrate` 新增 `projects.knowledge_scan_paths`（JSON `[{path, enabled}]`）
- `database.py`：`SCHEMA_SQL` 新增 `knowledge_index` 表 + `knowledge_fts` FTS5 虚拟表 + 3 个触发器（ai/au/ad 自动维护索引）

### 后端 API（`backend/api/knowledge.py` 重构）

新增 5 个端点（在通配 `{filename}` 路由之前定义，避免路由冲突）：

| 端点 | 说明 |
|---|---|
| `GET /search` | FTS5 全文搜索（项目 + 全局） |
| `GET /projects/{pid}/scan-paths` | 获取扫描路径 + 建议路径（检测仓库内存在的目录） |
| `GET /projects/{pid}/sync-preview` | 预览将同步的文件（不写入） |
| `PUT /projects/{pid}/scan-paths` | 更新扫描路径配置 |
| `POST /projects/{pid}/sync-from-repo` | 执行同步 |

**安全扫描**（借鉴 Hermes `_CONTEXT_THREAT_PATTERNS`）：
- 检测 prompt injection（`ignore previous instructions`、`you are now a` 等）
- 检测凭证外泄（`curl + SECRET/TOKEN`）
- 检测 SSH 后门（`authorized_keys`）
- 命中任一模式 → 跳过该文件并记录 reason

**同步逻辑**：
- 默认扫描 `docs/`、`Design/`、`Spec/`、`Docs/`（存在才扫）
- 深度限制 3 层，单文件上限 500KB
- 目标文件名：`{scan_dir}__{relative}.md`（如 `docs__架构说明.md`）
- 同步后写 `knowledge_index`，触发器自动更新 FTS5

**现有文档联动**：
- `PUT /projects/{pid}`（保存文档）：同步写 `knowledge_index`
- `DELETE /projects/{pid}/{filename}`：同步删 `knowledge_index`
- `main.py` startup：扫描现有 docs/ 目录，补录历史文档到 `knowledge_index`

## Phase B：FTS5 + Agent 主动搜索

### SearchKnowledgeAction（`backend/actions/chat/search_knowledge.py`）

```python
# tool schema 关键描述
"搜索项目知识库文档，查找已有的规范、设计说明、已知错误修复方法。
当遇到不确定的 API 用法、编译错误（查历史解决方案）、项目规范时主动调用。"
```

- FTS5 MATCH 查询，返回 `snippet()**` 高亮片段 + 前 400 字预览
- 同时搜索项目知识库（`project_id = ?`）和全局知识库（`project_id IS NULL`）
- 注册到 `ChatAssistantAgent.action_classes`

### FTS5 关键设计点

- 使用 Hermes 的精简模式：只索引 `content` 列，`content=knowledge_index` 外部内容表
- 触发器只传 `(rowid, content)` — 不带 UNINDEXED 列（避免 FTS5 索引建立失败）
- 搜索时 JOIN `knowledge_index` 取 filename/project_id
