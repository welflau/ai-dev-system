# CLI 降级路径工具协议注入修复

## 问题

使用 DeepSeek / Gemini 等非 Claude 模型时，`chat_with_tools` 进入 CLI 降级路径，
工具定义被丢弃，LLM 不知道有 `confirm_requirement` 工具，无法弹出需求创建卡片。

## 根因

`llm_client.py` CLI 降级代码：
```python
# 旧：只传 system + messages，tools 完全丢弃
if self.api_format == "cli":
    all_messages = [{"role": "system", "content": system}] + list(messages)
    response = await self.chat(all_messages, ...)
```

## 修复方案

### 1. `backend/llm_client.py`

新增 `_tools_to_action_protocol(tools)` 函数：
- 把 tool schemas 动态转换为 `[ACTION:NAME]{JSON}[/ACTION]` 文本协议说明
- 格式与 QueryEngine 现有解析器完全对齐（大写 NAME，独立 JSON 行，`[/ACTION]` 结尾）
- 包含：工具名、描述、参数说明、示例、触发时机规则

两处降级位置（`chat_with_tools` + `chat_with_tools_stream`）均注入：
```python
tool_instruction = _tools_to_action_protocol(tools)
combined_system = (system or "") + "\n\n" + tool_instruction
```

### 2. `backend/agents/chat_assistant.py`

新增两个模块级函数：
- `_parse_cli_action_text(reply, executor)` — 从 CLI 回复文本中解析 `[ACTION:xxx]...[/ACTION]`，调用 executor 执行，返回 action result
- `_strip_action_tags_from_reply(reply)` — 清理显示给用户的文本，去掉 ACTION 块

非流式 `chat()` 方法中，在拿到 result 后新增：
```python
if result.get("cli_fallback") and not action:
    cli_action = await _parse_cli_action_text(reply, executor)
    if cli_action:
        action = cli_action
        reply = _strip_action_tags_from_reply(reply)
```

流式路径（QueryEngine）已有 `[ACTION:xxx]` 解析，本次不动。

## 效果

- DeepSeek、Gemini 等模型在 CLI 模式下，识别到需求描述时会输出 `[ACTION:CONFIRM_REQUIREMENT]{...}[/ACTION]`
- 系统解析后弹出需求创建卡片，和 Claude 原生 tool_use 体验一致
- 纯对话、提问不受影响（LLM 按触发规则决定是否输出 ACTION）
