# 开发日志 — 2026-05-06 C. 研发效率统计前端 + Token 统计

## 功能概述

在统计面板基础统计（工单数/完成率/Agent 工作量）下方，追加效率专项看板：
- **需求交付效率**：平均交付周期、已完成需求数、Smart Probe 清晰度均分、最近需求列表
- **Agent LLM 耗时 & Token 消耗**：每个 Agent 的调用次数 / 总耗时 / 均耗时 / Input Token / Output Token，顶部汇总合计
- **Reflexion 返工排行**：返工次数最多的工单 Top 8

ChatAssistant（项目内 AI 助手）和 RoadmapPlanner 均正确显示在 Token 消耗表中：
- ChatAssistant：1次调用，21.2K input / 424 output tokens
- RoadmapPlanner：1次调用，387 input / 660 output tokens

## 技术细节

### 后端

**`efficiency.py`**：
- `agent_time` 查询加入 `SUM(input_tokens)`、`SUM(output_tokens)`
- 返回每个 Agent 的 token 消耗明细

**`llm_client.py`（关键 bug 修复）**：

`chat_with_tools` 路径（ChatAssistant tool_use）之前**完全不写 `llm_conversations`**，导致 AI 助手的 token 消耗在效率统计里不可见。

根本原因：只有普通 `chat()` 方法调用 `_save_conversation()`，而 `chat_with_tools` 只调用 `_call_anthropic_tools()`，后者没有保存记录。

修复：
1. `_call_anthropic_tools()` 返回值加入 `usage` 字段
2. `chat_with_tools()` 累计各轮 token，整轮结束时调用 `_save_tools_conversation()`
3. 新增 `_save_tools_conversation()`：将多轮对话汇总为一条记录写入 `llm_conversations`

### 前端

**`app.js`**：
- 新增 `_loadEfficiencyStats(grid)` 异步追加，不阻塞基础统计渲染
- Agent 耗时从条形图改为 6 列表格（Agent / 调用次数 / 总耗时 / 均耗时 / Input Token / Output Token）
- 表头显示合计 token

**`styles.css`**：
- 新增 `.eff-section`、`.eff-summary-row`、`.eff-kpi`、`.eff-table` 等样式

## 排查过程

### 问题：AI 助手 Token 全是 0

发现 ChatAssistant 不出现在 Agent LLM 耗时表中，检查后发现：
- 所有 Agent 调用（DevAgent、ProductAgent 等）走 `chat()` → 有记录
- ChatAssistant 走 `chat_with_tools()` → **无记录**
- `llm_conversations` 表里有 395 条 `agent_type=None, project_id=None` 的孤立记录，原因待查

修复 `chat_with_tools` 后，AI 助手的调用正确记录到 `llm_conversations`，统计面板显示正常。
