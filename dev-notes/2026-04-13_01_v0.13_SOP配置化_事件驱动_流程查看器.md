## 主要改动

### v0.13.0: SOP 配置化 + 事件驱动 + 流程查看器

#### 节点 1.1: SOP YAML 配置
- 新增 `backend/sop/` 目录
- `default_sop.yaml`: 6 阶段标准开发流程定义（架构→开发→验收→测试+打回）
- `loader.py`: SOP 解析引擎（PyYAML + 兜底解析器）
  - `load_sop()` → 加载 YAML 配置
  - `sop_to_transition_rules()` → 转换为 orchestrator 规则
  - `get_sop_stages()` / `get_sop_metadata()` → 供前端展示
- orchestrator `__init__` 从 SOP 加载 transition_rules，替代硬编码
- `reload_sop()` 热重载方法
- API: `GET /api/sop` + `POST /api/sop/reload`

#### 节点 1.2: 内部事件总线
- 新增 `backend/event_bus.py`
  - `InternalEventBus`: asyncio.Queue 事件总线
  - `publish()` 非阻塞发布事件
  - `start()` 消费循环，处理 `ticket_ready` 事件
- orchestrator 集成:
  - `start_event_bus()` 启动总线 + 注册 handler
  - `_publish_ticket_ready()` 工单完成后发布事件
  - 工单状态变更后检查是否在 transition_rules 中 → 发布事件 → 立即触发下一 Agent
- 轮询间隔从 10s 放宽到 30s（兜底）
- main.py: 启动事件总线 + 轮询 + CI/CD 三个后台任务

#### 节点 1.3: 前端 SOP 查看器
- 设置页新增"流程配置"标签（`settings-sop`）
- `loadSOPFlow()`: 调用 `/api/sop` 渲染流程图
  - SVG 流程图：阶段节点（彩色边框）+ 前进箭头 + 打回虚线红箭头
  - Agent 颜色映射（Architect=蓝, Dev=绿, Product=黄, Test=紫）
  - 阶段详情表格（Agent/触发/成功/打回/说明）
- `reloadSOP()`: 热重载按钮
- CSS: `.sop-flow-container` + `.sop-table`
