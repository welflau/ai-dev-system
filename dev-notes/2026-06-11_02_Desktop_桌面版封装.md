# Desktop — 桌面版封装（pywebview + pystray）

> 日期：2026-06-11
> 系列：Desktop（桌面版）

---

## 方案选型

选用 **pywebview + pystray** 而非 Electron：

| | pywebview | Electron |
|---|---|---|
| 额外依赖 | Python 包（~5MB）| Node.js 运行时（~150MB）|
| 代码改动 | 0 行现有代码 | 需要 package.json + 主进程 |
| 系统 WebView | Windows Edge/WebView2 | Chromium（内置）|
| 原生托盘 | pystray | 内置支持 |

---

## 新增文件

### `desktop.py`

主入口，四个核心部分：

1. **路径修正** — `_resource()` 函数兼容开发模式和 PyInstaller 打包模式（`sys._MEIPASS`）
2. **后端启动** — daemon 线程运行 uvicorn，端口固定为 18000（避免与开发服务 8000 冲突）
3. **加载画面** — 服务就绪前显示内联 HTML 动画，就绪后跳转到实际 URL
4. **系统托盘** — pystray 图标 + 右键菜单（显示/隐藏/退出），`run_detached()` 非阻塞

关键细节：
- `_wait_for_server()` 轮询 `/api/health`，最多等 30 秒
- pywebview `start(_on_shown)` 在窗口显示后回调，回调里等服务就绪再跳转
- 桌面版端口 `PORT=18000` 通过环境变量覆盖，不修改 `.env`

### `desktop.spec`

PyInstaller 打包配置：
- `console=False` — 无黑色控制台窗口
- 目录模式（COLLECT）启动更快，单文件模式解压慢
- 包含 `frontend/`、`backend/skills/`、`backend/sop/` 等数据目录
- `hiddenimports` 覆盖 FastAPI/uvicorn/sse_starlette 等动态加载模块

### `assets/icon.ico` / `assets/icon.png`

用 PIL 生成的应用图标（紫色圆形背景 + 白色网格）。

---

## 使用方式

```bash
# 开发模式直接运行
python desktop.py

# 打包为 exe
pyinstaller desktop.spec
# → dist/AI-Dev-System/AI-Dev-System.exe
```

---

## 已知限制

- WebView2 需要用户机器已安装（Windows 10/11 默认有，旧系统需另装）
- 打包后首次启动较慢（解压依赖），约 3~5 秒
- CLI 模式下 codebuddy 等工具需要用户环境已安装（不随 exe 打包）
