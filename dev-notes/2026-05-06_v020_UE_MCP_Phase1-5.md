# 开发日志 — 2026-05-06 v0.20 UE 编辑态 MCP 集成（Phase 1-5）

## 背景

v0.19 完成了 UE 离线链路（UBT 编译 / Playtest / Reflexion）。
编辑态能力一直缺失：AI 无法读 Editor 里的 Actor 状态、无需重编直接改属性、生成测试场景。

选型文档：`docs/20260425_01_UE编辑态MCP集成选型分析.md`
实施方案：`docs/20260506_UE_MCP_实施方案.md`

---

## 方案要点

**选型**：Italink/UnrealClientProtocol（UCP）——TCP JSON 直连，无独立进程，reflection-based 覆盖面广。

**协议**：4 字节小端长度前缀 + UTF-8 JSON，端口 9876（loopback only）

**插件安装流程**（已固化到模板实例化）：
- `propose_ue_framework` 确认卡加「启用编辑态 AI 控制（UCP 插件）」复选框
- `InstantiateUETemplateAction` 从 `backend/ue_plugins/` 快照 copy 到项目 `Plugins/`
- 用户只需手动重启 Editor（~2 min 首次编译），无需其他操作
- UCP 插件 `EnabledByDefault: true`，无需手动在 Plugins 里勾选

### Phase 1 — `UEEditorControlAction`

**文件**：`backend/actions/ue_editor_control.py`

支持的 op：

| op | 说明 |
|---|---|
| `get_actors` | 取当前关卡所有 Actor 路径 |
| `get_actors_of_class` | 按 class 过滤 Actor |
| `get_property` | 读 UObject 属性值 |
| `set_property` | 写 UObject 属性值（自动 git stash 兜底）|
| `spawn_actor` | 在关卡 Spawn Actor（自动 git stash）|
| `destroy_actor` | 删除 Actor（自动 git stash）|
| `call` | 原始 UCP 调用（object/function/params）|

安全机制：写操作前自动 `git stash push -m "ai-ucp-before-write"`，失败时 `git stash pop` 回滚。

连接失败：返回 `{"status": "editor_not_running"}`，Orchestrator 走 BLOCKED。

另提供全局工具函数 `probe_ucp(host, port, timeout)` 供健康检查调用。

### Phase 2 — Editor 健康检查

**新增端点**（`backend/api/ue_framework.py`）：
- `GET /ue-framework/editor-status` → `{connected, host, port, hint}`
- `POST /ue-framework/editor-control` → 直接调 UCP（调试/ChatAssistant 入口）

**`ci/strategies/ue.py`**：
- `editor_live` 环境卡片由占位符改为真实 `probe_ucp()` 探测
- 状态：`active`（9876 可达）/ `inactive`（未连接）

### Phase 3 — Skill Pack

**新增**：`backend/skills/packs/unreal_editor_control/prompt.md`

- 注入条件：`inject_to=[DevAgent, TestAgent]`，`traits_match: any_of=[engine:ue5, engine:ue4]`
- 内容：操作说明 + 对象路径约定 + 安全规则
- 注册：`skills/skills.json` 新增 `unreal-editor-control` 条目

### Phase 4 — 模板实例化集成

**`propose_ue_framework.py`**：
- 返回 `install_ucp_available`（检测 `backend/ue_plugins/` 快照是否存在）

**`instantiate_ue_template.py`**：
- 新增 `install_ucp: bool` 参数
- 若为 True：从快照 copy 插件到 `{project}/Plugins/UnrealClientProtocol/`
- 返回 `ucp_installed: bool`

**`frontend/app.js`**：
- `propose_ue_framework` 卡片加复选框「🤖 启用编辑态 AI 控制（UCP 插件）」
- `doInstantiateUEFramework()` 传递 `install_ucp` 给 API

### Phase 5 — Reflexion 增强

**`orchestrator.py`（`_build_context`）**：
- UE 项目构建 context 时探测 UCP（1s timeout）
- 注入 `context["ucp_available"] = True/False`

**`agents/dev.py`（`fix_issues`）**：
- `play_test_failed` 分支：若 `ucp_available`，调 `get_actors` 取关卡 Actor 列表
- 注入 `context["editor_state"] = {"actors": [...], "actor_count": N}`
- Reflexion 时 AI 能看到 Editor 现状，不再盲猜
