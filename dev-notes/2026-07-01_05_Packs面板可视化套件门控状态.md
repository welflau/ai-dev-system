# Packs 面板可视化套件门控状态

**日期**：2026-07-01
**状态**：已上线 ✅
**关联**：`dev-notes/2026-07-01_03_内置Rules_Skills套件化门控.md`

## 背景

套件化门控逻辑完成后，用户在 Packs 面板看不到「这个套件究竟激活了哪些内置 rules/skills」，体验不透明——安装了 `ue5-dev`，但不知道它是否真的在往 prompt 里注入 `ue5.md`。

## 改动

### 后端（`backend/api/projects.py`）

`get_project_packs` 接口在每个已安装 pack 记录里追加两个字段：

```json
{
  "pack_name": "game-dev",
  "enabled_rules": ["game-dev"],
  "enabled_skills": []
}
```

通过遍历 `skill_loader.rules` / `skill_loader.skills`，按 `pack` 字段归组，O(N) 一次扫描。

### 前端（`frontend/app.js`）

已安装 pack 行，当 `enabled_rules` 或 `enabled_skills` 非空时，在名称旁显示：

```
📦 通用游戏开发  🎓 已激活  [agents] [commands] ...
```

悬停 `🎓 已激活` 徽章可以看到具体内容：`rules: game-dev | skills: 0个`。

### 命令（`backend/api/commands.py`）

`/skills` 命令输出里，每条 rule/skill 后追加 pack 标注：

```
• global [always] — 全项目通用规范
• game-dev [traits] `pack:game-dev` — 游戏开发通用规范
```

用户在对话框输 `/skills` 就能看到哪些规则是「常驻核心」哪些是「套件控制」。

## 效果

OG 项目已装 `game-dev`，Packs 面板显示：
- `game-dev` → `🎓 已激活`（hover: `rules: game-dev`）
- 装了 `ue5-dev` 后 → `🎓 已激活`（hover: `rules: ue5 | skills: 3个`）

## 改动文件

- `backend/api/projects.py` — get_project_packs 追加 enabled_rules/enabled_skills
- `backend/api/commands.py` — /skills 输出加 pack 标注
- `frontend/app.js` — 已安装 pack 行加「🎓 已激活」徽章
