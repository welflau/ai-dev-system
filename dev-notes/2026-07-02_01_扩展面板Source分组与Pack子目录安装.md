# 扩展面板 Source 分组优化 + Pack 子目录安装

**日期**：2026-07-02
**状态**：已上线 ✅
**关联**：`dev-notes/2026-07-01_06_OpenSpec面板实现.md`

## 背景

Agent/扩展 页的 Skills / Commands / Rules 面板存在以下问题：

1. `.codebuddy/commands/` 及子目录不被扫描（用户 `.codebuddy/commands/opsx/` 里的命令全部消失）
2. OpenSpec 提供的 skills（`openspec-*`）和 commands（`opsx/`）被归入「项目」或「用户」组，没有独立分组
3. 来自 `.claude/.codebuddy/.ads` 的项目级命令被错误归类为「用户 Command」
4. Pack 的 rules/commands/skills 详情页显示库路径（`backend/config_packs/...`）而非项目实际路径
5. Pack 安装的文件平铺到 `.codebuddy/rules/` 等顶层目录，与用户自己的文件混在一起

## 改动

### 1. Commands 扫描支持子目录 + .codebuddy

`backend/api/commands.py` `_load_project_commands`：
- 新增 `.codebuddy/commands/` 扫描路径
- `glob("*.md")` → `rglob("*.md")`，支持子目录
- name 使用相对路径（`opsx/deploy`）

`backend/api/projects.py` `get_project_commands_all` 文件映射同步。

### 2. OpenSpec 独立 Source 分组

**后端**：
- `commands.py`：子目录名含 `openspec` 或 `opsx` → `source: "openspec"`
- `projects.py`：skills stem 以 `openspec` 开头 → `source: "openspec"`；response 增加 `openspec` 字段

**前端**（`frontend/app.js`）：
- `_SOURCE_META` 新增 `openspec`（紫色 `#a855f7`）
- `_injectSourceFilter` 显示「OpenSpec N」过滤按钮
- `_renderSourceGroups` 循环包含 `openspec`
- Skills / Commands 分组标题加 `📐 OpenSpec` 条目

### 3. Commands source 分类修正

`projects.py` `get_project_commands_all`：
- `source=claude/codebuddy/ads` → `project`（不再归为 `user`）
- response 增加 `project` 字段
- 前端 Commands 分组补 `📁 项目 Command`

### 4. Pack 安装到专属子目录

`backend/pack_installer.py`：
- rules / commands / skills / agents / scripts 安装到 `.{target}/packs/{pack_name}/` 子目录
- `dst = dst_root / "packs" / pack_name / rel`
- `rules.md`（追加 CLAUDE.md）、`mcps/`（merge settings.json）、`hooks/` 操作不受影响

`backend/api/projects.py`：
- Rules 扫描：优先查项目 `.claude/.codebuddy/packs/{pn}/rules/`，无副本回退到库路径
- Skills 扫描：直接扫 `packs/{pn}/skills/`，旧格式兼容
- Commands 扫描：直接扫 `packs/{pn}/commands/`，旧格式兼容

`frontend/app.js`：
- 已安装列表顶部迁移提示（橙色横幅，建议 ↩ 重装）
- 每行加路径说明 `.codebuddy/packs/{pack_name}/`

## 效果

| 问题 | 之前 | 之后 |
|------|------|------|
| .codebuddy/commands/opsx/ | 不显示 | OpenSpec 分组，带紫色徽章 |
| openspec-* skills | 「项目」组 | 独立「📐 OpenSpec」组 |
| .claude/commands/*.md | 归「用户」 | 归「📁 项目 Command」 |
| Pack rules 文件路径 | 显示库路径 | 显示项目副本路径（重装后） |
| Pack 文件位置 | 与用户文件混放 | `.codebuddy/packs/{name}/` 独立目录 |

## 改动文件

- `backend/api/commands.py` — rglob + .codebuddy + openspec source
- `backend/api/projects.py` — commands/rules/skills source 分类 + packs/ 扫描
- `backend/pack_installer.py` — 安装目标路径加 packs/{name}/ 前缀
- `frontend/app.js` — openspec _SOURCE_META + 过滤器 + 分组标题 + 迁移提示
