## 四、关键技术决策

### 4.1 模板实例化走 Epic 官方 TemplateDefs.ini，不自研

Epic 的 `<Engine>/Templates/TP_*/Config/TemplateDefs.ini` 已定义完整替换规则：
- `FolderRenames`: `Source/%TEMPLATENAME%` → `Source/%PROJECTNAME%`
- `FilenameReplacements`: 文件名里 3 种 token（`%TEMPLATENAME%` / `_UPPERCASE` / `_LOWERCASE`）
- `ReplacementsInFiles`: 文件内容替换（cpp/h/ini/cs）
- `FilesToIgnore` / `FoldersToIgnore`: 跳过 Binaries/Intermediate/TemplateDefs.ini 本身

我们的 Action 完整消费这些规则 → 产出结果跟 Epic Launcher 新建项目**字节级一致**，保证官方语义。

### 4.2 编译失败语义拆两层

- `status="error"` → Action **早期问题**（缺 engine_path / uproject / UBT）→ 走 BLOCKED 人工介入
- `status="compile_failed"` → UBT **真跑但编译失败**（errors[] 有内容）→ 走 reject_goto 自动修复

这层区分避免了"UBT 没跑也走修复循环"和"真需要修却卡 BLOCKED"两种错误。

### 4.3 reject_goto 通用派生（一箭多雕）

之前 SOP yaml 的 `reject_status` / `reject_goto` 声明了但后端不消费。我扩展 `sop_to_transition_rules` 自动派生：

```
任何 stage 有 reject_status + reject_goto
  → 规则加一条: reject_status → {goto_stage.agent}.fix_issues
```

一次性打通 4 条回跳链（engine_compile / play_test / multiplayer_stress / i18n_check），**以后加新 fragment 只要声明字段就自动工作**。

### 4.4 UE Skill Pack 用**按路径精准注入**

```json
"unreal-cpp-dev": {
  "inject_to": ["DevAgent"],
  "traits_match": {"any_of": ["engine:ue5", "engine:ue4"]},
  "paths": ["Source/**/*.h", "Source/**/*.cpp", "**/*.Build.cs"]
}
```

DevAgent 在 **UE 项目** 且 **正在改 UE 源码**时才吃这个 pack —— 写 Python 工具脚本不会被 UE 规范污染。

### 4.5 全流程 log 推到已有"操作日志"面板而非新造 UI

复用 `event_manager.publish_to_project` + 前端 `appendLogEntry`，新事件类型直接走现有渲染管线 + 按关键字染色。没新造"UBT 日志面板""模板进度面板"等专用组件，减少 UI 维护负担。
