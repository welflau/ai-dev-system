# 开发日志 — 2026-04-30 项目删除修复与批量暂停工单

## 一、项目删除 FOREIGN KEY 约束失败

### 问题

删除运行中的项目时报 `级联删除失败: FOREIGN KEY constraint failed`，项目无法删除。

### 根因分析

两个独立原因叠加：

**1. 缺表**：`delete_project` 的级联删除列表遗漏了后续新增的表：

| 表 | FK 约束 |
|---|---|
| `bugs` | `project_id NOT NULL REFERENCES projects(id)` |
| `failure_cases` | `project_id` |
| `knowledge_index` | `project_id` |

**2. 竞态写入**：项目有 95 个工单正在被 Orchestrator 处理，删子表期间 Orchestrator 仍在向 `ticket_logs`/`artifacts` 插入新行，轮到 `DELETE FROM projects` 时又有新 child 行存在，FK 报错。

另外排查过程发现：`PRAGMA foreign_keys=OFF` 在 SQLite 事务内是 **no-op**，Python sqlite3 模块会在第一条 DML 之前自动开启隐式事务，导致 PRAGMA 无效。修复方式是先 `commit()` 关闭活跃事务再设置 PRAGMA。

### 修复（`backend/api/projects.py`）

```python
# 1. 先通知 Orchestrator 停止该项目所有工单（清理 _processing / _project_active）
try:
    from orchestrator import orchestrator
    orchestrator._project_active.pop(project_id, None)
    # 清理进行中工单...
except Exception as oe:
    logger.warning(...)

# 2. commit() 关闭活跃事务 → PRAGMA 才能生效
await db._db.commit()
await db._db.execute("PRAGMA foreign_keys=OFF")
await db._db.commit()

# 3. 级联删除（补全缺失的 3 张表）
delete_sqls = [
    ...,
    "DELETE FROM bugs WHERE project_id = ?",
    "DELETE FROM failure_cases WHERE project_id = ?",
    "DELETE FROM knowledge_index WHERE project_id = ?",
    ...,
    "DELETE FROM projects WHERE id = ?",
]

# 4. finally 恢复 FK 检查
await db._db.execute("PRAGMA foreign_keys=ON")
```

## 二、AI 助手批量暂停工单

### 背景

用户批量创建 17-95 条需求后，所有工单涌入 Orchestrator 并发执行，需要一个快捷的方式全部暂停、按优先级逐步放行。

### 实现

新增 `PauseRequirementsBatchAction`（`backend/actions/chat/pause_requirement.py`）：

**工具名**：`pause_requirements_batch`

**入参**：
- `requirement_ids: List[str]` — 指定具体需求 ID 列表
- `filter: "all" | "in_progress" | "decomposed"` — 按状态批量暂停
- `reason: str` — 暂停原因（可选）

**执行逻辑**：
1. 批量将需求状态更新为 `paused`
2. 同步清理 Orchestrator 的 `_processing` / `_project_active`（已在跑的工单立即停止调度）
3. 写入 `ticket_logs` 记录
4. 发布 `requirement_status_changed` SSE 事件

**触发方式**（项目内 AI 助手）：
- "暂停所有工单" / "先全部停下" → `filter="all"`
- "暂停进行中的需求" → `filter="in_progress"`
- "暂停 REQ-xxx 和 REQ-yyy" → `requirement_ids=[...]`
