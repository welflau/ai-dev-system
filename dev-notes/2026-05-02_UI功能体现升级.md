# 开发日志 — 2026-05-02 UI 功能体现升级

## 背景

Phase 0-4 新增了大量后端能力（新 Agent、资产库、知识库、Smart Probe、对抗性审查），但前端完全无感知：新阶段状态显示英文原始值、看板没有设计列、资产库无任何入口。本轮专门补全 UI 层的体现。

---

## 改动一览（commit `9a444b5`）

### 1. 工单看板新增「设计中」列

**before**：看板只有「待启动 / 架构中 / 开发中 / 测试中 / 测试通过 / 已部署」6列，策划/UX/美术阶段工单不可见（藏在「待启动」里或没有列归属）

**after**：新增「🎨 设计中」列，覆盖全部设计阶段状态：

```
待启动 | 🎨 设计中 | 架构中 | 开发中 | 测试中 | 测试通过 | 已部署
         ↑
  planning_in/done
  ux_design_in/done
  art_design_in/done
```

**改动**：
- `frontend/index.html`：看板 HTML 插入 `col-design` 新列
- `frontend/app.js`：`columns` 数组加入 `'design'`
- `backend/models.py`：`BOARD_COLUMNS['design']` 定义设计阶段状态集合

---

### 2. 状态标签全面升级

**before**：6 个新状态（planning_in_progress 等）前端显示英文原始值，不可读

| 状态值 | 显示 |

| `planning_in_progress` | 📋 策划中 |

| `art_design_in_progress` | 🖼️ 美术设计中 |

**改动**：
- `frontend/app.js`：`STATUS_LABELS` + `getStatusColor()` 补全新状态
- `backend/models.py`：`STATUS_LABELS` 同步补全

---

### 3. 工单状态下拉补全

工单卡片上的状态下拉选择器新增 7 个选项（带 emoji），用户可手动切换到任意新阶段状态。

---

### 4. 产物图标更新

`getArtifactIcon()` 新增：
- `prd` → 📋
- `ux_design` → 🎨  
- `art_design` → 🖼️

之前这些产物类型显示默认图标📦。

---

### 5. 工单抽屉 — Smart Probe 评分条

PlannerAgent 生成 PRD 时会记录 Smart Probe 评分（5维度/25分满分），现在工单抽屉顶部直接展示：

```
📊 Smart Probe 需求清晰 ✅   清晰度 22/25
```

颜色编码：
- 🟢 ≥ 20：绿色「需求清晰 ✅」
- 🟡 15-19：橙色「已补假设 🟡」
- 🔴 < 15：红色「信息不足 🔴」

从 `artifacts[type=prd].content` 解析 `smart_probe_score` 字段。

---

### 6. 工单抽屉 — 代码审查摘要

代码审查报告（`artifacts[type=code_review]`）现在在工单抽屉内直接预览严重问题和警告，无需打开报告文件：

```
🔍 代码审查摘要
🔴 XSS 漏洞：用户输入未转义直接插入 innerHTML
🟡 缺少空值检查：data.items 可能为 null
```

只展示 critical_issues（全部）+ warnings（前2条），不展示建议，保持简洁。

---

### 7. 资产库浏览页面（全新）

**入口**：项目设置 → 侧栏「🖼️ 资产库」

**功能**：
- 关键词搜索（名称/标签/描述）
- 类型筛选（图标/插画/图片/精灵图/3D模型/动画/音频）
- 来源筛选（Iconify/Kenney/Pexels/AI生成）
- 瀑布流网格展示（每格显示图标预览/名称/来源）
- 点击格子复制文件路径（`used_count +1`）
- 统计栏显示：总数 33,263 条、各类型数量
- 分页加载（每次 40 条）

**技术**：调用 `GET /api/art-assets?q=&type=&source=&limit=40&offset=N`
