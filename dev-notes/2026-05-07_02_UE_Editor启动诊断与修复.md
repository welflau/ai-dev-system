# 开发日志 — 2026-05-07 UE Editor 启动诊断与自动修复

## 背景

v0.20 UE MCP 添加了「启动 Editor」按钮后，用户点击启动时 Editor 频繁报错退出。
本次排查了根本原因，并添加了自动崩溃诊断能力。

---

## 问题排查过程

### 错误现象

```
The game module 'TestTPS' could not be loaded.
There may be an operating system error, the module may not be properly
set up, or a plugin which has been included into the build has not been turned on.
```

### 排查路径

| 怀疑 | 验证结果 |
|---|---|
| UCP 插件依赖问题 | 移除 Plugins/ 后仍报同样错误，排除 |
| DLL 旧版本缓存 | 删除 Binaries/ 重编后依然报错，排除 |
| Config 里有遗留引用 | grep 检查无 UCP 引用，排除 |
| Intermediate 缓存 | 无残留配置，排除 |

### 根本原因

**`_write_uproject` 生成的 `.uproject` 缺少必要的 Plugins 声明。**

TP_ThirdPerson 模板的 `Build.cs` 依赖了：
```csharp
"StateTreeModule",      // → StateTree 插件
"GameplayStateTreeModule",  // → GameplayStateTree 插件
"EnhancedInput",        // → EnhancedInput 插件
```

这些是 UE5 引擎插件，需要在 `.uproject` 的 `Plugins` 段显式启用。
我们生成的 `.uproject` 只有 `Modules` 段，没有 `Plugins` 段，导致 Editor 加载 DLL 时发现依赖的插件"未开启"。

### 修复

**`backend/actions/instantiate_ue_template.py`** — `_write_uproject()` 加入默认插件声明：

```python
"Plugins": [
    {"Name": "EnhancedInput",    "Enabled": True},
    {"Name": "StateTree",        "Enabled": True},
    {"Name": "GameplayStateTree","Enabled": True},
]
```

同时手动修复了 `F:\Projects\TestTPS\TestTPS.uproject`（删旧 DLL 重编后 Editor 正常启动）。

## 新增能力：Editor 启动崩溃自动诊断

### 问题

Editor 崩溃时系统只显示"超时未检测到 UCP 连接"，用户需要截图发给 AI 助手才能知道原因。

### 解决方案

`_poll_ucp_ready()` 增加：
1. **进程监控**：检测进程是否提前退出（`proc.poll() is not None`）
2. **自动读 UE log**：读取 `Saved/Logs/{project}.log` 最后 200 行
3. **错误模式匹配**：识别已知错误类型，给出友好提示

### 已知错误模式

| 错误关键词 | 系统提示 |
|---|---|
| `module could not be loaded` | 游戏模块加载失败——请先点「编译」重新构建 |
| `plugin has not been turned on` | .uproject 缺少插件声明——下次生成自动修复 |
| `failed to compile` | 编译失败——请查编译日志 |
| `out of memory` | 内存不足——关闭其他程序后重试 |
| `access denied` | 权限不足——以管理员身份运行 |

### 效果

点「▶ 启动 Editor」崩溃后，实时日志面板自动显示：

```
❌ [editor] Editor 进程已退出（exit=1），正在分析日志…
[editor] 捕获到 3 条错误：
  ▷ The game module 'TestTPS' could not be loaded...
  ▷ LogCore: Engine exit requested
💡 [editor] 诊断：游戏模块加载失败——请先点「编译」重新构建，
   或检查 .uproject 的 Plugins 声明是否完整
```

不需要截图发给 AI 助手。

## 其他修复

### 基线编译走 CI 策略

「跑基线编译」改为委托 `CI策略.trigger_build("ubt_compile")`，
结果出现在「最近构建」，可点详情查完整日志。

### 构建完整日志持久化

- `ci_builds` 加 `log_file_path` 列
- 构建完成写 `data/build_logs/{id}.log`
- 详情弹窗加「📄 完整日志」链接

### Editor 进程卡片双按钮

根据 `dll_exists` 状态显示：
- 无 DLL：`[🔨 编译]`（可点）+ `[▶ 启动 Editor]`（禁用）
- 有 DLL：`[🔨 编译]`（可点）+ `[▶ 启动 Editor]`（可点）
- UCP 已连：`● 已连接（UCP 9876）`

### main.py 启动清除 __pycache__

根治旧字节码缓存导致代码更改不生效的问题。
