### Phase 2.5: v0.14.5-7 — MetaGPT 模式移植

参考源码：`D:\A_Works\MetaGPT`

#### v0.14.5: ActionNode 结构化输出
- 新增 `backend/actions/action_node.py`（移植自 MetaGPT `action_node.py`）
- 核心流程：`fill(req, llm)` → 编译 prompt（加 Schema 约束）→ 调 LLM → Pydantic 验证
- `_compile()`: 自动从 Pydantic Schema 提取字段类型，生成 JSON 格式约束
- `_lenient_parse()`: 宽松解析，缺字段用默认值，不崩溃
- 新增 `backend/actions/schemas.py`: 5 个输出 Schema
  - `ArchitectureOutput` — 架构设计
  - `DevOutput` — 代码开发（files + notes）
  - `ReviewOutput` — 验收评审
  - `TestReviewOutput` — 代码审查
  - `DecomposeOutput` — 需求拆单
- DesignArchitectureAction / WriteCodeAction / AcceptanceReviewAction 全部改用 ActionNode

**对比改造前后**：
```python
# 改造前（fragile）
result = await llm_client.chat_json([...])
if result and isinstance(result, dict) and result.get("files"):
    files = result["files"]  # 可能不是 dict

# 改造后（type-safe）
node = ActionNode(key="write_code", expected_type=DevOutput, instruction="...")
await node.fill(req=context, llm=llm_client)
output = node.instruct_content  # DevOutput 类型，保证有 files 字段
```

#### v0.14.6: Role 内部状态机
- `BaseAgent` 新增 `ReactMode` 枚举（移植自 MetaGPT `RoleReactMode`）
  - `SINGLE`: 单步执行（默认，向后兼容）
  - `BY_ORDER`: 按顺序执行所有 Action，前一步输出注入后一步
  - `REACT`: LLM 动态选择下一步（预留接口）
- `_react_by_order()`: 遍历 action_classes，依次执行，合并结果
- `_react_with_think()`: 预留，当前等同 BY_ORDER
- `_execute_legacy()`: 子类没用 Action 时的 fallback
- 默认 SINGLE 模式完全兼容现有 Agent

#### v0.14.7: Watch 订阅 + cause_by 索引
- `MemoryItem` 新增 `cause_by` 字段（移植自 MetaGPT `Message.cause_by`）
- `AgentMemory` 新增 `_index: Dict[str, List[MemoryItem]]`
  - `add(item)`: 添加并建立 cause_by 索引
  - `get_by_action(action)`: 按 Action 精确检索
  - `get_by_actions(actions)`: 按多个 Action 检索
  - `load_index_from_db()`: 从 artifacts + ticket_logs 加载历史索引
- `BaseAgent` 新增：
  - `watch_actions: Set[str]`: 声明关心的 Action 类型
  - `is_watch(cause_by)`: 检查是否关心某事件
  - `important_memory`: 只获取 watch 范围内的记忆
