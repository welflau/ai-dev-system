## 三、UE 自测 Layer 2（commit 8584218）

### 3.1 SOP loader stage_overrides 机制

**问题**：fragments 只能插入新 stage，无法 patch 已有 stage 的 config 字段。`_core.yaml` 注释写着「UE 项目深度迭代时建议打开，通过 SOP fragment 覆盖即可」，但机制不存在。

**修复**：`backend/sop/loader.py` `compose_sop()` 第 7 步，处理 fragment 的 `stage_overrides` 列表：

```python
for frag in frag_entries:
    for ov in (frag.get("stage_overrides") or []):
        composed[idx].setdefault("config", {}).update(ov_cfg)
```

### 3.2 engine_compile.yaml 启用 ue_precompile

`backend/sop/fragments/engine_compile.yaml` 新增：

```yaml
stage_overrides:
  - id: development
    config:
      ue_precompile: true
      ue_precompile_timeout: 120
```

UE 项目（`engine:ue5/ue4` trait）的 `development` stage config 自动获得 `ue_precompile=True`，Layer 2 正式生效。

### 3.4 完整 UE 自测三层架构

```
DevAgent.develop → SelfTestAction._run_ue_self_test()

  │            blocking > 0 → fail     → fix_issues（不等 UBT）

               fail → fix_issues（省掉完整 3-5 min UBT）
               pass ↓
  Layer 3  engine_compile stage        3-5 min，全量 UBT（最终兜底）
```
