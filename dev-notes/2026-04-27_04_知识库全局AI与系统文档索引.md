## 一、全局 AI 助手支持跨项目知识库搜索

### 问题

`_GLOBAL_CHAT_TOOLS` 白名单只有 `confirm_project`，`search_knowledge` 和
`search_ticket_history` 未加入，全局聊天完全无法调用知识库工具。

另外两个 Action 的查询逻辑在 `project_id=None` 时行为错误：
- `search_knowledge`：SQL `ki.project_id = NULL` 在 SQLite 中永远 false，实际只搜全局文档
- `search_ticket_history`：`tf.project_id = ?` 传 None，完全搜不到任何工单

### 修复

**`backend/agents/chat_assistant.py`**：
```python
_GLOBAL_CHAT_TOOLS = {"confirm_project", "search_knowledge", "search_ticket_history"}
```

**`backend/actions/chat/search_knowledge.py`**：
```python
if project_id:
    # 项目内：该项目文档 + 全局文档
    ... AND (ki.project_id = ? OR ki.project_id IS NULL) ...
else:
    # 全局模式：搜全部（不过滤）
    ... WHERE knowledge_fts MATCH ? ...
```

**`backend/actions/chat/search_ticket_history.py`**：同样逻辑分支，无 project_id 时搜全部项目。

**`backend/api/knowledge.py`**（HTTP 端点）：同步更新 `_do_search_knowledge`，API 级别也生效。

### 同期修复：FTS5 rebuild 后同连接搜不到数据

**症状**：HTTP API 返回 0 结果，但用新的 `sqlite3.connect` 直连 DB 能搜到。

**根因**：`executescript(SCHEMA_SQL)` 创建 FTS5 虚拟表后，同一 aiosqlite 连接的 schema 缓存未刷新，
导致 `rebuild` 后对同一连接的查询看不到新索引。

**修复**：rebuild 前先执行 `integrity-check` 强制刷新缓存：
```python
await db.execute("INSERT INTO knowledge_fts(knowledge_fts) VALUES('integrity-check')")
await db.execute("INSERT INTO knowledge_fts(knowledge_fts) VALUES('rebuild')")
```

## 二、系统自身 docs/ 和 dev-notes/ 纳入全局知识库

### 需求

`ai-dev-system/docs/`（33 个技术方案文档）和 `ai-dev-system/dev-notes/`（32 个开发日志）
应该被全局 AI 助手检索到，同时在知识库配置界面可见。

### 实现

**`backend/main.py`**：startup 时扫描系统目录，以 `sys_docs__` / `sys_devnotes__` 前缀写入
全局 `knowledge_index`（`project_id=NULL`）：

```python
SYSTEM_ROOT = _BDIR.parent  # F:\A_Works\ai-dev-system
for sys_dir, prefix in [
    (SYSTEM_ROOT / "docs",      "sys_docs__"),
    (SYSTEM_ROOT / "dev-notes", "sys_devnotes__"),
]:
    for md in sorted(sys_dir.glob("*.md")):
        fname = prefix + md.name
        await _upsert_knowledge_index(None, fname, content)
```

**修复重复索引问题**：

SQLite 的 `UNIQUE(project_id, filename)` 对 `project_id=NULL` 不生效（NULL ≠ NULL），
每次 startup 都会重复插入全局文档。

修复 `_upsert_knowledge_index`：`project_id=None` 时改用 DELETE + INSERT 保证幂等：
```python
if project_id is None:
    await db.execute("DELETE FROM knowledge_index WHERE project_id IS NULL AND filename = ?", ...)
    await db.execute("INSERT INTO knowledge_index(project_id, ...) VALUES (NULL, ...)", ...)
else:
    await db.execute("INSERT ... ON CONFLICT DO UPDATE ...", ...)
```

**新增 API 端点**：`GET /api/knowledge/system`
- 从 `knowledge_index` 读取 `sys_docs__` 和 `sys_devnotes__` 前缀的文档
- 按 group 分类返回（docs / devnotes），含 display_name（去掉前缀）和 size

**前端知识库 Tab**（`frontend/index.html` + `app.js` + `styles.css`）：

在「全局知识库」卡片前新增「🗂 系统知识库」卡片：
- 只读，无编辑/删除按钮
- 分两组展示：技术方案文档（33）/ 开发日志（32）
- 每行有「预览」按钮（通过 FTS5 搜索返回前 400 字）
- `loadSettingsKnowledge()` 并行加载系统文档 + 全局文档 + 扫描路径（若有项目）
