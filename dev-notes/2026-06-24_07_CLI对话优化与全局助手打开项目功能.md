# 2026-06-24 CLI 对话优化 & 全局助手打开项目功能

## 背景

本次迭代解决 CLI 模式下的几个关键问题，并为全局 AI 助手新增「打开已有项目」能力。

---

## 1. CLI 模式系统提示前缀泄漏修复

**问题**：`_messages_to_prompt` 把 system prompt 和用户消息拼成一段文字通过 stdin 传给 CLI，
导致每次 AI 回复前都输出几千字的系统提示作为前缀。

**修复**（`backend/llm_client.py`）：
- `_messages_to_prompt` 改为只返回最后一条 user 消息，不含 system
- 新增 `_extract_system_prompt` 方法单独提取 system 文本
- `_call_cli_stream`：首轮（无 resume）通过 `--system-prompt` 参数传递；`--resume` 时跳过（CLI 会话已记录）
- `_call_cli`（非流式）：同样通过 `--system-prompt` 参数传递
- `_messages_to_stdin` 有图片路径：移除 system 内容拼入 JSON block 的逻辑
- 日志改为完整显示命令，`--system-prompt` 内容替换为 `<system-prompt>` 占位符

---

## 2. 全局助手新增 `open_project` 工具

**问题**：用户说"打开 xxx 项目"，AI 不知道应触发前端跳转，而是给出 CLI 操作指引。

**方案一：预检测拦截（主要路径）**（`backend/api/chat.py`）：
- 新增 `_detect_open_project` 函数，正则匹配「打开/进入/切换到/open/switch to + 项目名」
- 命中后直接返回 `open_project` action，**跳过 LLM 调用**，响应约 0ms
- 正则：`(?:打开|进入|切换到|切换至|跳转到|open|switch\s+to)\s*[「【\[]?(.+?)[」】\]]?\s*(?:项目)?$`
- 支持精确匹配 + 模糊匹配（子串包含）

**方案二：LLM 工具（兜底）**（`backend/actions/chat/open_project.py`）：
- 新增 `OpenProjectAction`，接受 `project_id` 或 `project_name`，查 DB 模糊匹配后返回 action
- 注册为 `_GLOBAL_ONLY_TOOLS`，全局聊天时暴露给 LLM

**前端处理**（`frontend/app.js`）：
- 流式回调收到 `open_project` / `switch_project` 直接调 `showProjectDetail()`
- `appendChatBubble` 渲染"打开项目"小卡片

## 3. 进入项目后标题未更新 & 聊天不加载

**问题**：
1. `showProjectDetail` 调用时 `currentProject` 还未加载，标题显示为空
2. `loadChatHistory` 被 `chatMode === 'global'` 条件拦住，进入项目后聊天区空白

**修复**（`frontend/app.js`）：
- `loadProjectDetail` 拿到数据后立即更新 `chatPanelTitle`
- `_updateChatPanelForContext` 中去掉 `chatMode === 'global'` 限制，只要有 `currentProjectId` 就加载历史
