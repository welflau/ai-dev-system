## 一、Phase F —— MCP trait 过滤

### 1.1 问题

现有 MCP 架构：`mcp_servers.json` 里声明 enabled=true 的 server 启动时加载，ChatAssistantAgent 在项目内聊天时把**所有 running server 的工具**全部 union 成 tool 列表给 LLM。

问题：某些 MCP server 只对**特定项目类型**有意义。例：
- `git` MCP 只对 `vcs:git` 项目有用，UE5/P4 项目加载它只是污染 LLM 工具选项
- `godot-mcp`（未来可能接入）只对 `engine:godot4` 项目有用
- `filesystem` 是通用的，所有项目都能用

"通用 vs 特定"的边界应该显式声明，而不是靠 server 启动/关闭这个粗粒度开关。

### 1.2 改造：`enabled_for_traits` 字段

**schema 扩展**（`mcp_servers.json`）：

```json
{
  "git": {
    "type": "stdio",
    "command": "uvx",
    "args": ["mcp-server-git"],
    "enabled": false,
    "description": "...",
    "enabled_for_traits": {           // ← 新增
      "any_of": ["vcs:git"]
    }
  }
}
```

语义跟之前 Skill / SOP / Agent 的 `traits_match` / `available_for_traits` **完全一致**：`all_of` / `any_of` / `none_of` 三元组，空/缺省 = 对所有项目可用。

### 1.3 核心过滤点

**mcp_client.py**：

```python
def list_all_tool_schemas(self, traits: Optional[List[str]] = None) -> List[Dict]:
    schemas = []
    for name, conn in self._servers.items():
        if conn.status != "running":
            continue
        cfg = self._servers_config.get(name, {})
        trait_filter = cfg.get("enabled_for_traits")
        if trait_filter and traits is not None:
            if not _match_trait_filter(trait_filter, traits):
                continue
        schemas.extend(conn.tools)
    return schemas
```

`_match_trait_filter()` 是本地实现的三元组匹配器，跟 `skills/loader.py::_match_traits` / `actions/base.py::_match_traits` 语义一致但不复用 import（避免循环依赖）。

**ChatAssistantAgent.chat()**：

```python
# 从 project 解析 traits
project_traits = []
try:
    traits_raw = project.get("traits") or "[]"
    project_traits = json.loads(traits_raw) if isinstance(traits_raw, str) else list(traits_raw)
except Exception:
    pass

# 传给 _exposed_tool_schemas
tools = self._exposed_tool_schemas(scope="project", traits=project_traits)
```

`_exposed_tool_schemas(scope, traits)` 把 traits 透传给 `mcp_client.list_all_tool_schemas(traits=...)`。**全局聊天（scope="global"）不传 MCP**，维持原行为。

### 1.4 E2E 证实

**web + git 项目**：
```
preview → mcps: [filesystem(T), fetch(T), git(T)]   ← 都 applicable
```

**UE5 + P4 项目**：
```
preview → mcps: [filesystem(T), fetch(T), git(F)]   ← git 被 vcs:git any_of 过滤掉
```

`get_status()` 正确暴露 `enabled_for_traits: {any_of: [vcs:git]}` 给前端展示。

## 三、线上生效情况

### 3.1 已有项目影响（零回归）

所有 6 个活跃项目 traits 当前都是 `[]`：
- Tetris-Game2、Tetris-Game、claude-code、HelloWorld 等 → 继续走 default SOP 7 条规则
- **TestFPS** 项目除外（图 #8 里用户实际测试的 UE5 项目）：traits 包含 `engine:ue5, category:game, genre:fps, multiplayer` 等，被 compose 为 7 阶段含 `engine_compile` + `play_test` 的规则

### 3.2 新建项目链路

对话式："做个 UE5 FPS 多人游戏"  
→ LLM 识别意图 + Preset 匹配推荐 `ue5-game`  
→ 调 `confirm_project(traits=[...])`  
→ 前端卡片**实时展示**将组装成什么  
→ 用户确认 → `CreateProjectAction` 写入 DB（含 traits）

### 3.3 导入项目链路

本地文件夹 → `detect-local` API  
→ `ProjectTypeDetectorAction` 扫 24 条规则  
→ 返回 candidates + evidence + suggested_preset  
→ 前端 detectInfo 卡片渲染带证据的 traits  
→ 用户点"导入" → `create_project(traits=...)` 入库

### 3.4 调度链路

orchestrator `_poll_once` → `_get_all_actionable_statuses()` → 按项目 traits compose 的 SOP 规则 union  
→ 拿 ticket → `_get_rules_for_project(project_id)` → lookup rule → dispatch Agent.action  
→ 执行时 `_resolve_skills_prompt(context)` 按项目 traits 算 skills prompt → 注入

### 3.5 编辑链路

项目 → 设置 → 🏷 项目特征  
→ 增删 trait / 换 preset  
→ 实时调 `/preview-assembly` 展示变化  
→ 保存 → `PATCH /traits` 写库 + `orchestrator.invalidate_project_rules`  
→ 下一次工单用新规则

## 五、对其他子系统的影响

v0.17 让以下子系统都**必须考虑 trait 过滤层**，未来改动不能漏：

| 子系统 | 改动时要考虑 |
|---|---|
| **新增 Skill** | 必须填 `traits_match`，至少 `any_of: [platform:web]` 级别的限定；否则会"漏"到所有项目 |
| **新增 Action** | 如果是引擎特定（如 `UECompileCheckAction`），声明 `available_for_traits` |
| **新增 Agent** | 如果是领域特定（如 `ArtistAgent`），声明 `available_for_traits` |
| **新增 MCP** | 填 `enabled_for_traits`，或显式说明"通用" |
| **新增 SOP 阶段** | 以 fragment 形式实现，带 `required_traits` / `required_ticket_type` |
| **改 trait_taxonomy** | 小心回溯影响（已有项目的 trait 可能在新 taxonomy 下变成 unknowns） |

建议：
- 在 `docs/` 里加一份 **v0.17 架构速查表**，给未来 contributor 看"trait-first 是什么 + 怎么新增 XX 要遵守什么"
- 所有新 skill / action / agent 加入 PR 时 checklist 加一条"是否声明了 trait 限定"
