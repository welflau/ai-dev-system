# 聊天回答工具栏常显 & 保存按钮拆分

> 日期：2026-05-09
> 类型：Bug修复 + 功能增强

---

## 问题

AI 回答底部的复制/保存按钮默认不可见，只有鼠标悬停才出现（`opacity: 0` + hover 规则）。
且保存只有「保存到仓库」一个选项，缺少本地文件下载。

---

## 改动

### 1. `frontend/styles.css`

删除 `opacity: 0` 和 `.chat-msg:hover .chat-bubble-toolbar` 规则，改为始终显示：

```css
/* 改前 */
.chat-bubble-toolbar { opacity: 0; transition: opacity 0.15s; }
.chat-msg:hover .chat-bubble-toolbar { opacity: 1; }

/* 改后 */
.chat-bubble-toolbar { opacity: 1; }
```

### 2. `frontend/app.js` — 工具栏 HTML（L9781）

新增「保存为文件」按钮，保存按钮拆为两个：

| 按钮 | 图标 | 行为 |
|------|------|------|
| 复制 | 复制图标 | 复制气泡文本到剪贴板 |
| 保存为文件 | 下载箭头 | 浏览器下载本地 `.md` 文件 |
| 保存到仓库 | 文件图标 | POST 到 `/chat/save-to-repo`，写入项目 `docs/`（仅项目内聊天显示） |

---

## 两种保存的区别

| | 保存为文件 | 保存到仓库 |
|---|---|---|
| 触发方式 | 浏览器 download | API POST |
| 落地位置 | 用户本地磁盘 | 项目 `docs/chat-export-*.md` + git |
| 显示条件 | 所有聊天 | 仅项目内（有 currentProjectId） |
