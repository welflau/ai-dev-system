## 1. 工单抽屉左边拖拽调整宽度

### 改动

**index.html**：在 `<aside class="drawer">` 内最顶部加 `<div class="drawer-resize-handle">` 手柄元素。

**styles.css**：
- `.drawer` 加 `min-width: 320px; max-width: 900px`
- 新增 `.drawer-resize-handle` 样式（5px 宽，hover/dragging 变蓝）
- `.drawer` 改用 `transform: translateX(100%)` 隐藏（替换原来的 `right: -500px`），避免拖宽后隐藏不完全露出空白面板
- `body.has-docked-drawer .main-container` 的 `padding-right` 改为读 CSS 变量 `var(--docked-drawer-width, 500px)`

**app.js**：
- 新增 `_initDrawerResize()` 函数：mousedown → mousemove 计算 delta → 更新 `drawer.style.width` 和 `--docked-drawer-width` CSS 变量，mouseup 保存到 localStorage
- `_syncDockBtnState()` 进入 Dock 时同步当前宽度到 `--docked-drawer-width`
- `_initDrawerResize` 恢复 localStorage 宽度时同步 CSS 变量
- `DOMContentLoaded` 时调用 `_initDrawerResize()`

## 2. 看板水平滚动条不可见

### 根因

`.board` 自身设了 `overflow-x: auto`，嵌套在也有 `overflow-x: auto` 的 `.content-area` 内，`.board` 自己把内容剪掉了，`.content-area` 看不到溢出，两个滚动条都不显示。

另外 `.content-area` 原来设了 `overflow-x: hidden` 也阻止了滚动条出现。

### 修复（styles.css）

```css
/* .board 撑开到实际宽度，不自己滚动 */
.board {
    width: max-content;
    min-width: 100%;
    overflow-x: visible;
    padding-bottom: 20px;
}

/* .content-area 负责横向滚动 */
.content-area {
    overflow-x: auto;   /* 原来是 hidden */
    padding-bottom: 28px;
}
```

## 3. 全屏 AI 助手时右侧出现空白面板

### 根因

抽屉宽度被拖宽（如 700px）后，原来 `right: -500px` 只藏了 500px，剩余部分露在屏幕右侧。全屏模式切换时抽屉没有被关闭。

### 修复

1. **CSS**：`.drawer` 改用 `transform: translateX(100%)` 隐藏，无论多宽都完全移出屏幕
2. **JS**：`toggleChatFullscreen()` 进入全屏时调 `closeDrawer()`
3. **CSS 兜底**：`body.chat-fullscreen .drawer { display: none !important }`
