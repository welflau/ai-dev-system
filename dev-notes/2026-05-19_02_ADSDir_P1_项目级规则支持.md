## 实现内容

### `SkillLoader.load_project_rules(repo_path)`

扫描 `{项目仓库}/.ads/rules/*.md`，与全局 `skills/rules/` 格式完全一致：

```
{仓库}/.ads/rules/
├── coding-style.md
├── ue-conventions.md
└── security.md
```

每个文件格式（YAML frontmatter + body）：

```markdown
---
alwaysApply: true        # 默认 True，对该项目无条件生效
priority: medium
description: 项目编码规范
---

# 规范内容
- 波次配置必须使用独立 JSON 文件
- Blueprint 前缀统一用 BP_TS_
```

返回拼接后的 prompt 文本，注入格式：
```
<!-- Rule: project.coding-style -->
# 规范内容
...
```

### 注入路径

**两条路径均已接入**：

1. **REACT 模式 / run_action**（`BaseAgent._resolve_skills_prompt`）
   - 查 `projects.git_repo_path`
   - 调 `load_project_rules(repo_path)`
   - 追加到 skills + memory 之间

2. **ChatAssistant 对话**（`ChatAssistantAgent._build_system_prompt`）
   - 全局 rules 之后追加项目 rules
   - 优先级：全局 rules（最高）→ 项目 rules → skills → memory

### 优先级层次

```
注入顺序（在 system prompt 中从前到后）：

1. 全局 rules（backend/skills/rules/global.md，alwaysApply=true）
2. 项目 rules（{仓库}/.ads/rules/*.md）  ← P1 新增
3. Skills（use_skills/ + .Agent/skills/）
4. 项目记忆（agent_memory，最近 3-5 条）
```

---

## 使用方式

1. 在项目 Git 仓库根目录创建 `.ads/rules/` 目录
2. 新建 `.md` 文件，写入规范内容（frontmatter 可省略，默认全部生效）
3. 重启 ADS 服务（或等热重载）即可生效

无 `.ads/rules/` 目录时静默跳过，不影响现有逻辑。
