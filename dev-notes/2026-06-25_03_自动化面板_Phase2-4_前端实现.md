## 主面板三区

### 模板卡片区
- `GET /api/automation/templates` 拉取 5 个内置模板
- 卡片点击预填到添加弹窗（名称/提示词/频率）

### 已安排任务列表
- `GET /api/automation/tasks?project_id=` 拉取
- 每张卡片：状态点（绿色/灰色/橙色闪烁）+ 名称 + 项目 tag + schedule_label + 下次运行时间 + 三个操作按钮

### 近期运行记录
- `GET /api/automation/runs?project_id=` 拉取最近 30 条
- 每行：状态图标 + 任务名 + 项目 tag + 时间（"X 分钟前"）+ 错误详情按钮

## 添加弹窗

字段：名称 / 工作空间（项目下拉）/ 提示词（+ 模型选择器）/ 执行频率（每天/按间隔/单次三 Tab）/ 生效日期区间

**cron 构造规则：**
- 每天：`{min} {hr} * * {weekdays}` （时间选择器 + 周几多选）
- 按间隔：`*/{n} * * * *` / `0 */{n} * * *` / `0 0 */{n} * *`
- 单次：schedule_type=once，cron_expr=null，用 datetime-local 值作 next_run_at

## 操作按钮

| 按钮 | API | 效果 |
|------|-----|------|
| ▶ 立即 | `POST /tasks/{id}/run-now` | 乐观更新状态点为橙色闪烁 |
| 禁用/启用 | `POST /tasks/{id}/disable` 或 `enable` | 重新计算 next_run_at |
| 删除 | `DELETE /tasks/{id}` | 二次确认弹窗 |

## SSE 实时更新

- `automation_task_started` → 状态点变为 running（橙色闪烁）
- `automation_task_finished` → 刷新任务卡片 + 在记录列表头部插入新行 + showToast 提示
