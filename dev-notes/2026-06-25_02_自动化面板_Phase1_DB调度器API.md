## 数据库

新增两张表（`SCHEMA_SQL` + `CREATE TABLE IF NOT EXISTS`，现有 DB 自动建表）：

```
automation_tasks   -- 任务配置
automation_runs    -- 执行记录（ON DELETE CASCADE）
```

**关键索引：**
- `idx_automation_tasks_next_run(next_run_at, enabled)` — 调度器 tick 查询用

## 调度器 `automation_scheduler.py`

- 每 60s 执行一次 `_tick()`
- 查 `enabled=1 AND next_run_at <= now` 的任务
- 有效日期范围检查（valid_from / valid_until）
- 执行完成后用 `croniter` 计算下次 `next_run_at` 写回
- `once` 类型执行后自动 `enabled=0`
- 直接调 `llm_client.chat()`，不走 ChatAssistant
- 执行前后推两个 SSE 事件到 `project:{project_id}` 频道：
  - `automation_task_started`
  - `automation_task_finished`

## API 端点（`/api/automation/`）

```
GET    /templates
GET    /tasks?project_id=
POST   /tasks
GET    /tasks/{id}
PUT    /tasks/{id}
DELETE /tasks/{id}
POST   /tasks/{id}/enable
POST   /tasks/{id}/disable
POST   /tasks/{id}/run-now
GET    /tasks/{id}/runs
GET    /runs?project_id=      ← 面板"近期记录"列表专用
```
