# 全局 AI 助手流式输出统一

> 日期：2026-05-11
> 状态：已完成

---

## 问题

全局 AI 助手（项目列表页）使用非流式 `POST /api/chat`，体验与项目内助手不一致：
- 无逐字回答效果
- 无思考过程面板
- 等待完整响应才显示

项目内助手使用 `POST /api/projects/{id}/chat/stream`，有思考步骤 + token-by-token 文字。

---

## 修改内容

### 1. `backend/agents/chat_assistant.py`

新增 `chat_global_stream()` 方法：

```python
async def chat_global_stream(self, user_message, images, history, projects_brief, session_id=None):
    # 复用 llm_client.chat_with_tools_stream
    # 使用 global scope 工具（confirm_project 等）
    # 输出格式与 chat_stream 完全一致：
    #   text_delta / tool_start / tool_done / action / message_done
```

### 2. `backend/api/chat.py`

新增 `POST /api/chat/stream` 端点 + `_global_chat_stream_generator`：
- 保存用户/助手消息（session_id 关联）
- 转发 `chat_global_stream` 的 SSE 事件
- 异常兜底处理

移除旧的单独 `GET /api/chat/thinking-stream` 路径依赖（仍保留端点兼容旧逻辑）。

### 3. `frontend/app.js`

全局聊天路径统一：

```javascript
// 改前
resp = await originalApi(`/chat`, { method: 'POST', body: ... });

// 改后
resp = await _sendChatStreaming(`/chat/stream`, { ... });
```

同时移除旧的独立 thinking-stream SSE 订阅（已由流式端点覆盖）。

## 效果

全局 AI 助手现在与项目内助手完全一致：
- ✅ 逐字回答（token-by-token）
- ✅ 思考过程面板（工具调用步骤 + 耗时）
- ✅ 状态文字：正在思考… → 正在调用工具… → 正在生成答案…
- ✅ 消息保存到 session 历史
