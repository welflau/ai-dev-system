# 开发日志 — 2026-04-28（Insight 主动注入）

## 背景

来源：腾讯 KM《如何让 AI 分工帮我做游戏（三）》vibecli insight 系统。

**问题**：DevAgent 编码时从不查知识库，FTS5 搜索只在 ChatAssistant 用户对话时触发。过去踩的坑、历史工单的 Reflexion 根因分析，不会自动阻止下次再踩。

---

## 实现

### 1. Orchestrator `_build_context` 末尾新增调用

```python
context["prior_insights"] = await self._fetch_prior_insights(
    ticket["title"], project_id
)
```

### 2. 新方法 `_fetch_prior_insights`

用工单标题做 FTS5 短语搜索，并发查两个来源：

**来源 1：knowledge_fts**（知识库文档 + dev-notes）
```sql
SELECT ki.filename, snippet(...) AS snippet
FROM knowledge_fts JOIN knowledge_index ki ...
WHERE knowledge_fts MATCH ? AND (ki.project_id = ? OR ki.project_id IS NULL)
ORDER BY rank LIMIT 3
```

**来源 2：tickets_fts**（已完成工单 + Reflexion 根因）
```sql
SELECT t.title, snippet(...), reflection_detail
FROM tickets_fts tf JOIN tickets t ...
WHERE tickets_fts MATCH ? AND tf.project_id = ?
  AND t.status IN ('acceptance_passed','testing_done','deployed')
ORDER BY rank LIMIT 3
```

输出格式（注入 context["prior_insights"]）：
```
# 历史经验（编码前参考，避免重复踩坑）

## 相关知识库条目
- [dev-notes/xxx.md] ...匹配片段...

## 相关历史工单经验
- [武器系统] ...匹配片段...
  曾经根因: AWeaponBase 缺少 UFUNCTION() 宏
  修复策略: 先过 R2 规则再写 OnRep
```

### 3. `PlanCodeChangeAction._plan` 注入

```python
insights_block = f"\n{prior_insights}\n" if prior_insights else ""
req_context = f"""... {insights_block}{reflection_block} ..."""
```

### 4. `WriteCodeAction.run` 注入

同样在 prompt 中 `insights_block` 放在 `reflection_block` 前，优先级：历史经验 > 反思块。
