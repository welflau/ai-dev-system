## 背景与动机

工单抽屉中产出物、操作日志、阶段总结、Agent 讨论原本分散在 4 个独立区块，
用户需上下翻动才能拼出工单完整经历，且缺乏人工介入入口。

本次改造：
1. 将 4 个区块**合并为一条统一时间轴**（logs + comments 按时间混排）
2. 新增**人工评论**功能，Agent 执行前注入评论作为最高优先级指令
3. Agent 每个阶段完成后**自动写总结评论**，下游 Agent 可感知

## 数据库

新增 `ticket_comments` 表，与 `ticket_logs` 并列，**不改动 tickets 主表**，零迁移风险：

```sql
CREATE TABLE IF NOT EXISTS ticket_comments (
    id          TEXT PRIMARY KEY,
    ticket_id   TEXT NOT NULL REFERENCES tickets(id),
    project_id  TEXT NOT NULL REFERENCES projects(id),
    author      TEXT NOT NULL,       -- "你" | "DevAgent" | ...
    author_type TEXT NOT NULL,       -- "human" | "agent"
    content     TEXT NOT NULL,
    phase       TEXT,                -- 关联阶段（如 "development_done"）
    created_at  TEXT NOT NULL
);
```

## 新增 API

### `GET /api/tickets/{ticket_id}/timeline`

将 `ticket_logs` 和 `ticket_comments` 在内存中按 `created_at` 合并排序，
每条加 `_type`（"log" | "comment"）和 `_tier`（"primary" | "secondary"）字段。

**分级规则：**
- `primary`：状态变更（from_status ≠ null）、人工评论、phase_complete、manual_action
- `secondary`：chat: 前缀的 LLM 工具调用、llm_call、详细编译日志

### `POST /api/tickets/{ticket_id}/comments`

写入人工评论（author_type="human"），同时推送 SSE `comment_added` 事件。

### `DELETE /api/tickets/{ticket_id}/comments/{comment_id}`

仅允许删除 author_type="human" 的评论。

## Orchestrator 改动

### 人工评论注入（process_ticket）

在 `context = await self._build_context(...)` 之后，调用 Agent 之前注入：

```python
human_comments = await db.fetch_all(
    "SELECT author, content, created_at FROM ticket_comments "
    "WHERE ticket_id = ? AND author_type = 'human' ORDER BY created_at ASC",
    (ticket_id,),
)
if human_comments:
    context["human_instructions"] = "## ⚠️ 人工指令（最高优先级，必须严格遵循）\n" + lines
```

日志中可见：`📌 工单 xxx 注入 N 条人工评论到 Agent prompt`

### 新增 `_write_phase_comment()` 方法

位于 `_log()` 方法之后，在以下阶段完成时自动调用：

| Agent | 触发时机 | 写入内容示例 |
|---|---|---|
| PlannerAgent | planning_done | `✅ PRD 已生成。验收标准：...` |
| ArchitectAgent | architecture_done | `✅ 架构设计完成，预计开发 N 小时` |
| DevAgent | development_done | `✅ 开发完成，共提交 N 个文件。自测：...` |
| UXAgent | ux_design_done | `✅ UX 设计完成，交互稿已输出` |
| ArtAgent | art_design_done | `✅ 视觉规范完成，资产清单已生成` |
| DevAgent | html_prototype_done | `✅ HTML 原型已生成。核心机制：...` |

## 前端改动

### openTicketDrawer 替换

原有「操作日志」section 和「Agent 讨论」section 替换为单一「📋 进展时间轴」section，
底部附评论输入栏，抽屉打开后异步调用 `_loadUnifiedTimeline(ticketId)` 加载数据。

### 新增函数

```
_loadUnifiedTimeline(ticketId)     → 调用 /timeline，渲染到 #unifiedTimeline
_renderUnifiedTimeline(container, items) → 按 primary/secondary 分级渲染
renderTimelineItem(item)           → log → renderLogItem()；comment → renderCommentBubble()
renderCommentBubble(comment)       → human 黄色气泡 / agent 蓝色气泡
toggleSecondaryLogs()              → 展开/收起二级详细日志
submitTicketComment(ticketId)      → POST 评论 → 刷新时间轴
deleteTicketComment(ticketId, id)  → DELETE → DOM 即时移除
refreshTimeline(ticketId)          → 供 SSE 事件调用
```

### SSE 接入

```javascript
// 新增监听
eventSource.addEventListener('comment_added', (e) => {
    const data = JSON.parse(e.data);
    if (data.ticket_id) refreshTimeline(data.ticket_id);
});
// ticket_status_changed 也同步触发时间轴刷新
```

### 样式

- 人工评论：黄色背景（`rgba(236,201,75,0.10)`），右偏移 32px，右上角圆角小
- Agent 评论：蓝色背景（`rgba(99,179,237,0.08)`），右偏移 32px，左上角圆角小
- 输入栏：底部 border-top 分隔，textarea + 发送按钮横排
