# 开发日志 — 2026-04-26（v0.19.x UE 编译修复 · AI 主动诊断）

## 版本
- v0.19.x（接续 CI/CD 合并 Phase A-E）
- 主要解决：UBT 编译无法从 Python 启动 + AI 助手看不了编译日志的问题

---

## 一、问题背景

用户手动点「编译 (UBT)」按钮后，一直报 `[error] 启动 UBT 异常:` 空消息失败。日志显示引擎检测 OK、`.uproject` 找到、target 推断正确，但 `asyncio.create_subprocess_exec` 就是跑不起来。

同时用户反馈：**编译失败后 AI 助手什么都不做**，只会让用户手动粘贴日志——体验很差。

---

## 二、根因分析

### 2.1 UBT 启动失败：asyncio + Windows + UBA 的三体问题

| 层 | 情况 |
|---|---|
| UBT 手动跑 | ✅ `Result: Succeeded`（322s，6 步）|
| Python `asyncio.create_subprocess_exec` | ❌ 抛空异常（类型未知，消息被截断）|
| Python `subprocess.Popen`（线程读）| ✅ 正常启动并读 stdout |

UE 5.7 启用了 **UBA（Unreal Build Accelerator）**：
- UBA 会启 `UbaServer` 子进程，并保持 stdout pipe handle 不关
- Python asyncio `ProactorEventLoop`（Windows 默认）在创建 subprocess 时对 inherited handle 有限制
- 结果：`asyncio.create_subprocess_exec` 要么抛空 `PermissionError`，要么 stdout pump 永远等不到 EOF 挂住

### 2.2 AI 助手看不了日志：SSE 不落 DB

UBT 每行 stdout 通过 SSE 推到前端「操作日志」面板，**不写任何 DB 表**。AI 助手的 tool_use 只能查 DB，完全看不到这些流式日志。

---

## 三、修复方案

### 3.1 线程模式 subprocess（核心修复）

`backend/actions/ue_compile_check.py`：用 `subprocess.Popen + threading.Thread + asyncio.Queue` 替代 `asyncio.create_subprocess_exec`：

```python
# 旧（失败）
proc = await asyncio.create_subprocess_exec(*cmd, stdout=PIPE, stderr=STDOUT)

# 新（成功）
proc = subprocess.Popen(cmd, stdin=DEVNULL, stdout=PIPE, stderr=STDOUT, ...)
queue = asyncio.Queue()
def _reader_thread():
    for line in proc.stdout:
        loop.call_soon_threadsafe(queue.put_nowait, line.rstrip())
    loop.call_soon_threadsafe(queue.put_nowait, _sentinel)
reader = threading.Thread(target=_reader_thread, daemon=True)
reader.start()
# 主协程从 queue 异步消费，保留流式输出
```

相同模式同步到 `ue_playtest.py` 和 `ue_package.py`。

### 3.2 AI 助手主动诊断

新增 `GetBuildLogsAction`（`backend/actions/chat/get_build_logs.py`）：
- 读 `ci_builds.build_log` + `ticket_logs.detail_data` 的结构化错误
- 注册为 ChatAssistantAgent tool，AI 能自动调用
- 触发词：「编译报错了」「Build 失败」「UBT 出错了」等

`UECIStrategy._proactive_diagnose()` 在编译失败后**主动**：
1. 规则引擎分析错误（不调 LLM，0 成本）
2. 写 `ci_build_diagnosis` 类型的消息到项目聊天
3. 前端渲染诊断卡 + 「让 DevAgent 自动修复」按钮

### 3.3 Pipeline 卡片实时化

- **后端**：`get_pipeline_status` 动态发现所有 `build_type`（含 UE 的 `ubt_compile/playtest/package_client`）
- **前端**：`buildTypeByStageId` 映射加 UE 条目；`ci_build_completed`/`ci_build_failed` SSE 触发 `loadDeliveryPage()` 刷新

### 3.4 .uproject 修复

MyFPS 的 `.uproject` 没有声明 `StateTree`/`GameplayStateTree` 插件，Build.cs 引用这两个模块时 UBT 报 warning。补全 Plugins 字段。
