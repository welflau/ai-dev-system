## 一、功能增强

### 1. AI 工具调用日志可观测性升级

**背景**：Harness Phase 1A 的 `audit_log_hook` 只记录工具名和耗时，无法看到"调用了什么参数、返回了什么"。

**改动：**
- `hooks/builtin.py`：新增 `_extract_input_summary` / `_extract_output_summary` 辅助函数，`_INPUT_KEY` 映射 30+ 工具的主参数
- `audit_log_hook` 同时处理 `POST_TOOL_USE` 和 `TOOL_ERROR`，写入 `input_summary / output_summary / error_msg` 三个新字段
- SSE `log_added` 消息格式升级：`web_search (425ms) → harness | ✓ 结果摘要`
- `database.py`：`_auto_migrate` 新增三列到 `tool_audit_log`
- 前端 `renderLogItem`：AI 工具条目可点击展开，显示输入/结果/错误详情
- 新增 `.log-item-expand` 展开样式

**效果：** 操作日志从"知道调了什么工具"升级为"知道调了什么、结果是什么、有没有失败"。

---

### 2. AI 聊天工具调用显示在操作日志面板

**背景**：日志面板只查 `ticket_logs`，AI 聊天工具调用（写入 `tool_audit_log`）完全不可见。

**改动：**
- `api/tickets.py` GET `/projects/{id}/logs`：UNION `ticket_logs + tool_audit_log`，历史和实时均覆盖
- `audit_log_hook` 写库后向项目 SSE 推 `log_added`（实时追加）
- `index.html` logAgentFilter：新增 `💬 ChatAssistant` 选项
- `app.js renderLogItem`：`chat:` 前缀加 💬/🔧 图标，紫色圆点区分
- `styles.css`：`.log-item.log-chat-tool` 样式

---

### 3. AI 助手新增 manage_skill 工具

**背景**：用户要关闭 UE 相关 Skills，AI 只能说"去界面手动点"。

**改动：**
- 新建 `actions/chat/manage_skill.py`：支持 `enable / disable / status` 三个操作
- `skill_ids` 支持前缀匹配（`["unreal-"]` 批量操作所有 UE Skills）
- `scope = global / project` 两级
- 注册到 `ChatAssistantAgent` + `_CROSS_SCOPE_TOOLS`（全局聊天也能用）

**示例**：`把所有 unreal- 开头的 Skills 禁用` → AI 直接调 `manage_skill(action=disable, skill_ids=["unreal-"])`

---

### 4. Agent 面板显示 Prompt 预览

**改动：**
- `api/agents.py`：新增 `_extract_prompt_preview()`，用 `inspect` 从 Action 源码提取 `instruction=` 静态字符串
- 每个 Action 返回 `prompt_preview` 字段
- `ChatAssistant` 有 `agent_prompt` 字段（完整能力摘要）
- 前端弹窗：Action 卡片加 `📋 查看 Prompt ▾` 折叠按钮
- 通用 `toggleBlock(id)` 函数

---

### 5. 聊天气泡支持 Markdown 表格渲染

**背景**：AI 回复 git log 等内容时输出 Markdown 表格，显示为原始 `| ... |` 文本。

**改动：**
- `_renderMarkdownText` 新增表格检测：识别 `|` 开头 + 下一行 `|---|` 分隔行
- 生成 `<table class="chat-md-table">`，thead/tbody 正确分离
- `overflow-x: auto` 防长表格撑破气泡
- `.chat-md-table-wrap / .chat-md-table` 样式

### 6. 实时日志面板新增关键词搜索框

**改动：**
- `index.html`：在级别/Agent 下拉左侧加 `logSearchInput`
- `filterLogPanel` 重写：同时过滤 `.log-entry` 和 `.log-item` 两种类型
- 支持前缀语法：`tool:web_search` / `agent:ChatAssistant` / `error:` / 普通关键词

## 三、配置调整

| 参数 | 旧值 | 新值 | 原因 |
|------|------|------|------|
| `CHAT_MAX_TOKENS` | 100,000 | 300,000 | 复杂问题多轮工具调用累积快，100k 太紧 |
| `CHAT_MAX_TURNS` | 30 | 50 | 深度检索需要更多轮次 |
| `CHAT_MAX_SECONDS` | 180s | 300s | 工具调用密集时需要更长时间 |
| `AGENT_MAX_TOKENS` | 200,000 | 不变 | 工单 Agent 失控风险更高 |
