# AI 助手思考过程 UX 改善

> 日期：2026-05-09
> 对标参考：CodeBuddy 截图
> 相关提交：a5f26e5 / 769549a / b5b2d3d

---

## 背景

流式聊天中，AI 调用工具时的「思考过程」面板体验粗糙：
- 状态文字在发送后立即消失，没有阶段感
- 工具步骤只有单行，看不到参数细节
- 折叠后只显示步数，没有耗时信息
- 思考面板在流式路径下插入位置有 Bug（两处）

对标 CodeBuddy 后确定改善方向，本次一次性落地。

---

## 改动内容

### 1. 两阶段状态文字（`app.js`）

`chatTyping` 气泡不再在发送时立即移除，改为保留并动态更新文字：

```
发送后       → 「正在思考…」
tool_start   → 「正在调用工具…」
tool_done    → 「正在生成答案…」
text_delta   → 移除 chatTyping，显示流式气泡
```

流式气泡在第一个 `text_delta` 到来前保持 `display: none`，避免空白气泡闪烁。

### 2. 工具步骤双行展示（`app.js`）

步骤条目从单行改为双行结构：

```html
<!-- 改前（单行） -->
<span class="ctp-step-dot"></span>
<span class="ctp-step-label">工具名</span>
<span class="ctp-step-hint">参数</span>
<span class="ctp-step-status">…</span>

<!-- 改后（双行） -->
<div class="ctp-step-row1">
    <span class="ctp-step-dot"></span>
    <span class="ctp-step-label">工具名</span>
    <span class="ctp-step-status">…</span>
</div>
<div class="ctp-step-row2">参数细节</div>   <!-- 有参数才渲染 -->
```

历史消息的思考面板（`appendChatMessage` 路径）同步升级为相同双行格式。

### 3. 执行统计（`app.js`）

折叠标题从「思考了 N 步」改为「思考了 N 步 · X.Xs」：

```js
let _thinkingStartTime = 0;  // _chatThinkingBegin() 时记录
// 面板完成时计算耗时
const elapsed = ((Date.now() - _thinkingStartTime) / 1000).toFixed(1);
title = `思考了 ${N} 步 · ${elapsed}s`;
```

### 4. 思考面板插入位置 Bug 修复（两处）

**Bug 1**（`b5b2d3d`）：流式路径下 `_chatThinkingAppend` 找不到 `chatTyping`（已被移除），直接 return，导致工具调用无思考面板。
修复：兜底插入到流式气泡内部 `.chat-msg-content`。

**Bug 2**（`769549a`）：流式路径思考面板插入到 `.chat-msg-content` 内部，导致缩进与非流式路径不一致。
修复：改为插在 `.chat-msg.assistant._streaming` 之前，作为兄弟节点，与非流式路径对齐。

### 5. CSS 新增（`styles.css`）

新增 `.ctp-step-row1`、`.ctp-step-row2` 双行布局样式，以及耗时文字的颜色规则。
