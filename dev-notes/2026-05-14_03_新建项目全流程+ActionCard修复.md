## 一、新建项目全流程自动化

### 1. GitHub 自动建仓库（`create_github_repo`）

新增 `CreateGithubRepoAction`，通过 `gh` CLI 在 `AiDS-Projects` 组织下自动创建仓库。

```
用户: "建个雷霆战机游戏"
AI: → create_github_repo(repo_name="thunder-strike", org="AiDS-Projects")
      ← https://github.com/AiDS-Projects/thunder-strike.git
    → confirm_project(name="ThunderStrike", git_remote_url="https://...", traits=[...])
      ← 确认卡片（可编辑项目名）
```

**防御性修复（`create_project.py`）**：项目创建时调 `_ensure_github_repo_exists()`，
先 `gh repo view` 检查，不存在则自动 `gh repo create --private`，
防止 AI 跳过 `create_github_repo` 直接 `confirm_project` 导致后续 `git push` 报 `Repository not found`。

### 2. 项目名可编辑卡片

`confirm_project` 卡片项目名改为 `<input>` 可编辑：
- AI 根据描述自动生成名字（如 `ThunderStrike`）
- 用户可直接修改
- 实时预览 GitHub repo 名（kebab-case）和本地路径

```
项目名称  [ThunderStrike        ]
          AiDS-Projects/thunder-strike
本地路径：F:\ADS_Projects\ThunderStrike
```

### 3. 默认项目路径配置化

**系统设置 → ⚙️ 通用 Tab** 新增：
- 默认本地路径：`F:\ADS_Projects`（可修改）
- GitHub 默认组织：`AiDS-Projects`（可修改）

后端 `system_settings` 表存储，`CreateProjectAction` 读取该配置自动确定 `repo_path`。

## 二、Action 卡片全系修复

### Bug 1：所有 Action 卡片不显示（QueryEngine 路径）

**根因**：`chat_assistant.py` 中：
```python
yield {"type": "action", **event.action_data}
```
`event.action_data` 含 `"type": "confirm_project"`，Python dict unpack 后者覆盖前者，
`etype` 变成 `"confirm_project"` 而非 `"action"`，`api/chat.py` handler 不命中，SSE 被丢弃。

**修复**：`yield {"type": "action", "payload": event.action_data}`，api/chat.py 读 `ev.get("payload")`。

### Bug 2：批量需求卡片渲染为错误的单条卡片

**根因**：`_sendChatStreaming` 流结束时无论什么 action type 都调 `_buildConfirmCardHtml()`，
该函数只处理单条 `confirm_requirement`。当 `finalAction.type === "confirm_requirements_batch"` 时，
`priority/title` 均 `undefined`，渲染出空卡片，用户点击报"需求标题和描述不能为空"。

**修复**：
- 新增 `_buildAnyActionCardHtml(action)` 根据 type 路由
- 新增 `_buildBatchRequirementsCardHtml(action)` 独立批量卡片构建函数
- `_sendChatStreaming` 改用 `_buildAnyActionCardHtml`

## 三、可观测性增强

### QueryEngine 运行统计入库（Layer 2）

`engine.py` 在 `MessageDone` 前 emit `SESSION_END` Hook，携带：
- `rounds`：本次对话使用的 LLM 轮次
- `total_tokens`：消耗的 Token 总数及占预算百分比
- `elapsed_s`：总耗时
- `reason`：`done` / `max_rounds`（超轮次）

`audit_log_hook` 新增处理 `SESSION_END`：
- 写 `tool_audit_log`（`tool_name="query_engine:done"`）
- 推 SSE `log_added` 到日志面板，显示 `rounds:6 tokens:23456(7.8%) elapsed:2.3s`

### Hook 可观测性浮层

顶部指标条新增 `🔧 Hooks 3 | 50次`，点击弹出浮层：
1. 已注册 Hook 表格（调用次数/失败数/均耗时）
2. Shell 限流进度条（≥90% 显示 ⚠️）
3. 近 1 小时工具调用 TOP 8

## 五、实战问题排查记录

**ThunderStrike 工单卡在架构中（13 分钟）**

- **现象**：35 张工单待启动，3 张架构中卡住，工单显示"已超时"
- **根因**：`AiDS-Projects/ThunderStrike` 仓库未被创建，`git push` 报 `Repository not found`
- **解决**：手动 `gh repo create AiDS-Projects/ThunderStrike` + `git push origin main`，zombie 检测自动重置工单
- **预防**：`_ensure_github_repo_exists()` 在项目创建时自动补建仓库
