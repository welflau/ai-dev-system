# Skill 主动触发改造（v0.20）

> 日期：2026-05-09
> 关联设计文档：`docs/20260508_Skill主动触发改造方案.md`
> 状态：已完成（Phase 1-4）

---

## 背景与动机

原有 Skill 架构是「被动注入」：Agent 初始化时把所有匹配的 skill prompt.md 全文拼入 system prompt，每次对话都携带，不论当前问题是否需要。

实测消耗（UE5 项目 ChatAssistant 一次对话）：
- 被动注入：~2000 tokens/次（含 unreal-editor-control ~800 tokens）
- 主动触发：~400 tokens/次（索引表 ~200 + 按需加载 1 个 Skill ~600，多数对话不需要加载）

问题清单：不可见、无法追踪、浪费 Token、无法按项目配置、无法自定义。

---

## 改造后架构

```
启动时：
  只生成 Skill 索引表（名称+描述，~200 tokens），不加载全文

每次 LLM 调用：
  system_prompt = 基础指令 + Skill 索引表 + 项目上下文

AI 推理时：
  判断当前问题需要哪个 Skill
  → 显式调用 load_skill("skill-id")
  → Skill 全文动态注入到当前对话上下文
  → 思考日志显示「📚 加载 Skill: xxx」
```

## Phase 详情

### Phase 1 — load_skill 工具 + 索引注入

`LoadSkillAction.run()`：
1. 校验 `skill_id` 在项目可用列表内
2. 自定义 Skill（`custom.*`）走文件系统读取；全局 Skill 走 `skill_loader.get_skill_prompt()`
3. 返回 `{type: "skill_loaded", content: <全文>}` → LLM 当前对话上下文立即可用

`ChatAssistant._build_system_prompt()` 改动：
```python
# Before: 全量注入（~2000 tokens）
skills_section = f"\n## 专业技能 (Skills)\n{self._skills_prompt}\n"

# After: 索引表（~200 tokens）+ load_skill 指令
_skills_index = skill_loader.build_index_for_agent(self.agent_type, traits=project_traits)
skills_section = f"## 可用 Skills（按需加载）\n{_skills_index}\n如需深度规范知识，先调 load_skill..."
```

### Phase 2 — project_skills 表 + API

```sql
CREATE TABLE IF NOT EXISTS project_skills (
    id          TEXT PRIMARY KEY,
    project_id  TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    skill_id    TEXT NOT NULL,
    source      TEXT DEFAULT 'global',  -- 'global' | 'custom'
    enabled     INTEGER DEFAULT 1,
    custom_name TEXT,
    custom_path TEXT,
    created_at  TEXT NOT NULL,
    UNIQUE(project_id, skill_id)
);
```

`_get_available_skill_ids()` 优先读项目配置表，无配置时降级为全局默认（向后兼容）。

### Phase 3 — 自定义 Skill 上传

- `POST /api/projects/{id}/skills/upload`：接收 `.md` 文件，存入 `backend/data/project_skills/{project_id}/`
- `skill_id` 格式：`custom.{文件名去后缀}`，避免与全局 Skill ID 冲突
- 自动提取文件名 / frontmatter `name` 字段 / 第一个 `# 标题` 作为显示名

### Phase 4 — 前端配置页

Settings → Agent/扩展 → Skills Tab：
- **系统 Skills**：列出全局 Skill，每行有 Toggle 开关（项目级覆盖），已覆盖的显示「↩ 重置」按钮
- **自定义 Skills**：上传 `.md` 文件，显示已上传列表，支持删除

---

## 关键设计决策

1. **custom.* 前缀**：自定义 Skill ID 加前缀，与全局 Skill 命名空间分离，`load_skill` 按前缀分支加载逻辑
2. **project_traits 先于索引构建**：ChatAssistant 修复了 traits 解析顺序（先解析 traits，再调 `build_index_for_agent(traits=project_traits)`），确保 trait 过滤生效
3. **自定义 Skill 也加入索引**：`_build_system_prompt` 额外查询 `project_skills` 表追加 `custom.*` 行到索引表，AI 可见并按需加载
