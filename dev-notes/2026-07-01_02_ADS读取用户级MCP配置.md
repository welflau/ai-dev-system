# ADS 读取用户级 MCP 配置（~/.codebuddy、~/.claude）

**日期**：2026-07-01
**状态**：已上线 ✅

## 背景

用户在 `~/.codebuddy/mcp.json` 配了 7 个 MCP（rules-bridge、tapd、ue-igmaestro-mcp、gongfeng、crashsight、iWiki、km），`~/.claude/.mcp.json` 还有 1 个（unreal-mcp），但 ADS 的 MCP 面板一个都不显示，运行时也不连接。

排查发现 ADS 的 MCP 解析（面板显示 + 运行时连接）覆盖面太窄，存在三个盲点：

1. **不读用户主目录** `~/.codebuddy/`、`~/.claude/`（而 skill 面板早就读了，不一致）
2. **不读 `mcp.json` / `.mcp.json` 文件名**（只找 `settings.json` 里的 `mcpServers` 字段）
3. **运行时甚至不读项目 `.codebuddy/settings.json`**（只读 `.claude/settings.json`）

另外 CodeBuddy 用 `disabled: false` 表示启用，而 ADS/内置用 `enabled: true`，字段约定不一致会导致 CodeBuddy 格式的 server 被误判为未启用。

## 核心改动

### 运行时连接（`backend/mcp_client.py`）

重写 `_load_project_mcp_config`，新增三个 helper：

- `_normalize_enabled(cfg)` — 归一化启用状态：`disabled` 优先，其次 `enabled`，都无则视为启用（Claude Code 约定）
- `_read_mcp_servers_file(path, source, top_level)` — 读单个配置文件的 `mcpServers` 字段（或顶层，用于 `.ads/mcp_servers.json`），归一化后返回 `{name: cfg}`
- `_load_user_mcp_config()` — 加载用户主目录级配置：`~/.codebuddy/mcp.json`、`~/.codebuddy/settings.json`、`~/.claude/.mcp.json`、`~/.claude/mcp.json`、`~/.claude/settings.json`

`_load_project_mcp_config` 现按优先级从低到高合并：

```
0. 用户级 ~/.codebuddy、~/.claude
1. 项目 .claude/settings.json
2. 项目 .codebuddy/settings.json + .codebuddy/mcp.json
3. 项目 .ads/mcp_servers.json（顶层即 servers，优先级最高）
```

### 面板显示（`backend/api/projects.py`）

`get_project_mcp_all` 的「用户 MCP」一路改为复用 `_load_project_mcp_config`，覆盖用户级 + `.codebuddy` + `mcp.json`。

**敏感信息处理**：面板只返回 `name` / `description` / `command` / `transport` / `enabled` / 来源，**headers / env / url token 等敏感字段一律不返回给前端**。

### 前端（`frontend/app.js`）

- user MCP 无 `status` 时，按 `enabled` 显示「✅ 已启用 / ⚪ 未启用」徽章
- 显示 transport 类型（stdio / http / streamable-http）
- 更新 user 分组文案为「来自 ~/.codebuddy、~/.claude 及项目 .claude/.codebuddy/.ads 配置」

## 注意

重启后端后，ADS 对话时会真的去 spawn/连接这些 MCP（如 `ue-igmaestro-mcp` 的 python 进程、`rules-bridge` 的 node 进程）。

## 改动文件

- `backend/mcp_client.py` — 多路 MCP 配置解析（用户级 + .codebuddy + mcp.json + disabled 归一化）
- `backend/api/projects.py` — 面板显示用户级 MCP，隐藏敏感字段
- `frontend/app.js` — MCP 面板 enabled/transport 徽章 + 分组文案
