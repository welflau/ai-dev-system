## 一、思考过程历史记录不显示（双 Bug 修复）

### Bug 1：thinking_steps 未收集（后端）

**位置**：`backend/agents/chat_assistant.py` → `_ChatToolExecutor.execute()`

**问题**：`_emit_thinking()` 只在 `project_id or session_id` 存在时调用，全局聊天（两者均为 None）完全跳过，`executor.thinking_steps` 永远为空列表，`thinking_json` 存 NULL。

**修复**：`execute()` 始终调用 `_emit_thinking()`，`_emit_thinking()` 内部把收集与推送分开：
- `thinking_steps.append(...)` → 始终执行（done 阶段）
- SSE 推送 → 仍只在有 project_id/session_id 时执行

### Bug 2：API 不返回 thinking（后端）

**位置**：`backend/api/chat.py` → `GET /api/chat/sessions/{id}/messages`（全局聊天版本）

**问题**：全局聊天的会话消息 API 没有做 `thinking_json → thinking` 的转换，前端收到的 `thinking` 始终是 `undefined`。

**修复**：补上一行：
```python
msg["thinking"] = json.loads(msg.pop("thinking_json", None) or "null") or []
```

### Bug 3：历史记录思考面板对齐错位（前端）

**位置**：`frontend/app.js` → `appendChatBubble()`

**问题**：历史加载时 `thinkingHtml` 在 `.chat-msg-content` 里面，导致 avatar 与思考面板对齐而不是与气泡对齐。

**修复**：`thinkingHtml` 改为在 `container.appendChild(msgEl)` 之前单独插入，作为兄弟节点，与实时流式路径结构一致。

## 二、全局聊天流式输出（思考过程+逐字）统一

**背景**：全局聊天（项目列表页）使用非流式 API，无思考面板和逐字效果。

**修复**：
- 新增 `ChatAssistantAgent.chat_global_stream()` 方法（复用 `chat_with_tools_stream`）
- 新增 `POST /api/chat/stream` 全局流式端点 + `_global_chat_stream_generator`
- 前端改用 `_sendChatStreaming('/chat/stream', ...)` 路径

---

## 三、思考面板重构为通用模块

**背景**：旧思考面板函数硬编码全局 DOM，分屏格无法复用。

**修复**：提取为工厂函数 `createThinkingContext(container)`，返回 `{begin, append, finish}` 三个方法，不依赖全局 DOM 选择器。全局默认实例 `_globalThinking` 向后兼容。
