## 问题排查过程

### 问题 1：asyncio.create_subprocess_exec 抛空异常

**现象**：`[error] 启动 Editor-Cmd 异常:` 后面内容为空，耗时 0s。

**根因**：Python 3.14 + Windows `ProactorEventLoop` 与 UE 子进程管理不兼容，
`asyncio.create_subprocess_exec` 抛 `NotImplementedError()` 空消息异常。

**修复**：改用 `subprocess.Popen + asyncio.Queue + 线程读取`，与 `ue_compile_check.py` 保持一致。

### 问题 2：.pyc 字节码缓存导致修复不生效

**现象**：修改代码后重启服务器，日志仍显示旧行为。

**根因**：`backend/actions/__pycache__/ue_playtest.cpython-314.pyc` 时间戳为 12:22，
比 `.py` 文件(20:59)旧，但服务器 uvicorn reload 机制多进程叠加（8个进程同时监听 8000 端口），
旧进程一直在响应请求。

**修复**：
1. 杀死所有 Python 进程（`Get-Process python | Stop-Process -Force`）
2. 删除所有 `.pyc` 缓存
3. `main.py` 加 `reload_includes=["**/*.py"]` 避免 DB 文件触发热重载

### 问题 3：DLL 检查路径错误

**现象**：UBT 编译成功后 Automation 测试仍报"游戏模块未编译"。

**根因**：DLL 检查在找 `MyFPS.dll`，但 UE5 Editor 编译产出的是 `UnrealEditor-MyFPS.dll`。

**修复**：检查顺序改为：
1. `UnrealEditor-{proj}.dll`（UE5 Editor，优先）
2. `{proj}.dll`（UE4 Game）
3. `{proj}Editor.dll`（UE4 Editor）

### 问题 4：exit=1 被误判为失败

**现象**：UE 执行完 Automation 后 exit=1 但 0 个测试用例，被判为 playtest_failed。

**根因**：UE Automation Framework 找不到 `Project.` filter 匹配的测试时以 exit=1 退出，
这对新建 C++ 项目是正常行为（暂无测试用例）。

**修复**：`total==0 && failed==0` 时不论 exit code 都视为 success。
