## 交付内容

### 后端（`backend/`）

**`database.py`**：
- 新增 `chat_sessions` 表（id / project_id / title / created_at / updated_at）
- `chat_messages` 加 `session_id` 列（ALTER 迁移，默认 `'default'`）
- 启动时自动为现有消息创建 `default` session（向后兼容）
- 启动时创建 `__global__` 占位 project（满足 FK 约束，用于存储全局聊天消息）

**`api/chat.py`**：
- 项目内 session 端点（各 5 个）：
  - `GET /projects/{id}/chat/sessions` — 列出会话
  - `POST /projects/{id}/chat/sessions` — 新建会话
  - `DELETE /projects/{id}/chat/sessions/all` — 清空全部
  - `DELETE /projects/{id}/chat/sessions/{sid}` — 删除单条
  - `GET /projects/{id}/chat/sessions/{sid}/messages` — 获取会话消息
- 全局 session 端点（同上，前缀 `/api/chat/sessions`）
- `_save_chat_message()` 加 `session_id` 参数，首条用户消息自动更新会话标题
- 全局聊天端点保存消息到 DB（`project_id='__global__'`）
- `ChatRequest` 和 `GlobalChatRequest` 加 `chat_session_id` 字段

**`api/projects.py`**：
- 项目列表过滤 `__global__` 占位 project（不显示在 UI）

### 前端（`frontend/`）

**`index.html`**：
- 顶栏加 `＋`（新建对话）和 `🕐`（历史对话）按钮
- 删除原有 🗑️ 清空聊天按钮（新建对话已覆盖该功能）
- 历史面板 HTML（绝对定位，从 header 下方 top:48px 开始）

**`app.js`**：
- 新增全局变量 `_currentChatSessionId`
- 新增函数：`newChatSession` / `toggleChatHistory` / `loadChatSessions` / `switchChatSession` / `deleteChatSession` / `deleteAllChatSessions` / `_ensureChatSession`
- `sendChatMessage()` 发送前自动确保有 session，body 带 `chat_session_id`
- `loadChatHistory()` 改走 session API（全局+项目内），不再依赖 localStorage DOM 快照
- 切换项目/返回列表时重置 `_currentChatSessionId`，关闭历史面板

**`styles.css`**：
- 历史面板、分组标题、条目样式（hover / active / 删除按钮 hover 显示）

## 问题排查记录

| 问题 | 根因 | 修复 |
|---|---|---|
| 历史面板遮挡顶栏按钮，无法关闭 | CSS `top: 0` 覆盖 header | 改为 `top: 48px`，并在面板内加 ✕ 关闭按钮 |
