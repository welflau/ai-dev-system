# AI 自动开发系统 - 快速演示指南

## 🎉 系统概述

AI 自动开发系统 v0.6.0 — 从自然语言需求到可运行软件的全自动开发平台。6 个 AI Agent 协作完成需求分析、架构设计、代码生成、测试、代码审查和部署配置。

## 📋 演示步骤

### 第一步：启动后端服务

```bash
cd D:\A_Works\ai-dev-system\backend
python main.py
```

看到以下信息表示启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

### 第二步：打开前端界面

浏览器访问：**http://localhost:8000/app**

### 第三步：体验系统功能

#### 1. 查看系统状态
- 首页显示系统概览、项目统计和 LLM 连接状态
- LLM 状态徽章实时显示是否配置了 AI 引擎

#### 2. 提交一个开发需求
- 点击左侧菜单的 **"📝 提交需求"**
- 在"项目描述"中输入你的需求，例如：
  ```
  我想开发一个简单的待办事项应用，包含以下功能：
  1. 添加新的待办事项
  2. 标记事项为完成状态
  3. 删除已完成的事项
  4. 查看所有待办事项列表
  ```
- 在"技术栈偏好"中输入（可选）：`Python, FastAPI, HTML`
- 在"项目名称"中输入（可选）：`todo-app`
- 点击 **"提交需求"** 按钮

#### 3. 查看 Pipeline 看板
- 提交成功后自动跳转到项目详情
- **Pipeline 看板**横向展示 5 个阶段：需求 → 架构 → 开发 → 测试 → 部署
- 每个阶段带状态颜色：绿色(完成) / 蓝色脉冲(运行中) / 灰色(待执行)

#### 4. 一键执行
- 点击 **⚡ 一键全量执行** 按钮
- 系统自动依次执行所有任务：
  - ProductAgent → 需求分析 + PRD 生成
  - ArchitectAgent → 架构设计文档
  - DevAgent → 代码生成
  - TestAgent → pytest 测试用例
  - ReviewAgent → 代码审查报告（评分 A~F）
  - DeployAgent → Dockerfile + docker-compose + CI/CD + Nginx
- SSE 实时推送，看板自动更新进度

#### 5. 查看执行日志
- 点击 Pipeline 看板中的任意步骤
- 弹出 VS Code 风格暗色日志抽屉，显示执行详情和耗时

#### 6. 浏览生成的文件
- 在项目详情页点击 **"文件"** 标签
- 浏览 Agent 生成的所有文件（代码 + 文档 + 配置）
- 支持语法高亮预览（Python/JS/HTML/CSS/JSON 等）

#### 7. 下载项目
- 点击 **📦 下载项目** 按钮
- 一键打包 ZIP 下载全部生成的项目文件

#### 8. 配置 LLM（可选但推荐）
- 点击左侧菜单的 **"⚙️ LLM 配置"**
- 填入 API Base URL、API Key、模型名称
- 点击 **"测试连接"** 验证
- 配置 LLM 后，Agent 将使用 AI 生成更高质量的内容

## 🔧 技术架构

### 后端
- **框架**: FastAPI
- **语言**: Python 3.8+
- **数据存储**: SQLite (持久化)
- **实时推送**: SSE (Server-Sent Events)
- **工具集成**: 8 个工具（文件操作 + Git 操作）
- **API 文档**: http://localhost:8000/docs

### 前端
- **技术**: 纯 HTML5 + CSS3 + JavaScript (ES6+)
- **代码高亮**: highlight.js (CDN)
- **实时通信**: EventSource (SSE)

## 📊 当前系统能力

### 6 大 AI Agent
| Agent | 功能 | 模式 |
|-------|------|------|
| ProductAgent | 需求分析 + PRD 生成 | LLM + 模板降级 |
| ArchitectAgent | 7 种架构设计方案 | LLM + 模板降级 |
| DevAgent | 14 种代码生成任务 | LLM + 模板降级 |
| TestAgent | pytest 测试用例生成 | LLM + 模板降级 |
| ReviewAgent | 代码审查（10 种规则） | LLM + 规则引擎 |
| DeployAgent | Docker + CI/CD + Nginx | LLM + 模板降级 |

### 已注册工具（8个）
1. file_writer - 文件写入
2. file_reader - 文件读取
3. directory_lister - 目录列表
4. git_init - Git 初始化
5. git_add - Git 添加文件
6. git_commit - Git 提交
7. git_push - Git 推送
8. git_create_branch - Git 创建分支

### 测试覆盖
- 92 个测试用例全部通过
- GitHub Actions CI 自动运行

## 📝 注意事项

1. **后端服务必须先启动**，前端通过 API 与后端通信
2. **端口冲突**: 如果 8000 端口被占用，需要修改端口配置
3. **浏览器兼容性**: 建议使用 Chrome、Firefox 或 Edge
4. **LLM 配置**: 不配置 LLM 也能使用（模板降级模式），但 AI 生成效果更好
