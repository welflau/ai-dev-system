# /execute — 执行编码

读取 `CURRENT_PLAN.md`，找到第一个含未完成任务 `[ ]` 的 Wave，全自动执行。

**核心原则**：全程自动，仅在以下情况暂停：
- 测试/类型检查反复失败且无法自动修复
- 需要修改其他 Agent 负责的文件（跨边界冲突）

## 主循环

```
while (存在未完成的 Wave) {
  wave = 找到第一个有 [ ] 的 Wave
  if (串行 Wave) → 路径 A
  if (并行 Wave，≥2 Agent) → 路径 B
  if (并行 Wave，只有 1 Agent) → 退化为路径 A
}
所有 Wave 完成 → 全仓验证 → 提示 /review → /done
```

## 路径 A：串行 Wave

1. 按任务清单逐步实现代码
2. 同步创建/更新对应单元测试（Colocated：源码同级 `__tests__/` 目录）
3. 运行测试确认通过（失败时自动修复，最多 3 次重试）
4. 完成一个任务立即打勾 `[x]`，继续下一个

## 路径 B：并行 Wave

1. 为每个 Agent 合成 Subagent Prompt：
   ```
   你是 Agent [X]，负责 [角色]。
   目录范围：只在 [目录] 内操作，不得修改其他目录。
   任务清单：[从计划文件提取的 - [ ] 列表]
   完成后运行测试，在计划文件中打勾 [x]。
   ```
2. 通过 Task 工具**并行派发**所有 Subagent
3. 汇总结果，运行全仓类型检查 + 测试
4. 若有失败，自动修复（最多 3 次）

## Scope 外发现 → Backlog

编码中发现超出计划范围的优化点、技术债，自动追加到 `BACKLOG.md`：
```
- **[P2] 简短标题**：一句话描述。
  > 来源：{当前计划} · {时间}
```

## 完成输出

> "所有 Wave 完成，测试通过。建议运行 `/review` 做代码评审；评审通过后运行 `/done`。"
