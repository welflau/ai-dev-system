# /ue-gdd

根据一句游戏描述，生成结构化的游戏设计文档（GDD），保存到项目目录供后续命令参考。

## 用法

```
/ue-gdd <游戏描述>
/ue-gdd <游戏描述> --output <路径>   # 自定义输出路径
/ue-gdd --file <已有描述文档>        # 从文件读取描述
```

## 执行前置

读取 `.claude/ue-config.json`（若存在）：
```python
config = json.loads(open('.claude/ue-config.json', encoding='utf-8').read())
project_name = config.get('project_name', 'MyGame')
```

## 行为

1. **分析游戏描述**，提取：
   - 游戏类型（TPS/FPS/RPG/平台跳跃/双摇杆射击/...）
   - 核心玩法循环（3-5 个主要动作）
   - 目标玩家群体
   - 游戏规模（原型/Demo/完整游戏）

2. **生成结构化 GDD**，包含：
   - 游戏概述与定位
   - 核心玩法机制
   - 角色与敌人设计
   - 关卡与世界设计
   - UI/UX 设计
   - 技术架构建议（对应 UE 实现）
   - 开发里程碑

3. **保存文档**：
   - 默认路径：`.claude/ue-runtime/GDD.md`（纳入项目知识库）
   - 同时更新 `.claude/ue-runtime/PROJECT_MAP.md` 的概述部分

## GDD 文档结构

```markdown
# {游戏名} 游戏设计文档

> 版本：1.0 | 生成时间：{date}

## 一、游戏概述
- 类型、平台、目标受众
- 核心体验（一句话）

## 二、核心玩法
- 核心循环（主要动作流程）
- 关键机制列表

## 三、角色设计
- 玩家角色（能力/属性）
- 敌人类型（AI 行为/难度梯度）
- NPC（若有）

## 四、关卡与世界设计
- 场景类型
- 关卡结构（线性/开放/混合）
- 关键场景描述

## 五、UI/UX 设计
- HUD 元素
- 菜单结构

## 六、UE 技术架构建议
- 核心 C++ 类
- 推荐使用的 UE 子系统
- 资产清单估算

## 七、开发里程碑
- M1 原型（可玩核心循环）
- M2 竖版（1个完整关卡）
- M3 Demo（完整体验）
```

## 与其他命令的关系

```
/ue-gdd "游戏描述"      ← 生成 GDD.md
    ↓
/ue-architect           ← 读取 GDD.md 确定技术架构
    ↓
/ue-scaffold            ← 生成项目骨架
    ↓
/ue-auto               ← 全自动执行（内部调用 /ue-gdd）
```

## 示例

```
/ue-gdd 俯视角双摇杆射击，有敌人AI追击和道具拾取，单机，小型原型

/ue-gdd 第三人称动作 RPG，有近战连击和技能系统，类黑魂风格

/ue-gdd 2.5D 平台跳跃，收集型游戏，卡通风格，面向儿童
```
