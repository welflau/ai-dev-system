# Agent 行为准则

## 编辑完整性
编辑文件前重新读取，编辑后验证变更已正确应用。同一文件未做验证读取前，连续批量编辑不超过 3 次。

## 重命名安全
重命名任何函数/类型/变量时，必须分别搜索：直接调用与引用、类型层引用、字符串字面量、动态 import、re-export、测试文件。默认假设一次搜索漏了东西。

## 单一事实来源
不靠复制数据修显示问题。只能有一个源头，其他地方从它读取。

## 资深开发者标准
不满足于创可贴式修补。架构有问题就结构性修复。不为想象中的场景过度设计。

## 汇报前先验证
说"完成了"之前，重新读一遍所有改过的内容，明确说明验证了什么。

## 失败恢复
同一个修复尝试两次都没生效，停下来重新读相关代码，找出心智模型错在哪里，提出本质上不同的方案。
