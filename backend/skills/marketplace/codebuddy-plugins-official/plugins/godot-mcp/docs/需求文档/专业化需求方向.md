目前插件已经打通了对godot简单功能的mcp接口。要赋予制作中型体量游戏的能力，还需要做下面几步
1. 在用户提供简单的游戏方案时，将其扩充成完整的交互体验，关卡设计，养成曲线，等一系列的策划文档。然后根据文档设计代码模块，规定每个模块的功能职责。最后再根据功能职责完备代码逻辑
2. 需要额外打通godot的详细资产生成接口，将外部生成好的模型，贴图文件，动画，通过什么方式载入游戏内使用。游戏摄像头如何控制，玩家控制如何实现，动画如何播放等等
3. 逻辑和数值分离，创建详细的配置表系统，将配置内容以json格式存储。配置表就是需要控制json格式合法，代码就需要提供自修复系统，运行前预编译一下，分析输出，解决bug后再编译，直到编译通过
4. 以上内容，通过多层的skill实现，有时会配以固定的执行脚本，配置文件等，比如生成资产，需要先通过外部的第三方模型生成贴图，这时第三方的apikey就需要放到skill内，但是不是直接放，这个需要设计一下方案

在使用此 Skill 前，确保已配置 API 密钥：

1. 复制 `.env.template` 为 `.env.local`
2. 填入对应服务的 API Key
3. 确保 `.env.local` 已添加到 `.gitignore`

## 密钥检查命令

在执行外部 API 调用前，先检查密钥是否已配置：

```
检查密钥配置状态：

- SD_API_KEY: [已配置/未配置]
...
```

## 安全原则

1. **永不打印密钥**：日志和输出中仅显示密钥是否存在
2. **环境隔离**：开发/生产环境使用不同密钥
3. **权限最小化**：仅申请必要的 API 权限
4. **错误处理**：密钥无效时给出清晰指引

## 可用服务

### 3D 模型生成
- `generate_3d_model(prompt, style, format)`
- 支持服务：Meshy.ai, Tripo3D

### 贴图生成
- `generate_texture(prompt, size, style)`
- 支持服务：Stable Diffusion, Leonardo.ai

### 音频生成
- `generate_music(prompt, duration, genre)`
- `generate_sfx(prompt, duration)`
- `generate_voice(text, voice_id)`
