# CodeBuddy IDE 游戏引擎 MCP 一键部署 — 功能需求文档
## 1. 概述
本文档面向 CodeBuddy IDE 研发团队，描述 Unity / Unreal Engine / Godot 三个游戏引擎的 MCP 插件在 IDE 中一键部署的功能需求。目标：用户选择目标引擎后，IDE 自动完成 MCP Server 的下载、环境配置、配置写入和启动，无需手动编辑 JSON 配置文件。

## 3. Unity MCP 一键部署需求
### 3.1 部署流程
IDE 在用户触发「部署 Unity MCP」时，需执行以下步骤：
1) 检测 Node.js 环境
- 检查系统是否已安装 Node.js >= 18。
- 若已安装：直接使用。
- 若未安装：
    - 若 IDE 有内置 Node 运行时且版本满足要求，直接使用。
    - 若 IDE 具备自动安装能力，提示用户确认后自动安装 Node.js LTS。
    - 若不具备，给出明确提示引导用户手动安装，并在安装完成后允许用户重新触发部署。
2) 获取 mcp-unity Server
- 从仓库下载，运行，获取 Server~/build/index.js 及其所需文件。
- 提示玩家在unity中手动配置package
3) 自动写入 MCP 配置
将以下配置写入 CodeBuddy 的 MCP 配置文件，无需用户手动编辑：
```json
{
  "mcpServers": {
    "mcp-unity": {
      "command": "node",
      "args": ["<IDE管理的绝对路径>/mcp-unity/Server~/build/index.js"],
      "env": {
        "UNITY_PORT": "8090"
      }
    }
  }
}
```
4) 部署完成反馈 — 提示用户：部署成功。告诉玩家在unity editor中找到mcp unity的tool，serverwindow，选择start server。
- 每次 IDE 尝试使用 mcp-unity 的工具时（或检测到 mcp-unity 配置已启用时），如果连接失败或首次使用，向用户显示提示：
> 请确认已在 Unity Editor 中启动 MCP Server：Tools -> MCP Unity -> Server Window -> Start Server
- 此提示应在 IDE 的交互界面中可见（如 Chat 面板顶部提示条或工具调用失败时的错误引导）。
### 3.2 可选配置项
|配置项|环境变量|默认值|说明|
|WebSocket 端口|UNITY_PORT|8090|Unity Editor 侧的 WebSocket 监听端口|
|请求超时|UNITY_REQUEST_TIMEOUT|10 (秒)|MCP 请求超时时间|

## 4. UE MCP 一键部署需求
### 4.1 主插件：UEEditorMCP
#### 4.1.1 部署流程
UEEditorMCP 包含两部分：C++ 编辑器插件（在 UE 中编译运行）和 Python MCP Server（IDE 侧启动）。IDE 一键部署负责 Python MCP Server 部分，同时尽可能辅助 C++ 插件的部署。
1) UE C++ 插件部分
- 若 IDE 能感知用户 UE 项目路径（如通过工作区目录识别 .uproject 文件）：
    - 检测项目 Plugins/UEEditorMCP/ 是否已存在。
    - 若不存在，自动将插件文件克隆/复制到 Plugins/UEEditorMCP/，并提示用户重新编译项目。
- 若 IDE 无法感知 UE 项目路径：
    - 提示用户手动将 UEEditorMCP 插件放入项目的 Plugins/ 目录并编译。
    - 提供仓库地址供用户下载。
2) Python 环境配置
UEEditorMCP 设计上使用 UE 引擎内置的 Python 3.11 创建 venv。
若 IDE 能检测到 UE 引擎路径（通过环境变量、注册表或用户配置）：
- 定位 UE 内置 Python：
    - Windows: Engine/Binaries/ThirdParty/Python3/Win64/python.exe
    - macOS: Engine/Binaries/ThirdParty/Python3/Mac/bin/python3
    - Linux: Engine/Binaries/ThirdParty/Python3/Linux/bin/python3
- 在插件目录下自动创建 venv 并安装 mcp 包：
```bash
<UE_Python> -m venv Plugins/UEEditorMCP/Python/.venv
# 激活 venv 后
pip install mcp
```
若 IDE 无法检测 UE 引擎路径：
- 提示用户在插件目录下执行 setup 脚本（Windows: setup_mcp.ps1 / setup_mcp.bat，macOS/Linux: 需提供等效 shell 脚本）。
- 或允许用户手动指定 UE 引擎根目录后重新触发自动配置。
3) 自动写入 MCP 配置
主 Server 配置：
```json
{
  "mcpServers": {
    "ue-editor-mcp": {
      "command": "<venv_python_path>",
      "args": ["-m", "server_unified"],
      "env": {
        "PYTHONPATH": "<插件目录>/Python",
        "PYTHONUTF8": "1"
      }
    }
  }
}
```
日志 Server 配置（可选，建议默认部署）：
```json
{
  "ue-editor-mcp-logs": {
    "command": "<venv_python_path>",
    "args": ["-m", "server_unreal_logs"],
    "env": {
      "PYTHONPATH": "<插件目录>/Python",
      "PYTHONUTF8": "1"
    }
  }
}
```
4) 部署完成反馈 — 提示：UE MCP 部署完成，启动 UE 编辑器后插件会自动在端口 55558 开启 TCP 服务。
#### 4.1.2 使用时行为
UEEditorMCP 的 C++ 插件在 UE 编辑器启动时会自动在端口 55558 启动 TCP Server，IDE 侧的 Python MCP Server 通过 stdio 启动后会自动连接该端口。无需用户手动操作。
IDE 侧只需确保在启动 MCP 会话时自动拉起 Python 进程即可。若连接 55558 失败，提示用户确认 UE 编辑器是否已打开且插件已启用。
#### 4.1.3 跨平台说明
- 原项目 setup 脚本仅提供 Windows（PowerShell / bat），IDE 实现需补充 macOS / Linux 的 Python venv 创建逻辑。
- UE 内置 Python 路径因平台而异（见上方路径表），IDE 需按平台处理。
- C++ 插件编译是 UE 工程本身的事，IDE 不需要处理编译过程。
### 4.2 可选附属插件：AssetCustoms（资产自动化）
#### 4.2.1 定位
AssetCustoms 是 UE 编辑器内的资产标准化 / Photoshop-SP 往返编辑插件，不是独立的 MCP Server，但与 UEEditorMCP 配合可提升 AI 辅助资产管线的能力。
#### 4.2.2 部署方式
- 在 UE MCP 部署流程中，向用户展示「推荐同时部署 AssetCustoms 插件」的选项（默认勾选推荐，但不强制）。
- 若用户选择部署：
    - IDE 能感知 UE 项目路径时：将插件克隆/复制到 Plugins/AssetCustoms/，执行 deploy.bat（Windows）或等效脚本（macOS/Linux 需 IDE 适配）。
    - IDE 无法感知时：提供下载地址和部署指引。
- 无需写入 MCP 配置（它不是 MCP Server）。
#### 4.2.3 依赖
- UE 内置 Python + Pillow / PySide6 等（由 deploy 脚本自动安装到插件内部目录，不污染系统环境）。
- UE 5.5+。
### 4.3 附属插件：Memory（项目记忆 MCP Server）
#### 4.3.1 定位
Memory 是独立的 Python MCP Server，为 AI 提供结构化的项目记忆管理（读写搜索备份压缩），支持多人协作。非 UE 专属，但与 UE 项目配合使用效果最佳。
#### 4.3.2 部署方式
- 在 UE MCP 部署流程中，向用户展示「推荐同时部署 Memory MCP Server」的选项。
- 部署流程：
    1. 下载 Memory 仓库到项目的 MCP/Memory/ 目录（或 IDE 统一管理目录）。
    2. Python 环境：优先复用 UE 内置 Python 创建 venv；或使用系统 Python (>= 3.10)。自动执行：创建 venv -> 安装依赖 (pip install -r requirements.txt)。
    3. 写入 MCP 配置：
```json
{
  "project-memory-mcp": {
    "command": "<venv_python_path>",
    "args": ["-m", "servers.memory_server", "--root", "<项目根目录>"],
    "env": {
      "PYTHONPATH": "<Memory目录>",
      "PYTHONUTF8": "1"
    }
  }
}
```
#### 4.3.3 跨平台说明
纯 Python，跨平台无特殊问题。原项目 deploy 脚本仅提供 PowerShell，IDE 需实现跨平台的 venv 创建和依赖安装逻辑。

## 5. Godot MCP 一键部署需求
由于godot引擎的场景、资产等文件格式的特殊性，其实不用mcp，ai也可以模拟大多“操纵editor”的行为。如果用户使用的是海外模型，可以推荐让用户尝试直接给出“更改场景信息”这类的命令，暂不部署mcp。
### 5.1 部署流程
1) 检测 Node.js 环境 — 同 Unity MCP（见 3.1 第 1 步），需 Node.js 可用。
2) 下载并构建 godot-mcp Server
- 从仓库克隆或下载 server/ 目录。
- IDE 自动执行构建：
```bash
cd <server目录>
npm install
npm run build
```
- 构建产物位于 server/dist/index.js。
- 存放位置建议：IDE 管理的统一目录（如 ~/.codebuddy/mcp-servers/godot-mcp/）。
3) Godot 编辑器插件部分
- 若 IDE 能感知 Godot 项目路径（识别 project.godot 文件）：
    - 自动将 addons/godot_mcp/ 复制到项目的 addons/ 目录。
    - 提示用户在 Godot 中启用插件：Project -> Project Settings -> Plugins -> 启用 Godot MCP。
- 若 IDE 无法感知：
    - 提示用户手动将 addons/godot_mcp/ 放入项目并在 Godot 中启用。
4) 自动写入 MCP 配置
```json
{
  "mcpServers": {
    "godot-mcp": {
      "command": "node",
      "args": ["<IDE管理的绝对路径>/godot-mcp/server/dist/index.js"],
      "env": {
        "MCP_TRANSPORT": "stdio"
      }
    }
  }
}
```
5) 部署完成反馈 — 提示：Godot MCP 部署完成，请确认已在 Godot 编辑器中启用 Godot MCP 插件。
### 5.2 使用时行为
Godot 插件在编辑器中启用后会自动启动 WebSocket 服务端，IDE 侧 Node 进程通过 stdio 启动后自动连接。连接失败时提示用户确认 Godot 编辑器是否已打开且插件已启用。
### 5.3 跨平台说明
Server 是纯 Node.js，Godot 编辑器插件是 GDScript，均跨平台无特殊处理。

## 6. 通用需求
### 6.1 MCP Server 自动启动
所有游戏引擎 MCP Server 在 IDE 中应支持自动启动，即：
- 当用户在 Chat / Agent 中使用到对应引擎的 MCP 工具时，IDE 自动拉起对应的 MCP Server 进程。
- 无需用户手动执行命令或点击按钮。
- Server 进程生命周期由 IDE 管理（随 IDE 关闭而关闭，或在空闲超时后自动回收）。
### 6.2 一键部署入口
建议在 IDE 中提供统一的「游戏引擎 MCP」部署入口，例如：
- 侧边栏 / 设置页新增「游戏引擎 MCP」板块。
- 展示三个引擎卡片（Unity / UE / Godot），用户点击即可触发部署。
- UE 卡片内额外展示 AssetCustoms 和 Memory 的可选勾选项。
### 6.3 配置自动写入
部署完成后，IDE 自动将 MCP Server 配置注入到 CodeBuddy 的 MCP 配置文件中，用户不需要手动打开或编辑任何 JSON 文件。
### 6.4 环境依赖自动处理

### 6.5 部署状态管理
- IDE 应记录各引擎 MCP 的部署状态（未部署 / 已部署 / 配置异常）。
- 支持重新部署（覆盖安装）和卸载（清理文件 + 移除配置）。
- 版本更新时，IDE 可检测远端仓库新版本并提示用户更新。
